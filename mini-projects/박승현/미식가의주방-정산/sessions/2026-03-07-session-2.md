## 팅커벨 핵심 답변 요약

### 세션 이전 프로토콜 버그 발견 & 수정
- 85%에서 프로토콜 시작 → 세션 파일/progress 작성 중 컨텍스트 소진 → 다음 세션에서 커밋 마무리
- 새 규칙 추가: "90% 미만이면 프로토콜을 현재 세션 안에서 끝까지 완료할 것"

### 구글시트 실전 적용
- Excel 파일 구글시트 업로드 완료
- Apps Script 설치 가이드 (확장프로그램 → 코드 붙여넣기 → 전송 버튼 만들기)
- 전송 테스트 성공 (데이터 시트에 데이터 추가됨)

### SUMPRODUCT 수식 깨짐 진단 & 수정
- 원인 1: Excel→구글시트 변환 시 수식 참조 깨짐
- 원인 2: 월/주차가 텍스트로 저장되어 숫자 비교 실패
- 해결: Apps Script v2 — submitData()에 Number() 강제변환 + setupFormulas()로 구글시트 네이티브 수식 재생성
- 코드 복사 시 이모지/화살표 함수 깨짐 → var 문법 + 이모지 제거 버전 제공

### 암묵지 5가지 도출
1. 사용자 동선을 먼저 그린다 (데이터 구조보다 사람 흐름)
2. 반복이 보이면 구조를 의심한다 (DRY 원칙)
3. 만들고 직접 써봐야 안다 (Build→Test→Fix)
4. 모르는 건 빠르게 위임, 판단은 내가 (PM 역량)
5. 프로세스 버그도 즉시 패치 (시스템 사고)
