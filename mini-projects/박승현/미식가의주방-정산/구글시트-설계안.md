---

## 변경 전 vs 변경 후

| | 변경 전 | 변경 후 |
|---|---|---|
| **정산 방식** | 공존공간이 매출에서 수수료만 빼고 지급 → 미식가가 임대료/관리비/공과금을 다시 입금 | 매출에서 **전부 차감** → 최종 금액만 지급 |
| **미수금 발생** | 미식가가 재입금 안 하면 미수금 (현재 4천만원) | 구조적으로 미수금 거의 없음 (현금매출분만) |
| **파일 구조** | 엑셀 3~4개 파일, 136개+ 시트 | 구글시트 1개 파일, 4개 시트 |
| **입력** | 매주 새 시트 생성 + 수동 계산 | 매주 시트1에 1줄 세부 입력 → 시트3에 정산 자동 출력 |

## 시트 구조 (v2 — 폼 입력 + 전송 + 자동 정산)

> **핵심 설계 원칙**: "사람이 하는 부분은 사람에 맞춰서 쉽고 직관적으로"
> - 시트1: **세로형 입력 폼 1개** — 노란 칸만 채우고 전송
> - 데이터 시트: 전송된 데이터가 쌓임 (숨겨도 됨)
> - 시트2: 정산 요약이 자동으로 계산
> - 시트3: 대시보드 (차트/KPI)

```
시트1 (입력 폼)  ──전송──→  데이터 (축적)  ──자동──→  시트2 (정산 요약)  ──자동──→  시트3 (대시보드)
  폼 하나만!       버튼       한 줄씩 쌓임              수수료/임대/지급 계산          차트/KPI
  입력 후 초기화                                                                    시트4 (미수금)
```

---

### 시트1: 주간 입력 (세로형 폼)

**폼은 항상 하나. 입력 → 전송 → 초기화 → 다음 주차 입력.**

```
📋 미식가의 주방 — 주간 정산 입력
🟡 노란 칸에 입력 → 하단 [전송] 버튼 클릭

📅 기본 정보
      월        [ 🟡 ]  ← 1~12
      주차      [ 🟡 ]  ← 1~5
      시작일    [ 🟡 ]  ← 예: 3/3
      종료일    [ 🟡 ]  ← 예: 3/9

💰 매출
      카드매출  [ 🟡 ]  원
      현금매출  [ 🟡 ]  원

⚡ 공과금 (3주차에 지난달분 입력)
      전기요금  [ 🟡 ]  원 (고지서 그대로)
      가스요금  [ 🟡 ]
      수도요금  [ 🟡 ]
      → 95%    [ ⬜ ]  ← 자동 계산

📝 기타 (발생할 때만)
      정화조    [ 🟡 ]
      회식매출  [ 🟡 ]
      기타비용  [ 🟡 ]
      비고      [ 🟡 ]

    ┌──────────────────────┐
    │   ▶  전송             │  ← 구글시트에서 Apps Script 버튼으로 연결
    └──────────────────────┘

📌 최근 전송 기록 (최근 5건)
    월/주 | 카드매출 | 현금매출
```

#### 전송 버튼 동작 (Apps Script)

1. 필수값 검증 (월, 주차, 매출 중 하나 이상)
2. 중복 체크 (같은 월/주차 → 덮어쓸지 확인)
3. 데이터 시트에 한 줄 추가 (월/주차 순서대로 정렬)
4. 입력 폼 초기화
5. 완료 알림

#### 정산 타이밍 규칙

| 주차 | 차감 항목 | 설명 |
|---|---|---|
| **1~2주차** | 수수료만 | 카드15%+현금13%+VAT |
| **3째주** | 수수료 + **지난달 공과금** | 공과금 고지서 확정 후 정산. 미식가 최종 지급에 반영 |
| **4째주** | 수수료 + **임대료+관리비+캡스** | 임대료 300만(+VAT), 관리비 55,000, 정수기 15,900, 캡스 50,000 |

> 1~2주차는 수수료만 빠지니 미식가 지급액이 많고, 3주차에 공과금, 4주차에 임대료가 빠지는 구조.

### 시트2: 대시보드

#### 상단 — KPI 카드 (4개)

```
┌─────────────┐ ┌─────────────┐ ┌─────────────┐ ┌─────────────┐
│  이번달 매출  │ │ 이번달 수익  │ │  미수금 잔액  │ │ 공과금 누적  │
│ 73,076,800  │ │ 12,450,000  │ │ 36,867,242  │ │  2,023,812  │
│   ▲ +5.2%   │ │   ▲ +3.1%   │ │   ▼ -15%    │ │   ─ 동일    │
└─────────────┘ └─────────────┘ └─────────────┘ └─────────────┘
```

#### 차트 1 — 월별 매출 추이 (막대 그래프)

- X축: 월 (1~12)
- Y축: 금액
- 막대: 카드매출 (파랑) + 현금매출 (초록) 누적 막대
- 선: 전년도 동월 매출 (회색 점선, 2025년 데이터 있으므로)

#### 차트 2 — 공존공간 수익 구성 (도넛 차트)

- 카드수수료 (15%)
- 현금수수료 (13%)
- 고정임대료
- 관리비
- 공과금

#### 차트 3 — 주간 매출 추이 (꺾은선 그래프)

- X축: 주차 (1월1주 ~ 12월5주)
- Y축: 주간 총매출
- 선: 올해 매출 (파랑), 전년도 (회색 점선)

#### 차트 4 — 미수금 잔액 추이 (영역 그래프)

- X축: 월
- Y축: 미수금 잔액
- 목표: 0원까지 감소하는 추이

#### 하단 — 월별 정산 요약 테이블

| 월 | 총매출 | 수수료 | 임대료 | 관리비 | 공과금 | 총차감 | 지급액 | 현금수령 | 미수금차감 | 미수금잔액 |
|---|---|---|---|---|---|---|---|---|---|---|
| 1월 | | | | | | | | | | |
| ... | | | | | | | | | | |
| 합계 | | | | | | | | | | |

### 데이터 시트 (숨김 가능)

전송 버튼으로 들어온 데이터가 한 줄씩 쌓이는 저장소.
사용자가 직접 건드릴 일 없음. 필요하면 숨겨놓기.

| 컬럼 | 내용 |
|---|---|
| A~D | 월, 주차, 시작일, 종료일 |
| E~F | 카드매출, 현금매출 |
| G~I | 전기, 가스, 수도 |
| J~L | 정화조, 회식매출, 기타비용 |
| M | 비고 |
| N | 입력일시 (자동) |

---

### 시트2: 정산 요약 (전체 자동)

> **사용자가 직접 입력하지 않는다.**
> 데이터 시트에서 월/주차별로 매칭하여 정산 계산을 자동으로 완료.

**한 줄 = 한 주. 12개월 × 5주 = 60행 + 월 소계 12행.**

| 컬럼 | 이름 | 설명 |
|---|---|---|
| A~D | 월, 주, 시작일, 종료일 | 고정값 + 데이터 시트 참조 |
| E~G | 카드매출, 현금매출, 총매출 | 데이터에서 끌어옴 |
| H~K | 카드수수료15%, 현금수수료13%, 합계, VAT | 자동 계산 |
| L~M | 임대료, 임대료VAT | **4주차에만** 300만+VAT |
| N | 관리비+고정비 | **4주차에만** 120,900원 |
| O | 공과금 95% | **3주차에 입력된 값** 기반 자동 |
| P | 기타비용 | 정화조+회식5%+기타 |
| Q | **총 차감액** | 수수료+임대+관리+공과+기타 |
| R | **미식가 카드지급** | 카드매출 - 총 차감 |
| S | **현금 수령액** | 현금매출 - 현금수수료 - VAT |
| T | 비고 | |

## 추후 과제 (자동화)

> 승현님 요청: "숫자, 영수증 사진만 입력하면 알아서 정산이 시트로 들어가는 것"

| 단계 | 내용 | 도구 |
|---|---|---|
| 1단계 (지금) | 구글시트 수동 입력 + 자동 계산 | 구글시트 |
| 2단계 | POS 데이터 자동 연동 → 시트 자동 입력 | Apps Script / Zapier |
| 3단계 | 영수증 사진 → OCR → 자동 입력 | Google Vision API / 슈파 |
| 4단계 | 정산 보고서 자동 생성 + 알림 | Apps Script + 카톡/이메일 |
