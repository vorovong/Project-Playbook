# "기획 모드" — 사업 기획 논의 진입

> 승현님이 "기획 모드"라고 하거나 사업 관련 논의를 시작할 때 실행.
> 내부전략.md + GONGZONE 모듈 + 기획/ 파일을 맥락에 맞게 로드.

---

## 실행 순서

| 순서 | 행동 | 누가 | 설명 |
|------|------|------|------|
| 1 | **맥락 확인** | 메인 | 내부전략.md(이미 로드) 기반으로 사업 구조 파악 |
| 2 | **영역 파악** | 메인 | "어떤 사업/기획 논의할까요?" — 아래 맵에서 해당 영역 특정 |
| 3 | **모듈 로드** | 메인 | 해당 영역의 파일들만 로드 (컨텍스트 다이어트) |
| 4 | **기획 브리핑** | 서브에이전트 | 해당 영역의 최근 progress + git log 수집 |
| 5 | **작업 시작** | 메인 | 기획 모드 암묵지 자동 적용하며 진행 |

## 서브에이전트 프롬프트 (기획 브리핑)

```
수집 항목:
1. progress/박승현/ 에서 "{영역 키워드}" 포함된 최신 기록 → 어디까지 했는지
2. git log -5 --oneline -- "mini-projects/박승현/*{관련 경로}*" → 최근 변경
3. (IR/제안서인 경우) mini-projects/박승현/ 하위 관련 프로젝트 폴더 → 현재 상태

반환 형식:
---진행 상태--- (1~3줄)
---최근 변경--- (커밋 목록)
---다음 할 것--- (1~3줄)
```

## 기획 모드 암묵지 (자동 적용 — 25개)

> 전체 목록: `팅커벨/OWNER/암묵지.md` 트리거+모드 매핑 참조

| 층위 | 번호 | 핵심 |
|------|------|------|
| 🔵 가치 | V2, V3, V4, V5, V8, V9, V10, V11 | 에센스=방향, 본질 먼저, 구조 필요, 극초기 넘김 |
| 🟢 프로세스 | P1, P2, P3, P4, P5, P9, P12, P13 | 벤치마킹→크리틱→제안, 메인먼저, 전략/기획 분리 |
| 🟡 역할 | R7, R8, R9 | 약점→프레이밍, 감 검증, 예방적 스캔 |
| 🟣 감각 | S3, S5, S6, S8, S10, S12 | 에코챔버, 맥락 부적합, UI 거부, 빠진 것 |

### 핵심 발동 규칙

- **에센스 불일치 감지**: 기획 내용이 내부전략.md의 에센스와 어긋나면 즉시 크리틱 (V2, V5)
- **전략/기획 혼재 방지**: 전략 논의와 기획 실행이 섞이면 분리 제안 (P5)
- **"프론티어" 트리거**: 이 단어가 나오면 P13 자동 발동 → 프론티어 분석
- **숫자 검증**: 예산/정산 관련 → S7 적용 (깔끔하면 의심)
- **글쓰기**: 기획서/제안서 작성 시 writing-style.md 반드시 참조

## 출력 형식

```
> 🧚 기획 모드 — {영역 이름}

--- 🔵 현재 상태 ---
해당 영역의 진행 상태 요약

--- 🔵 다음 할 것 ---
이번에 다룰 의제/작업

--- 📋 로드한 맥락 ---
로드한 파일 목록 (✅ 표시)

기획 모드 암묵지 25개 적용 중. 시작할게요!
```

## 규칙

- 기획 모드에서 개인 이야기(LIFE) 나오면 모드 전환 안내 (V7)
- 방향 결정됐으면 바로 실행 (P6) — "진행할까요?" 묻지 않음
- 새 기획 아이디어 → 초안 먼저 만들고 성장형 업데이트 (P3)
- 기획서팩토리 분석 필요하면 → "기획서 학습" 주문으로 전환
