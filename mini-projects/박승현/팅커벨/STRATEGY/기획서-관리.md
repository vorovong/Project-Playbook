## 목적

박승현은 기획자로서 **청중별 맞춤 기획서/보고서**를 만드는 일이 많다.
같은 본질이라도 읽는 사람에 따라 프레이밍, 강조점, 협조 요청이 달라진다.

이 파일은:
1. 어떤 기획서가 있는지 추적
2. 각 기획서의 청중과 목적을 기록
3. 내부 전략과의 연결점을 명시
4. 버전 관리 (같은 청중에게 업데이트 버전을 줄 때)

## 관리 규칙

1. **새 기획서 작성 시**: 이 파일에 등록 + 내부전략.md와의 연결점 확인
2. **기획서 업데이트 시**: 버전 번호 + 변경 사항 기록
3. **협조 요청 추적**: 요청한 것이 진행되는지 상태 업데이트
4. **청중 확장 시**: 새로운 청중(화성시장, 중기부, 투자자 등) 섹션 추가

## 팅커벨 역할

- 승현님이 새 기획서를 만들 때 → 내부전략에서 해당 청중에 맞는 포인트 추출 도움
- 기존 기획서 업데이트 시 → 이전 버전과 비교, 변경점 정리
- 협조 요청 후속 관리 → "이건 아직 응답 안 왔어요" 리마인드
