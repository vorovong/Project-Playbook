> **승현님**: 지금 내가 옆에 세션에서 '주문(시동어)'를 체계화 하고 있는 스킬이랑 다를게 뭘까? 체계화를 위해 과연 이게 맞는걸까? 내가 잘못알아서 정리를 잘못하는걸까? 튜터모드로 알려줘.(지금 이런 튜터 모드도 주문이나 스킬 아닐까?)

**팅커벨**: 핵심 튜터링 실행.
- **주문 = 설계(메뉴판)**: 어떤 명령이 있고 각각 뭘 하는가
- **Skill = 실행(버튼)**: 설계된 주문을 `.claude/commands/`에 연결하는 기술적 수단
- 비유: 메뉴판이 확정 안 됐는데 키오스크 버튼 만들면, 메뉴 바뀔 때마다 버튼도 다시 만들어야 함
- 순서: 체계화(메뉴판) 먼저 → Skill(버튼) 나중. 팅커벨이 순서를 뒤집은 것이 실수
- "튜터 모드도 주문이다" = 승현님의 메타인지. AI 행동을 바꾸는 모든 것 = 주문

실행 방식 4가지 비교:
| CLAUDE.md | 벽에 붙인 사훈 | 항상 읽힘 | 무거움 |
| commands/ | 서랍 속 매뉴얼 | AI가 찾아 읽어야 함 | 까먹을 수 있음 |
| Hook | 문 앞 경비원 | 시스템 자동 실행 | 보장 |
| Skill | 키오스크 버튼 | `/명령` 입력 | 보장 |
