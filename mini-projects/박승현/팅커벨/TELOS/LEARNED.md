## 15년 현장에서 배운 것

### 사업에서

| 배운 것 | 맥락 |
|---------|------|
| 뜻만으로는 사람이 버틸 수 없다 | 활동가, 사회적기업 경험 |
| 리소스가 재생산되지 않으면 동료가 떠난다 | 반복된 팀원 이탈 |
| 상인의 프로덕트와 서비스가 콘텐츠가 된다 | 신도시양조회 경험 |
| 콘텐츠가 되면 산 꼭대기까지 찾아온다 | 상권 변화 관찰 |
| 프로덕트와 서비스가 모인 것이 상권이다 | 팔딱산 운영 |
| 직접 해봐야 구조를 알 수 있다 | 1.0에서 2.0으로의 전환 |
| 민간 중심이 가능하다 | 글로컬상권창출팀 |

### AI에서

| 배운 것 | 맥락 |
|---------|------|
| 맥락(Context)이 결과의 질을 결정한다 | 클로드 코드 학습 |
| 세션 간 기억은 MD 파일로 저장해야 한다 | 안녕/잘자 규칙 리셋 경험 |
| 마크다운이 AI의 유니버설 인터페이스다 | 패스트 무버 리서치 |
| 스캐폴딩(구조)이 모델 지능보다 중요하다 | PAI 학습 |
| 설계가 실행보다 중요하다 | FSM, 팅커벨 설계 과정 |

## 팅커벨 메모

승현님의 학습 방식: 직접 해보고 → 실패하고 → 구조를 발견하고 → 시스템으로 만든다.
이론에서 출발하지 않고 경험에서 출발하는 귀납적 학습자.
