## 에센스 심화

- 시장 보고에서는 "AI SaaS"로 표현했지만, 회의에서 드러난 실체는 다름
- PT 사인 시스템처럼 가게마다 다른 문제를 해결
- **차별점의 근본**: 공존공간이 2년간 142회 협업하며 쌓은 "각 가게가 뭘 필요로 하는지 아는 현장 지식"
- 이 현장 지식이 없으면 그냥 또 하나의 generic SaaS → 경쟁 불가

## "파트너"의 두 가지 의미 (2026-03-02 승현님 직접 정의)

1. **브랜딩** — 사장님들에게 "도구"가 아니라 "파트너"라는 친근한 이미지. 위에서 내려다보는 지원이 아니라 옆에서 같이 하는 관계.
2. **실질적 효과** — AI를 활용해서 사장님들의 잡무를 확 줄여주는 것. 세무, 마케팅 같은 것부터 시작. 구체적인 잡무 목록은 슈파 프로젝트 하면서 발굴.

## 가격 전략 (2026-03-02 승현님 직접 정의)

> **핵심**: 서비스 이용료는 **무료거나 소액**. AI 생태계에서는 **유저수 확보가 더 중요**하다.

수익 모델의 본질:
- 서비스 자체로 돈을 버는 게 아님
- 유저수 확보 → **데이터** 축적 → 데이터 기반 수익
- 유저수 확보 → **피지컬 AI** (로봇 등) 연결 → 하드웨어 수익
- 이건 기존 SaaS "구독료"와 근본적으로 다른 접근

**에센스 연결**: "파트너로서 직접 효과"를 무료/소액으로 제공 → 자연스럽게 유저 확보 → 현장 데이터가 쌓임 → 이 데이터가 진짜 자산

## 시장 보고 vs 회의에서 드러난 실체

| 구분 | 시장 보고 (대외용) | 회의 (내부 실체) |
|------|-------------------|-----------------|
| 포지셔닝 | 소상공인 AI SaaS | 소상공인 맞춤 자동화 |
| 서비스 형태 | SNS·리뷰·매출리포트·예약 자동화 | 가게별 고유 문제 해결 (PT 사인 시스템 등) |
| 가격대 | 구독료 (미정) | 무료/소액 → 유저수 → 데이터/피지컬 AI |
| 차별점 | 회원 증가 → 운영비 분산 | 공존공간의 현장 지식 = 진입 장벽 |

## 두 가지 모델의 공존 가능성

```
Layer 1: 공통 플랫폼 (SNS·리뷰·매출 — 무료/소액, 유저 확보)
  ↑ 이것으로 소상공인 접점 확보 + 데이터 축적

Layer 2: 맞춤 자동화 (가게별 커스텀 — 프로젝트형)
  ↑ 이것이 진짜 차별점 + 현장 데이터 심화
```

## 아직 정리 안 된 것

- [ ] Layer 1과 Layer 2의 우선순위 (MVP를 어디서 시작할지)
- [ ] 맞춤 자동화의 반복 가능한 패턴 추출 (매번 처음부터 만들지 않으려면)
- [ ] 공존공간 2년 현장 데이터를 슈파에 어떻게 구조화할지
- [ ] 비전 AI 활용 가능성 (제조업 데이터셋 — 2/27 회의)
- [ ] 사장님 잡무 목록 발굴 (세무, 마케팅 외)
- [ ] 경쟁 제품과의 포지셔닝 차이 정리
