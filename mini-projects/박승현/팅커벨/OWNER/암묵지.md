# 암묵지 — 승현님의 판단 패턴 저장소

---
status: v1.0
created: 2026-03-08
updated: 2026-03-08
---

> 승현님이 무의식적으로 적용하는 판단·행동 패턴을 발견하고, 검증하고, 팅커벨에 심는 곳.
> "요리사의 '대충 넣어'를 정확한 레시피로 바꾸는 과정."

---

## 분류 체계 — 4층위

| 층위 | 뭘 담는가 | 질문 키워드 |
|------|----------|------------|
| 🔵 **가치** | 왜 하는가. 판단의 근거 | "왜 그렇게 판단하셨어요?" |
| 🟢 **프로세스** | 어떻게 하는가. 반복 패턴 | "머릿속에서 어떤 순서로 생각하세요?" |
| 🟡 **역할** | 누가 하는가. 위임/판단 경계 | "이건 왜 직접 하시고, 저건 왜 맡기셨어요?" |
| 🟣 **감각** | 언제 움직이는가. 직관/트리거 | "이상하다고 느낀 순간이 있었어요? 뭐가 걸렸어요?" |

## 포착 규칙

- **안녕**: 지난 세션 후보 중 2~3개 검증 질문
- **잘자**: 이번 세션에서 드러난 패턴 2~3개 발견 질문
- **특별 세션**: 일괄 스캔 시 10개+ 집중 질문
- 검증 안 된 후보는 3회 세션 경과 후 자동 리마인드

### 🟢 프로세스 — 어떻게 하는가 (13개)

| # | 패턴 | 근거 | 등급 |
|---|------|------|------|
| P1 | **⭐ 벤치마킹 → 크리틱 → 명시적 제안 루프** — 도메인/프론티어/패스트무버 비교 분석 → 크리틱 → 명시적 제안의 3단계. 거의 모든 중요 결정에서 반복 (5회+) | "이 분야의 도메인을 가지고 있거나 프론티어, 패스트무버들의 비교 분석을 통해 크리틱하고 너가 다시 명시적으로 제안해줘" (IR 03-04 x3, 팅커벨 03-07, 암묵지 03-08) | ⚙️ |
| P13 | **🔍 "프론티어" = 방향 탐색형 분석** — 승현님이 방향/아이디어를 언급 + "프론티어"라고 하면, 해당 분야의 프론티어·패스트무버·온라인 동향을 비교 분석 → 크리틱 기반 결론/제안. "크리틱" 주문(결과물 평가)과의 경계는 미확정 — 사용 맥락 관찰 중 | "프론티어라고 하면 자동으로 내가 언급했던 방향을 분석해서 추론하고 그 분야의 도메인을 가지고 있는 프론티어, 패스트무버, 온라인의 분석 등을 다양하게 비교 분석해서 크리틱한 것을 바탕으로 결론이나 제안" (기획모드 03-08) | 🔍 |
| P2 | **역질문으로 시작** — AI가 먼저 조사하고 확인 질문하게 만듦 | "너가 찾을 수 있는 방식을 동원해서 역으로 질문해줘" (03-01) | ✅ |
| P3 | **초안 먼저, 성장형 업데이트** — 문서/설계도를 초안 → 반복 개선 | "일단 초안을 만들고 계속 필요시 성장형으로 업데이트 해주면 좋겠어" (03-01) | ✅ |
| P4 | **메인 먼저, 모듈 분리** — 처음부터 큰 그림 → 세부로 나눔 | "메인 방향 만들고 업무 파트마다 분리하는 게 좋지 않아?" (03-01, 03-03, 팅커벨 03-01) | ✅ |
| P5 | **전략(방향)과 기획(실행)은 분리** — 지도에 상세 넣으면 읽기 어려움 | "전략과 기획은 분리해야 한다. 지도에 모든 상세를 넣으면 지도가 읽기 어려워진다" (내부전략 03-02) | ✅ |
| P6 | **방향 정해지면 확인 질문 불필요, 바로 실행** — "진행할까요?" = 로봇 신호 | "팅커벨 거기 있어? 또 로봇 느낌인데... 적용해줘" (IR 03-04) | ⚙️ |
| P7 | **이미 아는 건 초안 먼저, 모르는 것만 질문** | "너 이미 너무 잘 알고 있는데... 왜 질문해? ㅠ 조금 아쉽다" (IR 03-04) | ✅ |
| P8 | **실수 3회 반복 → 자동 구조적 해결 프로토콜** | "3번째 같은 실수 하면 왜 그랬는지 + 벤치마킹 + 크리틱 + 제안 자동 실행해줘" (팅커벨 03-01) | ✅ |
| P9 | **시스템 만들면 작동 방법까지 같이 설계** | "안녕"/"잘자"에 LIFE 연결을 승현님이 먼저 제안 (팅커벨 03-02) | ✅ |
| P10 | **도식화/구조적 표현 우선** — 텍스트 나열 < 시각적 구조 | "이해 못했어. 도식화된 것이 내가 더 잘 이해할 수 있어" (슬림화 03-07) | ✅ |

| P12 | **답 확인 후 다음으로** — 이해 안 되면 넘어가지 않음 | "이렇게 단순하게 나올 수가 없을 텐데? 점검해줘" (IR 03-04) | ✅ |
