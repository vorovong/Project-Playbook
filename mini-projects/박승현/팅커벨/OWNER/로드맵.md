## v1.0 — 과도기 (현재)

### 뭘 하는 단계인가

- 조각을 쌓는다 (지식저장소/조각함)
- 에센스를 검증한다 (대화를 통한 심화)
- 승현님을 파악한다 (Lv.1 → Lv.2)
- 수호 프로토콜을 실전에서 검증한다
- 피드백 순서(크-칭-크-제)를 체득한다
- TELOS 리뷰를 완료한다

### 현재 구조

```
팅커벨/ (MD 파일 38개)
├── tinkerbell.md (190줄) ← 작동 명세서. 200줄 한계 근접
├── TELOS/               ← 초안. 리뷰 안 됨
├── LIFE/                ← 시작됨. health.md가 가장 풍부 (228줄)
├── STRATEGY/            ← 에센스 v0.4. 승현님 교정 반영 완료
├── 기획/                ← 4개 사업 구조화 완료
├── OWNER/               ← 상태추적 + 성장큐 + 로드맵(이 파일)
├── 지식저장소/          ← 조각함만 존재. 조각 2개
├── LOG/                 ← 실수 노트
├── REFERENCES/          ← 참고 인물 3인
└── sessions/            ← 대화 원문 (3/1: 147KB)
```

- [ ] 에센스 확정 (v0.4 → v1.0)
- [ ] 조각함 10개+ 쌓기

- [ ] 지식저장소 대화 저장 구조 확립 (프롬프트 + 원본 보존)
- [ ] tinkerbell.md 200줄 초과 전에 분리 계획

## v1.5 — 클로드 코드 스킬 전환

### 전환 조건 (전부 충족 시)

- [ ] 에센스 확정 (v1.0)
- [ ] TELOS 리뷰 완료
- [ ] 조각함 10개+
- [ ] 수호 프로토콜 실전 검증 완료
- [ ] 승현님 파악 레벨 Lv.2+

### 핵심 과제

- 현재 구조(38개 MD)를 스킬로 압축하는 설계
- 팅커벨은 "스킬"보다 "운영체제"에 가까움 → 어떻게 스킬 프레임에 맞출지
- 매 세션 자동 로드 vs 필요 시 로드 구분
- AI 이식성 유지 (클로드 코드에 종속되지 않게)

### 아직 모르는 것

- 스킬의 기술적 한계 (파일 수, 용량, 로드 속도)
- SKILL.md로 분리할 때 참조형(규칙) vs 작업형(워크플로우) 구분
- 지식저장소가 스킬 안에 들어갈 수 있는지, 외부 참조여야 하는지

## v2.0 — FSM Phase 3 (서브에이전트 + 자동화)

### 비전

- 팅커벨이 서브에이전트들의 대장으로서 실제 작동

- 권한 매트릭스 실제 적용

### 전환 조건

- v1.5 안정 운영
- 폐관수련 8주 완료

## 팅커벨 메모

이 로드맵은 고정이 아니다. 조각이 쌓이면서 경로가 바뀔 수 있다.
중요한 건 버전 넘버가 아니라, **각 단계의 전환 조건이 진짜로 충족됐는지**.
조건 없이 올라가면 빈 껍데기가 된다.
