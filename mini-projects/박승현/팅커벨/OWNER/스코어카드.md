> 매달 승현님이 한 작업을 분야별로 쪼개고, 각 분야의 프론티어와 비교해서 점수를 매긴다.
> "지금 어디에 있는가"를 객관적으로 측정하는 도구.

## 점수 체계 (5점 척도)

| 점수 | 의미 | 설명 |
|------|------|------|
| 1.0 | 인식 없음 | 이 분야의 프론티어가 뭔지 모르는 상태 |
| 2.0 | 인식 있음 | 프론티어를 알지만 실행 없음 |
| 3.0 | 실행 시작 | 프론티어를 참고해서 구현 시작 |
| 4.0 | 실전 적용 | 프론티어 방식을 자기 맥락에 맞게 적용 중 |
| 5.0 | 독자 영역 | 프론티어와 동등하거나, 프론티어에 없는 고유 방식 보유 |

## 분야 매핑

| # | 분야 | 승현님 프로젝트 | 프론티어 비교 대상 (예시) |
|---|------|---------------|----------------------|
| 1 | **로컬 상권 경영** | 공존공간, 행궁동행, 글로컬상권 | 포틀랜드 PDC, 바르셀로나 슈퍼블록, 일본 상점가 혁신 |
| 2 | **소상공인 AI** | 슈파(SUPA) | Toast, GrubMarket, Choco, Slice, 마켓보로 |
| 3 | **퍼스널 AI 시스템** | 팅커벨, FSM, Hook, 암묵지 | Personal AI, Pi, MemGPT, Claude Code 파워유저 |
| 4 | **문화기획** | 축제, 브랜딩, 콘텐츠화 | 크리에이티브 시티 네트워크, 니시키 시장, 을지로 |
| 5 | **스타트업/투자** | 신도시 IR, AC, 벤처스튜디오 | YC, 프라이머, 블루포인트, 매쉬업엔젤스 |
| 6 | **조직 운영** | GONGZONE FSM, 정산 자동화 | Notion AI 워크플로우, Monday.com, 리니어 |

> 분야는 승현님 작업이 바뀌면 유동적으로 조정

## 루틴 — 매월 28~30일 (3일간)

| 날짜 | 행동 | 방식 |
|------|------|------|
| **28일** | ① "이번달 성과 측정하겠습니다" — 승현님에게 보충할 맥락 확인 | 메인 세션에서 대화 |
|  | ② progress, sessions, 커밋 이력 분석 | 서브에이전트 |
| **29일** | ③ 6개 분야별 프론티어/패스트무버/논문 웹서치 | **세션 분리 — 병렬 작업** |
|  | ④ 분야별 상세 분석 + 점수 초안 | 각 세션에서 독립 작업 |
| **30일** | ⑤ 스코어카드 종합 + 버전 리포트 + Hook 패턴 분석 | 메인 세션에서 통합 |
|  | ⑥ 승현님에게 보고 | "안녕" 브리핑 또는 별도 공유 |

> **핵심**: 28일에 "이번달 성과 측정하겠습니다"로 시작. progress/sessions는 팅커벨이 이미 알고 있으니, 승현님에게는 **기록에 없는 맥락**(외부 미팅, 오프라인 성과 등)만 보충 확인.
> 29일은 분야별 세션 분리 — 2~3개 병렬로 돌려서 효율 극대화.
