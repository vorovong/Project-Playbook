# 세션 관리 규칙

> 핵심 전환: "끝에 한꺼번에" → "하면서 계속 저장"

---

## 중간저장 — 세션의 심장박동

> 작업 완료 → 커밋 시 → progress도 함께 갱신. 이게 전부.

### 언제 실행?

| 트리거 | 설명 |
|--------|------|
| **작업 단위 완료** | 기획서 크리틱 끝 → 과업지시서 전환 등 자연스러운 전환점 |
| **승현님 요청** | "정리해줘", "커밋해줘", "중간저장" |
| **팅커벨 자동** | 커밋할 때 progress도 함께 갱신 (항상) |

### 중간저장 내용 (5단계)

| 순서 | 행동 | 설명 |
|------|------|------|
| 1 | **progress 갱신** | `docs/templates/progress-template.md` 양식대로. 이 시점까지의 대화 흐름 + 결과 |
| 2 | **내일 할 일** | progress "어려웠던 것 / 다음에 할 것"에 기록 |
| 3 | **성장큐 업데이트** | `OWNER/성장큐.md`에 더 알고 싶은 것 기록 |
| 4 | **schedule 체크** | 완료 항목 ✅ |
| 5 | **커밋 & 푸시** | 결과물 + progress + 성장큐 함께 |

> **핵심**: 중간저장이 끝나면, 이 시점에서 세션이 죽어도 다음 세션이 이어받을 수 있다.

---

## 방어선 — 컨텍스트 소진 방지

| 방어선 | 시점 | 행동 | 구현 |
|--------|------|------|------|
| **0차** | 작업 단위 완료 시 | **중간저장** — progress + 커밋 & 푸시 | 팅커벨 습관 |
| **1차** | ~85% (자동 compact) | **pre-compact.sh가 자동 커밋 & 푸시** | Hook (자동) |
| **2차** | 수동 | 승현님이 "세션 이전 프로토콜" → 세션 파일 정리 + 커밋 | 주문 (수동) |

> **0차가 핵심**. 중간저장을 잘 하면 1차·2차가 발동할 일이 줄어든다.

---

## "세션 이전 프로토콜" — 대화 원본 보존 + 커밋 & 푸시

**수동 트리거**: 승현님이 "세션 이전 프로토콜"이라고 말하면 즉시 실행

| 순서 | 행동 | 설명 |
|------|------|------|
| 1 | **승현님 프롬프트 원본 보존** | sessions/ 파일에 **원문 그대로** 기록. 요약·편집·순서 변경 금지 |
| 2 | **팅커벨 답변 핵심 기록** | 승현님 프롬프트와 1:1 매칭 |
| 3 | **작업 파일 저장** | 진행 중이던 파일 최신 상태 저장 |
| 4 | **progress 업데이트** | 해당 프로젝트 progress 갱신 |
| 5 | **커밋 & 푸시** | 전체 변경사항 커밋 & 푸시 완료 |

> **기획자의 프롬프트 원본 = 기획의 뿌리.** 훼손 금지. 이 실수 2회 기록 (2026-03-04).

---

## 자정 넘어 연달아 작업할 때 — 보고 합산

자정을 넘겨 연달아 작업하면, "안녕" 브리핑에서 **전날 + 오늘 보고를 합산**.
날짜가 바뀌었다고 전날 작업 내용을 빠뜨리지 않는다.

---

## 세션 라이프사이클 (v2)

```
세션 시작 ("안녕")
  ├─ 팅커벨 3파일 로드
  ├─ 브리핑 (서브에이전트 수집)
  ├─ 🔮 이전 세션 암묵지 질문 (여기서!)
  └─ 상태 체크
       │
       ▼
작업 진행
  ├─ 작업 A 완료 → [중간저장] → 커밋 & 푸시 ✅
  ├─ 작업 B 완료 → [중간저장] → 커밋 & 푸시 ✅
  └─ (85% 도달 시 → pre-compact Hook 자동 저장)
       │
       ▼
세션 종료 ("/잘자") — 3단계
  ├─ progress 최종 확인 (추가분만)
  ├─ 커밋 & 푸시 (미커밋 있으면)
  └─ 조언 + 인사
```
