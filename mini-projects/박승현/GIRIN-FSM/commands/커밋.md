# "커밋" — 커밋 & 푸시 주문

> 현재 작업물을 커밋하고 푸시. progress 동반 규칙 자동 적용.

---

## 실행 순서

| 순서 | 행동 | 설명 |
|------|------|------|
| 1 | **git status** | 변경/추가된 파일 목록 확인 |
| 2 | **progress 체크** | mini-projects/ 변경 있으면 → progress/ 갱신 필요 여부 확인 |
| 3 | **커밋 메시지 제안** | 변경 내용 기반 메시지 초안 → 승현님 확인 |
| 4 | **git add + commit + push** | 승인 후 실행 |
| 5 | **결과 확인** | git status로 clean 상태 확인 |

---

## 자동 적용 규칙

| 규칙 | 내용 |
|------|------|
| **progress 동반** | mini-projects/ 변경 → 관련 progress/ 변경도 같은 커밋 또는 동시 푸시 |
| **보안 파일 제외** | .env, credentials, 키 파일 → 커밋 대상에서 자동 제외 + 🟡 경고 |
| **커밋 메시지 스타일** | git log 최근 5개 참조하여 기존 스타일 유지 |

---

## 출력 형식

```
> 🧚 커밋

--- 변경 파일 ---
📁 {파일 목록, 개수}

--- 🟡 progress ---
✅ progress 갱신 완료 / 🟡 progress 갱신 필요 — 먼저 갱신할까요?

--- 커밋 메시지 ---
"{메시지 초안}"

이대로 커밋 & 푸시할까요?
```

---

## 규칙

- 변경 없으면 (clean) → "커밋할 게 없어요~" 로 끝
- progress 갱신이 필요한데 안 됐으면 → 먼저 progress 갱신 제안
- 커밋 메시지는 반드시 승현님 확인 후 실행
- 보안 파일 감지 시 → 절대 커밋하지 않고 🟡 경고
