# "안녕" — 세션 시작 명령어

> commands.md에서 분리. 승현님이 "안녕"이라고 하면 실행.

---

## 실행 순서

| 순서 | 행동 | 누가 | 설명 |
|------|------|------|------|
| 0 | **현재 시간 확인** | 메인 | `date` 명령어로 현재 날짜·시간·요일 확인 |
| 1 | **git pull** | 메인 | 최신 변경사항 먼저 받기 |
| 2 | **팅커벨 로드** | 메인 | 아래 3개 파일을 읽고 세션 내내 유지 |
|   | ① `mini-projects/박승현/팅커벨/tinkerbell.md` | | 팅커벨 정체성 + 말투 + 불약 |
|   | ② `mini-projects/박승현/팅커벨/TELOS/MISSION.md` | | 승현님의 본질. 모든 판단의 출발점 |
|   | ③ `mini-projects/박승현/팅커벨/STRATEGY/내부전략.md` | | 에센스 4개 + 현재 사업 구조 |
| 3 | **🧹 브리핑 수집 심부름꾼** | 서브에이전트 | 아래 항목을 전부 읽고 **요약 메모만 반환** |
| 4 | **브리핑 출력** | 메인 | 심부름꾼 요약 + 팅커벨 3파일 기반으로 브리핑 작성 |
| 5 | **승현님 오늘 상태 체크** | 메인 | 컨디션/감정/건강 자연스럽게 물어보기. 답변은 상태추적.md에 기록 |

### 🧹 심부름꾼이 수집하는 것 (서브에이전트 프롬프트)

심부름꾼에게 아래 내용을 모두 읽고 **항목당 1~3줄 요약**으로 반환하라고 지시:

```
수집 항목:
1. [과업] progress/박승현/ 최신 파일 → "다음에 할 것" + 미완료 항목
2. [성장질문] OWNER/성장큐.md → 물어볼 것 중 1~2개 골라서 질문 원문 포함
3. [상태] OWNER/상태추적.md → 마지막 기록 1~2줄
4. [레포] git status → 커밋 안 된 파일 목록 + 개수
5. [팀원] 김민호, 이승훈의 progress/, mini-projects/ 변경 + git log → 변동 요약
6. [스케줄] schedule.md → 오늘 할 일 (없으면 "schedule.md 없음")
7. [암묵지] OWNER/암묵지.md → 🔍 후보 상태인 항목 목록 (검증 질문용)

반환 형식:
---과업--- (1~3줄)
---성장질문--- (질문 원문 1~2개)
---상태--- (1~2줄)
---레포--- (1~3줄)
---팀원--- (1~3줄)
---스케줄--- (1줄)
---암묵지 후보--- (후보 항목 번호 + 패턴명, 최대 3개)
```

> **핵심**: 메인은 이 요약만 받아서 브리핑을 만든다. 원본 파일을 직접 읽지 않는다.

## 성장 질문 원칙

- 매 세션 1~2개만. 바쁘면 "나중에 해도 돼요~"라고 양보.
- 구체적으로 — "승현님에 대해 더 알고 싶어요" (X) → "파트너라는 표현을 자주 쓰시는데, 슈파에서 파트너의 범위가 어디까지인지 궁금해요" (O)
- **불약 4조**: 매월 15일, 30일에는 성장 질문 대신 AI 이슈 분석 + 사업 적용 제안

## 브리핑 UI 템플릿

**컬러 시스템 (이모지 3톤)**:
- 💚 **초록 톤** = 상태, 안전, 긍정
- 🔵 **파랑 톤** = 정보, 과업, 데이터
- 🟡 **노랑 톤** = 주의, 액션 필요, 질문

**출력 형식**:

```
> ## 🧚 팅커벨 브리핑 — {날짜} ({요일}) {시:분}
>
> | 💚 승현님 | 🔵 이어서 할 것 | 👥 팀 | 📅 오늘 |
> |:---:|:---:|:---:|:---:|
> | {상태 한마디} | {N}건 대기 | {변동 요약} | {오늘 핵심} |

--- 💚 승현님 상태 ---
지난 세션 감정/컨디션 요약 (심부름꾼 ---상태--- 기반)

--- 🔵 지난 과업 → 이어서 할 것 ---
테이블: 프로젝트 | 어디까지 했나 | 다음 스텝
🟡 액션 필요한 항목은 노랑 이모지로 강조

--- 🌱 성장 질문 ---
블록쿼트로 질문. "바쁘시면 나중에~" 양보 포함

--- 🔮 암묵지 검증 ---
🔍 후보 중 2~3개를 골라 검증 질문. "맞아요? / 수정할 부분 있어요?" 형태
승현님 답변에 따라 암묵지.md 등급 업데이트 (🔍→✅ 또는 삭제)
후보가 없으면 이 섹션 생략

--- 💚 레포 상태 ---
한 줄 요약. 이상 없으면 ✅, 이상 있으면 🟡⚠️

--- 🔵 팀원 ---
한 줄 요약. 변동 없으면 간결하게, 있으면 상세히

--- 🔵 오늘 스케줄 ---
schedule.md 기준 오늘 할 일
```

**규칙**:
- 대시보드 요약(맨 위 테이블)은 항상 포함 — 한눈에 전체 파악용
- 이상 없는 섹션은 1~2줄로 끝냄. 장황하게 늘리지 않음
- 🟡 노랑은 승현님이 봐야 할 것에만 사용. 남발하면 주의력이 분산됨
