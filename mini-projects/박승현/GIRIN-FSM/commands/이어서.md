# "이어서" — 세션 이어받기 주문

> 컨텍스트 소진/세션 전환 후 이전 작업을 이어받을 때 실행.

---

## 실행 순서

| 순서 | 행동 | 누가 | 설명 |
|------|------|------|------|
| 1 | **팅커벨 로드** | 메인 | 팅커벨 3파일 (항상 적용 규칙) |
| 2 | **맥락 복구** | 서브에이전트 | 아래 수집 항목을 읽고 요약 반환 |
| 3 | **원본 검증 보고** | 메인 | 원본 상태 + 이어서 할 작업 요약 출력 |
| 4 | **승현님 확인** | 메인 | "이어서 이 작업 할까요?" 확인 후 재개 |

### 서브에이전트 프롬프트 (맥락 복구)

```
수집 항목:
1. sessions/ 또는 mini-projects/박승현/ 내 최신 세션 파일 → 마지막 작업 내용 + 프롬프트 원본 존재 여부
2. progress/박승현/ 최신 파일 → "다음에 할 것" 섹션
3. git log -5 --oneline → 최근 커밋으로 작업 흐름 파악

반환 형식:
---원본 상태--- (✅ 훼손 없음 / 🟡 확인 필요: {상세})
---마지막 작업--- (1~3줄)
---이어서 할 것--- (1~3줄)
---최근 커밋--- (5줄)
```

---

## 출력 형식

```
> 🧚 이어서 — 세션 이어받기

--- 원본 상태 ---
✅ / 🟡 상태 한 줄

--- 마지막 작업 ---
어디까지 했는지 1~3줄

--- 🔵 이어서 할 것 ---
다음 스텝 1~3줄

이어서 진행할까요?
```

---

## 규칙

- 원본 훼손 발견 시 → 🟡 경고 + 복구 시도 전 승현님 확인
- 이전 세션 파일이 없으면 → progress + git log만으로 복구
- 복구 완료 전 새 작업 시작하지 않음
- "이어서"가 "안녕"을 대체하지 않음. "안녕"은 풀 브리핑, "이어서"는 작업 복구에 집중
