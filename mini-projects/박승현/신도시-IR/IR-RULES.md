# 신도시 IR — 승현님 교정사항 (제약조건)

> IR 작업할 때만 로드. 새 세션에서 이것들을 틀리면 안 됨.
> 승현님 원본 그대로 보존.

---

## 표현 규칙

- **단위**: M(백만원) 사용 금지 → 억원/만원 한국식
- **CSP**: 언급 금지 → "전략적 파트너"로 대체
- **SaaS**: 비교 부분에서만 사용. 슈파 = "AI 기반 커머스 플랫폼"
- **행궁동행**: 언급 불필요. 공존 기금은 넣되 사단법인은 빼기
- **데이터 가격**: 시나리오가 아닌 보수적 단일 숫자로 표기
- **피드백 순서**: 크리틱 → 칭찬 → 크리틱 → 제안 (동조 먼저 금지)
- **이미 아는 정보는 질문하지 말 것** → 초안 먼저, 빠진 것만 물어보기

## 핵심 구조 (승현님 정의)

> "AI를 활용해서(거의 무료로) 경영 효율화 → 유저가 많아지면 물류를 공동구매처럼 싸게 → 경영 지원도 해주고 실제 비용도 낮춰주는것이 슈파 서비스의 핵심 구조"

- Phase 1: 행궁동·수원. 경영 지원 + 비용 절감
- Phase 2: 국내 확장. 데이터 판매. **여기서 실제 돈 버는 구조**
- Phase 3: 피지컬 AI
- 초기 유저: 글로컬상권창출팀 네트워크 활용
- 물류: 직접 안 함. 도메인 회사 연결 = **Asset Light**
- KPI: 행궁동 100개소 → 수원 1,000개소
- 구매몰 이용률: 35% → 최저가 시 급증
- 팀: 3명 비개발자 + 개발팀 가정. AI 네이티브 조직으로 어필
- 공존 기금: Phase 2부터 적립. 승현님 사명과 맞닿는 부분

## 숫자 불일치 주의

- Google Docs 기존 제안서 vs 대화 내용 → **대화 기준으로 통일**
- 팀: 기존 5인 → 3인+슈파AI
- 행궁동: 기존 300개소 → 100개소
