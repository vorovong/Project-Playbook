> CO-STAR = Context, Objective, Style, Tone, Audience, Response
> 기획서 초안 요청 시, 아래 기본값이 자동 적용된다.
> 개별 기획서에서 오버라이드 가능.

## 기본값

### C — Context (맥락)

기본 참조 파일:
- `STRATEGY/내부전략.md` — 4개 사업 에센스
- `기획/공존공간.md` — 회사 실적, 성장 경로
- `기획서-관리.md` — 기존 기획서 목록, 청중별 프레이밍

**기획서 작성 시 반드시 포함할 맥락:**
- 이 기획서가 어떤 사업의 일부인지
- 왜 지금 이것을 쓰는지 (시기적 맥락)
- 이전에 같은 청중에게 뭘 제출했는지

### O — Objective (목표)

기획서마다 다르지만, 반드시 한 줄로 정의:
```
"이 기획서의 목표는 [청중]이 [행동]하게 만드는 것이다."
예: "수원시 담당자가 3년차 사업비를 승인하게 만드는 것이다."
```

### S — Style (스타일)

→ `DNA/style-guide.md` 참조
- 에센스 먼저, 숫자로 증명
- 테이블과 화살표 흐름도 활용
- 기획자의 정직함 (모르는 건 명시)

### T — Tone (톤)

| 청중 | 톤 |
|---|---|
| 정부 평가위원 | 음슴체, 데이터 중심, 정제된 공문서 |
| 지자체장 | 음슴체 + 한 줄 핵심, 비전 강조 |
| 민간 파트너 | 약간 부드러운 음슴체, 상호 이익 강조 |
| 투자자 | 간결, 시장 규모 + 수익 모델 중심 |
| 내부 | 편한 구어체 |

### A — Audience (청중)

→ `DNA/audience/` 폴더의 해당 파일 참조
- 기획서 작성 전 반드시 "누가 읽는가"를 먼저 확인
- 같은 사업이라도 청중에 따라 강조점이 다름
  - 정부: "정책 목표 달성", "정량적 성과"
  - 시장: "수원의 위상", "전국 최초"
  - 민간: "수익", "시장 기회"

### R — Response (출력 형식)

기본: 마크다운 문서
- 제목 → 개요 테이블 → 본문 (에센스 순서) → 예산 → 일정 → 기대효과
- 최종 납품은 한글(HWP) 또는 구글독스이지만, 초안은 마크다운으로 작성

## 사용법

기획서 요청 시 이렇게 말하면 됨:
```
"글로컬상권 3년차 사업계획서 써줘.
청중: 수원시 담당 공무원.
핵심: 2년 성과 + 3년차 확장."
```

→ AI가 자동으로:
1. co-star-defaults.md 로드
2. audience/정부-평가위원.md 로드
3. templates/사업계획서.md 로드
4. style-guide.md 로드
5. 초안 생성
