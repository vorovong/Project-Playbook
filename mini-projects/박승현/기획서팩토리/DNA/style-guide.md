> 이 파일은 AI가 기획서 초안을 쓸 때 참조하는 "승현님의 글쓰기 DNA"다.
> 실물 기획서가 들어올 때마다 업데이트한다.

## 1. 톤 & 어미

| 상황 | 톤 | 어미 |
|---|---|---|
| 기획서·계획서·제안서 | 명사형 (음슴체) | ~함, ~임, ~완료, ~예정 |
| 성과보고서 | 데이터 중심, 간결 | ~함, ~달성 |
| 시장/구청장 보고 | 음슴체 + 핵심 한 줄 | ~함, ~필요 |
| 민간 제안 | 약간 부드러운 음슴체 | ~함, ~드림 |

## 2. 구조 패턴 — "에센스 먼저"

모든 기획서는 이 순서를 따른다:

```
1. 왜 하는가 (에센스/배경)     ← 가장 먼저. 읽는 사람이 "왜?"에 답을 얻어야 함
2. 뭘 증명했는가 (숫자/실적)   ← Before/After 비교
3. 어떻게 할 것인가 (구조)     ← 화살표 흐름도, 레이어 구조
4. 모르는 건 솔직히 (미결정)   ← "아직 정리 안 된 것" 명시
```

### 2-1. "역사 서사" 패턴 — 배경을 스토리로 (실물 #001에서 발견)

행정용 기획서의 배경 섹션에서 반복되는 패턴:
```
과거(역사적 맥락) → 쇠퇴(문제 발생) → 노력(기존 시도) → 이중성(아직 해결 안 된 것) → 제안(우리가 할 것)
```

예시 (도시, 시도):
```
정조의 신도시(1796) → 인구 37,000→9,000 쇠퇴 → 도시재생 100억 투입 → "관광도시인데 살기 힘든 동네" 이중성 → 직주락 생태계 제안
```

**핵심**: 문제를 단순화하지 않고 **"이중성"을 보여주는 것**이 승현님 특유의 프레이밍.

### 2-2. 프레임 태깅 — 사업을 하나의 키워드 체계로 분류

실물에서 발견된 패턴: 모든 세부사업에 직주락(Work/Live/Play) 태그를 붙여서 전체 구조가 하나의 프레임 안에 있음을 증명.
```
팔딱산 → (락)
행그라운드 → (직)
신도시호텔 → (주, 락)
```

## 3. 숫자 표현법

**Before/After 비교를 선호:**
```
❌ "외국인 소비가 증가하였음"
✅ "외국인 소비액 14억 → 29억 (+107%)"
```

**의미를 한 줄로 붙인다:**
```
❌ "소상공인 협업 건수가 20건에서 142건으로 710% 증가하여 유의미한 성과를 달성하였음"
✅ "소상공인 협업 20회 → 142회 (+710%). 상인이 스스로 참여하는 구조가 작동"
```

## 4. 시각화 습관

**테이블을 많이 쓴다** — 서술보다 표를 선호

**화살표 흐름도로 성장 경로 표현:**
```
행궁동 (증명 완료)
  → 남문시장 (40억, 2025.12 선정)
    → 수원 전역
      → 타 지역
```

**레이어 구조로 복잡한 것을 단순화:**
```
Layer 1: 공통 플랫폼 (접점 확보)
  ↑
Layer 2: 맞춤 자동화 (차별점)
```

**마스터플랜 지도** (실물 #001에서 발견):
- 물리적 범위를 지도 위에 표시
- PLAYER(이해관계자) 목록을 지도 옆에 배치
- 범례로 사업 부지, 상권, 걷고 싶은 길 등 구분

**통계 데이터 테이블** (실물 #001에서 발견):
- 시계열 비교: 연도별 인구변화, 사업체수 변화
- 출처를 반드시 명시: "(자료제공 : 통계지리정보서비스)"
- 여러 종류의 데이터로 **하나의 결론**을 증명 (인구 감소 + 사업체 증가 = 이중성)

**다이어그램** (실물 #001에서 발견):
- 문화권 개념도: 중심(팔달산) 둘레에 4축 배치 (주거/로컬크리에이터/스몰브랜드/역사문화관광)
- 서비스 구조도: 3개 카테고리 × 하위 서비스 격자 배치

## 5. 특유의 표현 — "기획자의 정직함"

| 패턴 | 예시 |
|---|---|
| **"~가 아니다"** 경계 표시 | "정책 제안 프레임이지, 공존공간의 사업 자체는 아님" |
| **"아직 정리 안 된 것"** 명시 | 체크리스트로 미결정 사항 솔직히 나열 |
| **출처를 밝힘** | "승현님 직접 정의", "회의에서 드러난 실체" |
| **리스크 + B플랜 쌍** | 리스크만 나열하지 않고 대안까지 |
| **대외용 vs 내부용 구분** | 같은 사업도 청중에 따라 프레이밍이 다름 |

## 6. 금지어 / 피해야 할 것

| 피하기 | 이유 |
|---|---|
| 과도한 수식어 ("획기적인", "혁신적인") | 승현님 스타일은 숫자로 증명 |
| 장황한 배경 설명 | 에센스 한 줄 + 숫자가 더 강력 |
| 추상적 기대효과 ("지역 발전에 기여") | 구체적 지표로 |
| 모호한 일정 ("추후 논의") | 날짜/조건을 명시 |

## 7. 공문서 세부 규칙

→ 상세: `GONGZONE-FSM/writing-style.md` 참조
- 날짜: `2025. 2. 23.` (마침표+띄어쓰기)
- 시간: 24시각제 `09:00`
- 금액: `금500,000원(금오십만원)`
- 외래어: 한글 먼저, 영문 괄호 `업무 협약(MOU)`
- 항목 기호: `1. → 가. → 1) → 가)`

## 8. 항목 기호 체계 (실물 #001에서 확인)

행정 기획서에서 사용하는 3단계 계층:
```
◯ 대항목 (원)
  Ÿ 중항목 (탭 + 부호)
    - 소항목 (탭탭 + 대시)
```

핵심 문장은 **박스 처리**하여 시각적으로 강조:
```
행궁동 일대 골목상권을 활성화하기 위해서 예술가, 전문가,
첨단 기술자 등 창조 계급이 머물기 매력적인 Creative City로
조성되어야 함
```
