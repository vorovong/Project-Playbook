## 한 줄 목표

"기획서 써줘" 한마디로, 승현님 스타일에 맞는 높은 완성도의 초안이 나오는 시스템.

## 왜 만드는가

- 승현님은 문화기획자. 기획서가 밥줄
- AI한테 시키면 "일반적인" 기획서가 나와서 처음부터 다시 쓰게 됨
- 청중별 프레이밍을 매번 수작업으로 바꿔야 함
- → 승현님의 "기획서 DNA"를 AI가 참조하게 만들면, 초안 품질이 확 올라감

## 작동 원리

```
[승현님의 기획서 DNA]          ← DNA/ 폴더
     ↓ AI가 참조
[기획서 초안 자동 생성]        ← templates/ + CO-STAR
     ↓ 승현님 교정
[피드백 기록]                  ← feedback/ 폴더
     ↓ DNA 업데이트
[다음 초안 품질 ↑]
     ↓ 루프 반복
[스킬화]                       ← SKILL.md 분리
```

## 폴더 구조

```
기획서팩토리/
├── DNA/
│   ├── style-guide.md          ← 승현님 글쓰기 스타일
│   ├── co-star-defaults.md     ← 기본 CO-STAR 설정
│   └── audience/
│       ├── 정부-평가위원.md     ← 정부사업 평가위원 프레이밍
│       ├── 지자체장.md          ← 시장/구청장 보고용
│       └── 민간-파트너.md       ← 기업 협력 제안용
│
├── templates/
│   ├── 사업계획서.md            ← 가장 자주 쓰는 유형
│   ├── 성과보고서.md
│   ├── 정부사업-제안서.md
│   └── 협력제안서.md
│
├── examples/
│   └── (승현님이 올리는 실물 기획서 + 패턴 분석)
│
├── feedback/
│   └── (기획서별 교정 내역 + 학습 기록)
│
└── README.md (이 파일)
```

## 스킬화 조건

- [ ] 템플릿 3종 이상 완성
- [ ] 실물 기획서 3개 이상으로 패턴 추출
- [ ] 피드백 루프 3회 이상 작동 확인
- [ ] 청중별 프레이밍 2종 이상 검증
→ 전부 충족하면 SKILL.md로 분리

## 참고한 패스트무버

- **CO-STAR 프레임워크** (싱가포르 정부기술청) — 프롬프트 구조의 기본기
- **Shipley Method** — "글 쓰기 전에 전략을 먼저 세운다"
- **클라이원트** — 한국 공공입찰 AI 특화
- **문서 DNA 구조화** — CLAUDE.md 원리를 기획서에 적용
