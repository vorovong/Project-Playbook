# 컨설팅 보고서 어시스턴트 — 자료 색인

이 파일을 먼저 읽고, 필요한 요약본만 찾아가세요.

---

## 사용법

### 보고서 자동 생성 (핵심 기능)

컨설팅 보고서를 자동으로 생성하는 흐름:

```
1. 입력 준비
   - 전사본 (컨설팅 녹음 텍스트)
   - 업체 기본정보 (업체명, 접수번호, 실시일, 수행 컨설턴트)
   - 데이터 (POS 매출, 재무 등 수기 숫자)

2. 김비서가 처리
   - 전사본에서 핵심 내용 추출
   - 양식 6개 항목에 맞게 분류/배치
   - 과거 보고서 사례 참고하여 문구 품질 향상

3. 출력
   - 서울신용보증재단 경영개선 컨설팅 보고서 (HWPX)
```

### 양식 항목 (6개)

| # | 항목 | 전사본에서 추출하는 것 |
|---|---|---|
| 1 | 사전조사/현황분석 | 업종 특성, 매출 분석, POS 데이터, 경영환경분석(내부+외부), 판매채널, 인구통계 |
| 2 | 컨설팅목적/추진과제 | 온라인 활성화, 매출 증대, 신메뉴, 고객 유입, 브랜딩 등 |
| 3 | 수행내역 | 벤치마킹, 키워드 분석, SNS 전략, 네이버 시스템, 교육 등 |
| 4 | 컨설팅결과/(기대)효과 | 스마트플레이스 최적화, 매출 증대, 콘텐츠 전략 등 |
| 5 | 활용자료/제공자료 | 체험이벤트 계획, 재무제표, 소상공인 자료 등 |
| 6 | 현장 사진 | 1~3일차, 타임스탬프 앱, 내부(필수)/외부 |

### 읽기 (준비)

1. 보고서 작업 요청이 오면 이 index.md를 먼저 읽는다
2. `rules/작성규칙.md`를 **항상** 읽고 적용한다
3. 주제/업종에 맞는 과거 보고서 요약본을 참고한다 — 경로: `바탕 화면/DB/컨설팅/sources/`
4. 요약본으로 부족하면 원본을 읽는다

### 쓰기 (자동 생성)

1. **전사본 분석** — 전사본을 읽고 6개 항목에 해당하는 내용을 분류한다
2. **데이터 결합** — 수기로 받은 매출/재무 데이터를 사전조사/현황분석에 넣는다
3. **과거 사례 참고** — 같은 업종의 과거 보고서가 있으면 문구/구조를 참고한다
4. **초안 작성** — 6개 항목을 채운 초안을 승훈님에게 보여준다
5. **승훈님 확인** — 수정 사항 반영
6. **HWPX 생성** — 확인된 내용으로 한글 파일 생성

### 검증 (출력 후 반드시 실행)

| # | 검증 항목 | 확인 방법 |
|---|---|---|
| 1 | **전사본 내용 누락 없음** | 전사본의 주요 논의 사항이 빠짐없이 반영되었는지 |
| 2 | **데이터 정확성** | 수기로 받은 숫자가 정확히 들어갔는지. 추측한 숫자 없는지 |
| 3 | **항목 분류 정확성** | 사전조사에 들어갈 내용이 수행내역에 섞여 있지 않은지 |
| 4 | **실용적 표현** | "혁신적", "차별화된" 같은 AI스러운 표현 없는지 |
| 5 | **분량 적절성** | 각 항목이 너무 짧거나 너무 길지 않은지. 핵심만 담겼는지 |

## 요약본 작성 규칙

보고서가 완성될 때마다 아래 4단계를 **모두** 완료한다:

1. **원본 정리** — 완성된 HWPX 파일을 `바탕 화면/DB/컨설팅/원본/{업체명}/`에 저장한다
2. **요약본 생성** — 아래 양식에 맞춰 `바탕 화면/DB/컨설팅/sources/`에 요약본(.md)을 만든다
3. **index.md 업데이트** — 자료 목록 + 주제별 빠른 찾기 + 업데이트 기록을 갱신한다
4. **저장** — git commit + push

### 보고서 요약본 양식

```
# 컨설팅 보고서: {업체명}

- 원본: {원본 파일 경로}
- 컨설팅 실시일: YYYY-MM-DD
- 업종: {업종}
- 수행 컨설턴트: {이름}

---

## 업체 개요
(업종, 위치, 규모 등)

## 핵심 현황
(사전조사에서 발견한 주요 이슈)

## 추진 과제
(컨설팅 목적과 추진 방향)

## 수행 내역 요약
(실제 수행한 것)

## 결과/효과
(컨설팅 결과와 기대 효과)

## 재활용 포인트
- 같은 업종 보고서에서 참고할 만한 문구나 접근법
```
