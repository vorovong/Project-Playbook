# 정부 지원사업 맞춤 알림 서비스 — 기술 구현 리서치

한국 소상공인 대상으로 정부 지원사업 공고를 자동 수집하고, 사장님 조건에 맞는 공고를 매칭하여 알림을 보내는 서비스의 기술적 구현 방법을 조사한 결과.

---

## 1. 공공 데이터 API 현황

### 1-1. 기업마당 (bizinfo.go.kr) — 지원사업정보 API

| 항목 | 내용 |
|------|------|
| **제공 기관** | 중소벤처기업부 (기업마당) |
| **데이터 내용** | 중앙부처, 지자체, 유관기관의 최신 지원사업 공고 + 행사정보 |
| **데이터 형식** | JSON, XML |
| **포함 필드** | 분야, 사업명, 신청시작일자, 신청종료일자, 소관기관, 수행기관, 접수기관, 등록일자, 상세URL 등 |
| **API 문서** | https://www.bizinfo.go.kr/web/lay1/program/S1T175C174/apiDetail.do?id=bizinfoApi |
| **활용 신청** | 공공데이터포털(data.go.kr)에서 활용 신청 후 인증키 발급 |

**핵심 판단:** 기업마당 API가 가장 실용적인 1차 데이터 소스. 중앙부처 + 지자체 공고를 통합 수집하고 있어 커버리지가 넓다.

### 1-2. 중소벤처24 (smes.go.kr) — 공고정보 API

| 항목 | 내용 |
|------|------|
| **제공 기관** | 중소기업기술정보진흥원 (TIPA) |
| **데이터 내용** | 중소벤처기업부 유관기관에서 수집한 공고정보 |
| **포함 필드** | 공고명, 기간, 지원기관, 신청상태, 첨부파일 등 |
| **공공데이터포털** | https://www.data.go.kr/data/15113191/openapi.do |
| **인증키 발급** | 중소벤처24 운영팀(044-300-0990)에 인증키 신청서 제출 후 이메일 수령 |
| **Open API 페이지** | https://www.smes.go.kr/main/dbCnrs |

**주의:** 중소벤처24 API는 별도 신청 절차가 필요하고, 공공데이터포털 경유가 아닌 직접 발급 방식이라 초기 셋업에 시간이 걸릴 수 있다.

### 1-3. 공공데이터포털 (data.go.kr) — 공통 사항

| 항목 | 내용 |
|------|------|
| **개발 계정** | 일일 1,000건 호출 제한 |
| **운영 계정** | 일일 최대 100,000건 호출 가능 (서비스 운영 중일 때 신청) |
| **인증 방식** | 서비스키 (API Key) 기반 |
| **신청 절차** | 회원가입 → 활용 신청 → 즉시 또는 승인 후 키 발급 |
| **데이터 형식** | REST API (JSON/XML) |

### 1-4. 기타 데이터 소스

| 소스 | URL | 비고 |
|------|-----|------|
| 소상공인24 | sbiz24.kr | 소상공인진흥공단 운영, 소상공인 특화 |

## 2. 크롤링 가능 여부

### 2-1. API 우선 전략

기업마당 + 중소벤처24 API로 대부분의 중앙부처/지자체 공고를 커버할 수 있으므로, **API를 1차 데이터 수집 방법으로 사용하고 크롤링은 보조 수단**으로 활용하는 것이 권장된다.

### 2-2. 크롤링이 필요한 경우

- 지자체 자체 홈페이지의 지역 특화 공고 (API에 미포함)
- 공고 상세 본문 (API는 제목/기간/링크만 제공하는 경우가 많음)
- 첨부파일(HWP/PDF) 내용 추출

### 2-3. 법적 이슈 정리

| 구분 | 내용 |
|------|------|
| **robots.txt** | 법적 강제력은 없으나 준수가 권장됨. 무시 시 분쟁 소지 |
| **합법 기준** | 사이트 운영자 의사에 반하지 않는 크롤링은 합법 (대법원 2022.5.12 선고 2021도1533) |
| **정부 공고** | 공공정보 성격이 강해 크롤링 허용 가능성 높음. 단, 서버 부하를 줄이는 예의(rate limiting) 필수 |
| **위반 위험** | 개인정보 수집, 서버 과부하, 저작물 무단 복제 시 정보통신망법/저작권법 위반 가능 |
| **안전한 방법** | (1) robots.txt 준수, (2) 적절한 크롤링 간격(crawl-delay), (3) 공개 정보만 수집, (4) 출처 명시 |

### 2-4. 기술 스택 (크롤링 시)

```
- Python + Scrapy: 정적 페이지 크롤링 (빠르고 확장성 좋음)
- Python + Selenium/Playwright: 동적 렌더링 페이지 (JS 기반 사이트)
- BeautifulSoup: 간단한 파싱
- Schedule/APScheduler: 정기 수집 스케줄링
```

## 3. 매칭 알고리즘

### 3-1. 기본 구조: 규칙 기반 필터링 (Rule-based Filtering)

소상공인의 프로필과 공고의 자격 요건을 매칭하는 가장 직관적인 방법.

```
사장님 프로필              공고 자격 요건
─────────────            ──────────────
업종 (음식/소매/제조 등)    대상 업종
지역 (시/군/구)            대상 지역
매출 규모                  매출 기준 (이하/이상)
업력 (년)                  업력 기준
종업원 수                  규모 기준
```

**매칭 로직:**
1. 하드 필터: 업종, 지역, 매출 등 자격 요건 불일치 시 제외
2. 소프트 스코어: 가점 항목(여성기업, 사회적기업, 장애인기업 등) 매칭 시 점수 부여
3. 랭킹: 점수순 정렬 후 상위 N건 추천

### 3-2. 고도화: 텍스트 기반 매칭

공고 본문에서 자격 요건을 자동 추출하기 어려운 경우 NLP 활용.

| 단계 | 방법 |
|------|------|
| 키워드 추출 | TF-IDF, 정규표현식으로 업종/지역/금액 추출 |
| 임베딩 매칭 | 사장님 프로필을 벡터화 → 공고 텍스트 벡터와 유사도 계산 |
| LLM 구조화 | GPT/Claude로 공고 본문 → 구조화된 자격 요건 JSON 변환 |

### 3-3. 참고: 선행 연구

- **딥러닝 기반 중소기업 사업공고 분석 및 융합 파트너 매칭 시스템** (KISTI 보고서, TRKO202000005640)
  - 기계 학습 기반 프로파일 속성 학습 → 적합 사업공고 매칭
  - 산업 융복합 지수 + 사용자 행태 분석 활용

### 3-4. 현실적 MVP 제안

```
Phase 1: 규칙 기반 필터링 (업종+지역+매출)
Phase 2: LLM으로 공고 본문 → 구조화 (자격요건 JSON 추출)
Phase 3: 임베딩 기반 유사도 + 사용자 클릭/저장 데이터 활용한 개인화
```

## 4. LLM 활용 방안

### 4-1. 공고문 요약

| 용도 | 방법 | 예시 |
|------|------|------|
| **공고 요약** | 긴 공고문(HWP/PDF)을 3-5줄로 요약 | "이 사업은 연매출 3억 이하 음식업 소상공인에게 최대 2000만원 지원..." |
| **자격 요건 구조화** | 공고 본문 → JSON 추출 | `{업종: ["음식", "소매"], 지역: "서울", 매출상한: 3억, 업력: "3년 이상"}` |
| **핵심 일정 추출** | 접수 기간, 발표일, 사업 기간 등 | 자동 캘린더 등록 |

### 4-2. 신청 자격 판단

```
프롬프트 구조:
[공고 자격 요건 텍스트] + [사장님 프로필] → "이 공고에 신청 가능한가요?"

출력: 적합/부적합 + 이유 + 부족한 조건
```

- RAG(Retrieval-Augmented Generation) 활용: 공고 원문을 벡터 DB에 저장 → 질문 시 관련 문서 검색 후 답변 생성
- 환각(hallucination) 방지를 위해 반드시 원문 근거 표시

### 4-3. Q&A 챗봇

| 질문 유형 | 처리 방식 |
|-----------|----------|
| "내가 받을 수 있는 지원사업이 뭐야?" | 프로필 기반 매칭 → 목록 제시 |
| "이 사업 신청하려면 뭐가 필요해?" | RAG로 해당 공고 원문에서 답변 |
| "신청서 어떻게 써?" | 공고별 신청 가이드 템플릿 제공 |

### 4-4. 기술 스택

| 구성 요소 | 추천 도구 |
|-----------|----------|
| LLM | OpenAI GPT-4o / Claude 3.5 Sonnet (API) |
| 임베딩 | OpenAI text-embedding-3-small / BGE-M3 (한국어 성능 우수) |
| 벡터 DB | Pinecone / Chroma / Qdrant |
| RAG 프레임워크 | LangChain / LlamaIndex |
| 문서 파싱 | PyMuPDF (PDF), python-hwp (HWP), Docling |

### 4-5. 비용 추정

| 모델 | 입력 | 출력 | 공고 1건 요약 비용 |
|------|------|------|-------------------|
| GPT-4o | $2.5/1M tokens | $10/1M tokens | 약 30-50원 |
| GPT-4o-mini | $0.15/1M tokens | $0.6/1M tokens | 약 2-5원 |
| Claude 3.5 Sonnet | $3/1M tokens | $15/1M tokens | 약 40-70원 |
| Claude 3.5 Haiku | $0.8/1M tokens | $4/1M tokens | 약 10-20원 |

> 일일 새 공고 20-50건 기준, 월 LLM 비용: GPT-4o-mini 사용 시 약 3,000~7,500원

## 5. 알림 방법 비교

### 5-1. 채널별 비용 비교

| 채널 | 건당 비용 (VAT별도) | 장점 | 단점 |
|------|---------------------|------|------|
| **카카오 알림톡** | 8~13원 (발송량에 따라) | 열람율 높음(90%+), 카톡 내 확인, 링크 삽입 가능 | 카카오 채널 친구 필요, 템플릿 사전 검수 |
| **SMS (단문)** | 13원 | 전화번호만 있으면 발송, 별도 가입 불필요 | 80바이트 제한, 링크 클릭률 낮음 |
| **LMS (장문)** | 29원 | 긴 내용 가능 | 비용 높음, 스팸 인식 |
| **MMS (사진)** | 60원 | 이미지 첨부 가능 | 비용 최고 |
| **앱 푸시** | 무료 (FCM/APNs) | 비용 0원, 즉시 도달 | 앱 설치 필요, 알림 끄면 무용 |
| **이메일** | 거의 무료 | 긴 내용, 첨부파일 가능 | 열람율 낮음(10-20%) |

### 5-2. 알림톡 상세 (SOLAPI 기준)

| 월간 발송량 | 알림톡 단가 | 할인율 |
|-------------|------------|--------|
| 1~9,999건 | 13원 | 0% |
| 10,000~99,999건 | 10원 | 23% |
| 100,000~499,999건 | 8원 | 38% |
| 500,000건~ | 7원 | 46% |

### 5-3. 추천 전략

```
MVP 단계: 카카오 알림톡 (열람율 높고, 소상공인 친화적)
  - 월 1,000명 × 주 2회 = 월 8,000건 → 약 104,000원/월
  - 알림톡 API 연동 업체: SOLAPI, 알리고, NHN Cloud, 카카오 i 커넥트

성장 단계: 알림톡 + 앱 푸시 병행
  - 앱 설치 유도 후 앱 푸시(무료)로 전환
  - 중요 공고만 알림톡, 일반 공고는 앱 내 확인
```

### 5-4. 알림톡 연동 참고

- 카카오 비즈니스 채널 개설 필수
- 메시지 템플릿 사전 등록 + 검수 (1-2 영업일)
- API 연동은 SOLAPI, 알리고 등 중계 플랫폼 이용이 편리
- 2026년부터 친구톡 텍스트 전용은 폐지 → 브랜드메시지(텍스트+이미지)로 전환

| 소상공인 눈높이가 아님 | 사장님 언어로 쉽게 설명 |

## 7. 종합 아키텍처 제안

```
┌─────────────────────────────────────────────────┐
│                  데이터 수집 레이어                │
│                                                   │
│  기업마당 API ──┐                                 │
│  중소벤처24 API ─┼── Scheduler (매일 1-2회) ──→ DB │
│  지자체 크롤링 ──┘                                │
└───────────────────────┬─────────────────────────┘
                        │
┌───────────────────────▼─────────────────────────┐
│                  가공 레이어                       │
│                                                   │
│  LLM 요약 ──→ 자격요건 구조화(JSON) ──→ 벡터화   │
│  (GPT-4o-mini)        (임베딩)                    │
└───────────────────────┬─────────────────────────┘
                        │
┌───────────────────────▼─────────────────────────┐
│                  매칭 레이어                       │
│                                                   │
│  사장님 프로필 × 공고 자격요건 ──→ 스코어링       │
│  (업종/지역/매출/업력)                             │
└───────────────────────┬─────────────────────────┘
                        │
┌───────────────────────▼─────────────────────────┐
│                  알림 레이어                       │
│                                                   │
│  카카오 알림톡 (SOLAPI) ──→ 사장님 카카오톡       │
│  앱 푸시 (FCM) ──→ 앱 알림                        │
└───────────────────────┬─────────────────────────┘
                        │
┌───────────────────────▼─────────────────────────┐
│                  사용자 인터페이스                  │
│                                                   │
│  웹/앱: 공고 목록 + 상세 + Q&A 챗봇              │
│  카카오 채널: 간편 조회 + 알림 설정               │
└─────────────────────────────────────────────────┘
```

### 기술 스택 요약

| 영역 | 기술 |
|------|------|
| Backend | Python (FastAPI) 또는 Node.js |
| DB | PostgreSQL + pgvector (벡터 검색) |
| 스케줄러 | APScheduler / Celery + Redis |
| LLM | OpenAI GPT-4o-mini (비용 효율) |
| 벡터 검색 | pgvector 또는 Chroma |
| 알림 | SOLAPI (알림톡 + SMS fallback) |
| 프론트 | Next.js (웹) / React Native (앱) |
| 인프라 | AWS / NCP(네이버클라우드) |
