## 0. 크리틱에 대한 직접 응답

승훈님의 질문은 정확하다. 기존 10개 기능 중 **7개는 단순 LLM wrapper**다. GPT API를 감싼 것뿐이다. ChatGPT가 무료로 하는 일을 웹앱에 넣은 것은 제품이 아니라 데모다.

해자(moat)가 되려면 세 가지 중 하나는 있어야 한다:

| 해자 유형 | 설명 | 예시 |
|---|---|---|
| **데이터 해자** | 쓸수록 데이터가 쌓이고, 그 데이터가 다음 사용자에게 가치를 만든다 | 캐시노트 (522조원 거래 데이터) |
| **네트워크 해자** | 사용자가 늘수록 각 사용자의 가치가 올라간다 | 테이블링 (소비자+사장님 양면) |
| **연동/전환비용 해자** | 한번 연결하면 빼기 어렵다 | 네이버 스마트플레이스 (검색 노출) |

기존 10개를 이 기준으로 평가한다.

## 2. 30개 기능 제안

기존 10개를 포함하되, "아무나 못 하는" 차별화 요소가 있는 20개를 추가한다. 각 기능에 대해 차별화 포인트와 데이터 해자를 명시한다.

### 범례

- **데이터 해자 등급:** 없음 / 약 / 중 / 강
- **난이도:** 낮 / 중 / 높
- 기존 10개는 [기존]으로 표시

### A. 핵심 Hook (매일 사용, 진입점)

| # | 기능명 | 한줄 설명 | 차별화 포인트 | 데이터 해자 | 난이도 |
|---|---|---|---|---|---|
| 1 | **리뷰 통합 관리 + 인사이트** [기존] | 네이버/배민/쿠팡이츠/카카오맵 리뷰를 한 곳에서 AI 답변 + 감성 분석 + 반복 불만 알림 | 딜리봇은 배달앱만. 슈파는 **전 채널 통합 + 불만 패턴 분석**. 답변 생성이 아니라 "이 가게의 약점 진단"이 핵심 | **강** — 가게별 리뷰 히스토리가 쌓일수록 진단 정확도 상승. 업종별 벤치마크 데이터 생성 | 중 |
| 2 | **정부 지원사업 맞춤 알림** [기존] | 사업자등록번호 입력 → 업종/지역/매출에 맞는 지원사업을 카카오 알림톡으로 발송 | 기업마당/중소벤처24 API 연동. 정부 포털 UX 불편을 해결. **민간 통합 서비스 거의 없음** (기술조사 보고서) | **강** — 사업자 프로필 + 신청 이력이 쌓일수록 매칭 정확도 상승. "이 업종은 이 지원사업 합격률이 높다" 데이터 생성 | 낮 |
| 3 | **오늘의 브리핑** (신규) | 매일 아침 카카오톡으로 "오늘의 할 일 요약" 발송: 리뷰 현황, 지원사업 마감, 계약 만료, 날씨 영향 예측 | **슈파의 모든 기능을 하나의 접점으로 수렴**시키는 메타 기능. 이것이 매일 앱을 여는 이유가 된다. 단독 기능이 아니라 다른 기능의 데이터를 엮는 것이므로 단순 복제 불가 | **강** — 사용자의 확인/무시 패턴이 쌓이면 "이 사장님에게 중요한 정보"를 학습. 개인화 자동 진행 | 중 |

| **약/없음** | 12, 14, 30 (3개) | 단독으로는 해자 약함. 다른 기능과 묶여야 가치 |

## 4. 강점 3개

### S1. 데이터 플라이휠이 설계되어 있다

30개 기능 중 17개(57%)가 "강" 해자 등급이다. 이 기능들의 공통점은 **사용자가 늘수록 데이터가 쌓이고, 그 데이터가 다음 사용자에게 가치를 만드는 구조**다. 이것은 해외 성공 사례의 공통 패턴과 일치한다(종합보고서: "데이터 플라이휠 설계 필수").

특히 사업자 프로필(7번) → 오늘의 브리핑(3번) → 업종별 벤치마크(9번)의 체인이 핵심이다. 프로필이 데이터의 중앙 노드가 되고, 브리핑이 매일 사용의 이유가 되고, 벤치마크가 "슈파 안 쓰면 이 정보를 못 본다"는 락인을 만든다.

### S2. 공존공간 2년 현장 경험이 차별화 자산이다

기능 21(거래명세서), 25(공동구매), 26(환대지수)은 공존공간이 이미 현장에서 검증한 것이다. 이것은 순수 기술 스타트업이 따라할 수 없는 자산이다. 행궁동 관광 소비액 +157%, 환대지수 64→89점이라는 실적은 "이론이 아니라 돌아간다"는 증거다.

### S3. "아무나 못 하는" 기능이 20개 추가되었다

기존 10개 중 4개가 "해자 없음" 판정을 받았다. 새로 추가된 20개 중 "해자 없음"은 0개다. 크로스 마케팅(16번), 업종별 벤치마크(9번), 공급업체 리뷰(18번) 등은 네트워크 효과가 내장되어 있어 사용자 수 자체가 방어벽이 된다.

## 5. 약점 3개

### W1. 30개 중 MVP에 넣을 수 있는 것은 3~5개뿐이다

해자가 강한 기능일수록 난이도가 높다. "강" 해자 17개 중 7개가 "높" 난이도다. 현실적으로 MVP에는 "낮" 난이도 + "강/중" 해자인 기능만 들어갈 수 있다. 그 교집합은 **2(정부지원), 7(사업자프로필), 16(크로스마케팅)** 정도다. 나머지는 Phase 2 이후.

### W2. 데이터 해자는 "데이터가 쌓인 후"에야 작동한다

벤치마크(9번), 공급업체 리뷰(18번), 수요 예측(22번)은 사용자 1,000명 이상이어야 의미 있다. 초기 100명 단계에서는 이 기능들이 "빈 방"이다. 초기 단계에서 데이터 해자가 아닌 **실행 가치(리뷰 답변으로 시간 절약, 지원사업으로 돈 절약)**로 사용자를 모아야 한다. 콜드 스타트 문제를 정면으로 해결해야 한다.

### W3. 수익 모델과의 연결이 불명확하다

소상공인의 지불 의향은 **"무료~1만원"**(니즈조사 보고서). 30개 기능을 만들어도 월 9,900원 프리미엄으로 커버할 수 있는 개발/운영비는 한정적이다. 정부 지원사업 알림의 월 운영비만 15~18만원(기술조사 보고서). AI 전화 응대(10번)는 통화 인프라 비용이 추가된다. 기능이 늘수록 비용은 선형으로 증가하지만, 매출은 그렇지 않다.

## 6. 개선 제안 3개

### R1. MVP 조합을 "해자 + 난이도" 교차점에서 선택하라

**추천 MVP 조합 (3+1):**

| 기능 | 역할 | 해자 | 난이도 |
|---|---|---|---|
| **리뷰 통합 관리(1)** | Hook — 매일 사용 | 중→강 | 중 |
| **정부 지원사업 알림(2)** | 차별화 — 블루오션 | 강 | 낮 |
| **사업자 프로필 자동 구축(7)** | 인프라 — 모든 기능의 기반 | 강 | 낮 |
| + **오늘의 브리핑(3)** | 리텐션 — 매일 카톡 | 강 | 중 |

이 조합의 논리: 사업자 프로필(7)이 데이터 중앙 노드. 리뷰(1)와 지원사업(2)이 실질 가치를 제공. 브리핑(3)이 매일 여는 이유를 만든다. 4개 모두 "해자 강" 또는 "해자 중→강"이며, 난이도가 낮~중이다.

### R2. "해자 없음" 기능은 "해자 있음" 기능의 부속품으로 재설계하라

SNS 콘텐츠(20), 구인 공고(12), 메뉴판(23), 소개글(30)은 단독으로는 해자가 없다. 그러나 이것들을 **오늘의 브리핑(3) → "이번 주 SNS에 올릴 콘텐츠" 추천**, **사업자 프로필(7) → "이 업종에 맞는 구인 공고 템플릿" 자동 제안**, **업종별 벤치마크(9) → "이 메뉴의 적정 가격은 X원" 제안** 같이 연결하면, 해자 있는 기능의 가치를 올리는 도구가 된다. 독립 기능이 아닌 "해자 기능의 출력물"로 재정의하는 것이다.

### R3. 콜드 스타트를 공존공간 네트워크로 해결하라

데이터 해자의 최대 약점은 "첫 1,000명을 어떻게 모으느냐"다. 제안: 공존공간이 2년간 관계를 맺어온 **행궁동 상권 가게 + 시범 파트너 30곳**이 초기 데이터를 만든다. 이 30곳의 실제 데이터로 "행궁동 음식점 평균 원가율 38%", "행궁동 카페 평균 리뷰 점수 4.2점" 같은 마이크로 벤치마크를 먼저 만들고, 이것을 다른 상권에 확산할 때 "행궁동 사례"로 보여주는 것이다. 이것이 승훈님이 가진 자산 중 가장 과소평가된 것이다(추정).

## 7. 최종 판정

승훈님의 크리틱 "난이도가 낮은 건 좋은데, 그럼 아무나 할 수 있지 않아?"에 대한 답:

**맞다. 기존 10개 중 4개는 아무나 할 수 있다. 그래서 그것만으로는 사업이 아니다.**

그러나 해자는 "기능 하나"에서 오는 것이 아니다. **기능 간 연결 + 데이터 축적 + 네트워크 효과**에서 온다.

- 리뷰 답변 AI는 아무나 만든다.
- 그러나 리뷰 데이터 + 매출 데이터 + 마케팅 데이터를 합쳐서 "매출 하락 원인을 진단하는 AI"는 아무나 못 만든다.
- 정부 지원사업 알림은 API 연동하면 된다.
- 그러나 1,000개 사업자 프로필의 지원사업 신청/합격 데이터로 "이 업종이면 이 지원사업 합격률이 높다"를 알려주는 것은 아무나 못 한다.

**핵심은 개별 기능의 난이도가 아니라, 기능 간 데이터 흐름의 설계다.** 이 30개 기능은 그 설계의 초안이다.
