# 슈파(SUPA) 추가 기능 후보 리서치

기존 8개 영역(리뷰 답변, SNS/마케팅, 메뉴관리, 세금/장부, 직원관리, 고객응대, 식자재, 정부지원사업) **밖에 있는** 기능 후보를 5개 방향으로 조사했다.

---

## 1. 소상공인이 하는 잡무 중 아직 안 다룬 것

### 1-1. 시설/인테리어 관리

| 항목 | 내용 |
|------|------|
| **설명** | 냉장고, 에어컨, 정수기, 가스레인지 등 장비 점검 주기 관리. 고장 시 수리업체 연결. 인테리어 리뉴얼 견적 비교. 소방/위생 점검 대비 체크리스트 |
| **체감 가치** | **중** — 평소에는 잊고 지내다가 고장 나면 매출 직격탄. 예방 관리 가치는 높지만 "매일 필요한" 기능은 아님 |
| **기술 난이도** | ★★☆☆☆ (장비 등록 + 주기 알림 + 수리업체 DB 연동) |
| **기존 서비스** | **숨고, 크몽** — 수리업체 매칭은 있지만 "내 가게 장비 이력 관리"는 없음. **당근마켓 동네업체** — 수리 연결은 되지만 체계적 관리 불가 |
| **MVP 판단** | **나중에.** 빈도가 낮아 첫 사용 유인이 약함. Phase 3(올인원 비서)에서 "시설 관리 달력" 형태로 추가 적합 |

### 1-2. 보험 관리

| 항목 | 내용 |
|------|------|
| **설명** | 화재보험(의무), 영업배상책임보험, 산재보험, 고용보험 등 소상공인 필수 보험 가입 현황 점검. 만기 알림. 보장 범위 비교. 사고 시 청구 절차 안내 |
| **체감 가치** | **중** — "가입은 했는데 뭘 가입했는지 모르겠다"가 대부분. 사고 발생 시 체감 가치 폭발하지만, 평소에는 관심 없음 |
| **기술 난이도** | ★★★☆☆ (보험 상품 DB + 사업자 프로필 매칭 + 만기 알림) |
| **기존 서비스** | **보험다모아, 카카오페이 보험** — 개인보험 중심. 소상공인 사업용 보험 비교 특화 서비스는 거의 없음 |
| **MVP 판단** | **나중에.** 보험은 규제 영역이라 진입 장벽 있음. 다만 "내 보험 현황 정리 + 만기 알림"은 간단히 구현 가능하므로 Phase 2에서 부가 기능으로 고려 |

### 1-3. 법률/노무 상담

| 항목 | 내용 |
|------|------|
| **설명** | 임대차 분쟁(권리금, 보증금 반환, 계약 갱신), 근로기준법 관련 질문(해고, 야간수당, 주휴수당), 고객 클레임 대응, 행정처분 대응 등 |
| **체감 가치** | **높음** — 문제가 생기면 변호사 상담료 최소 10~30만원. "이게 합법인지 불법인지"만 알아도 큰 도움. 특히 임대차, 노무 이슈는 소상공인의 만성 고민 |
| **기술 난이도** | ★★★☆☆ (법령 DB + LLM 기반 FAQ 챗봇. 단, "법률 조언"의 법적 경계 주의) |
| **기존 서비스** | **로톡** — 변호사 매칭 플랫폼. **대한법률구조공단** — 무료 상담이지만 접근성 낮음. **AI 법률 상담**: 로앤컴퍼니의 AI 변호사 서비스 등 등장 중이나 소상공인 특화는 없음 |
| **MVP 판단** | **Phase 2 고려.** "소상공인 빈출 법률 Q&A" AI 챗봇은 기술적으로 빠르게 구현 가능. 단, "법률 상담"이 아닌 "법률 정보 안내"로 범위를 명확히 해야 함. 정부지원사업 알림과 시너지(법률 지원사업 연결) |

### 1-4. 구인/구직 (채용 관리)

| 항목 | 내용 |
|------|------|
| **설명** | 알바 구인 공고 작성, 여러 플랫폼(알바몬, 알바천국, 당근)에 동시 게시, 이력서 검토, 면접 일정 관리, 근로계약서 자동 생성 |
| **체감 가치** | **높음** — "사람 구하기가 장사보다 어렵다"는 소상공인 최대 고충 중 하나. 특히 급한 대타 구인은 시간 싸움 |
| **기술 난이도** | ★★★☆☆ (구인 공고 AI 생성 + 멀티 플랫폼 게시는 API 제한 있음 + 근로계약서 템플릿) |
| **기존 서비스** | **알바몬, 알바천국** — 대형 플랫폼이지만 "AI 공고 작성"은 없음. **당근 알바** — 동네 기반. **긱(GIG)** — 단기 알바 매칭 |
| **MVP 판단** | **Phase 2 유력.** "구인 공고 AI 자동 작성 + 근로계약서 자동 생성"만으로도 가치 높음. 직원관리(기존 5번)와 자연스러운 확장 |

| **체감 가치** | **높음** (해당자에게) — 폐업률 높은 현실(연간 약 90만 개 폐업)에서 "어떻게 정리해야 할지 모르겠다"가 큰 고통. 하지만 타겟이 좁음 |
| **기술 난이도** | ★★☆☆☆ (절차 가이드 + 지원사업 매칭) |
| **기존 서비스** | **소상공인시장진흥공단 희망리턴패키지** — 폐업 컨설팅/교육 제공하지만 접근성 낮음 |
| **MVP 판단** | **나중에.** 타겟이 좁고 앱의 첫인상과 맞지 않음("폐업 도와주는 앱"으로 보이면 안 됨). Phase 3에서 은근히 넣되, 정부지원사업 알림에 "희망리턴패키지" 같은 항목을 자연스럽게 포함하는 방식이 적절 |

### 1-7. 계약/문서 관리

| 항목 | 내용 |
|------|------|
| **설명** | 임대차계약서, 근로계약서, 거래처 계약서, 인허가 서류 등을 한 곳에 보관. 만료일 알림. AI로 계약서 핵심 조항 요약. 불리한 조항 경고 |
| **체감 가치** | **중~높음** — "계약서를 어디 뒀는지 모르겠다"가 현실. 분쟁 시 계약서가 핵심 증거. 갱신 시기 놓치면 손해 |

| **기존 서비스** | **모두싸인** — 전자계약. **도큐사인** — 해외. 소상공인 특화 계약 관리 앱은 부재 |
| **MVP 판단** | **Phase 2 고려.** "계약서 사진 찍으면 AI가 핵심 요약 + 만료일 자동 등록"은 강력한 기능. 법률 상담(1-3)과 시너지 |

## 2. 소상공인 특화 AI 활용 사례

### 2-1. AI 전화 응대 (AI 콜봇)

| 항목 | 내용 |
|------|------|
| **설명** | 영업 중 받지 못하는 전화를 AI가 대신 받아 예약 접수, 영업시간 안내, 메뉴 안내 등 기본 응대. 통화 내용 텍스트 요약으로 사장님에게 전달 |
| **체감 가치** | **매우 높음** — "요리 중에 전화 받아야 하는" 고통 1위 해결. 전화 못 받으면 곧바로 매출 손실. 특히 1인 매장 |
| **기술 난이도** | ★★★★☆ (STT + TTS + LLM + 전화 인프라(클라우드 PBX). 통화 품질과 자연스러움이 핵심) |
| **기존 서비스** | **AI콜봇(KT)** — 대기업 고객센터 중심. **네이버 AI콜** — 예약 자동 전화. **띵스플로우 AI전화** — 소상공인용 AI 전화 응대 등장. **해외: Slang AI** — 레스토랑 AI 전화 응대 |
| **MVP 판단** | **Phase 2 핵심 후보.** 체감 가치 최상급이지만 전화 인프라 구축이 필요해 MVP 첫 버전은 어려움. 다만 "부재중 전화 AI 콜백" 같은 가벼운 형태로 시작 가능. 경쟁이 막 시작되는 타이밍이라 선점 가치 있음 |

---

### 2-2. AI CCTV/매장 분석

| 항목 | 내용 |
|------|------|
| **설명** | 기존 CCTV 영상을 AI로 분석해 고객 동선, 피크타임 패턴, 직원 서비스 품질, 도난 감지 등 인사이트 제공 |
| **체감 가치** | **중** — 데이터 기반 매장 운영의 이상적 모습이지만, 소상공인이 "이걸 왜 봐야 하는지" 체감하기 어려움. 프랜차이즈 본사에 더 적합 |
| **기술 난이도** | ★★★★★ (영상 처리 인프라, 프라이버시 규제, CCTV 연동) |
| **기존 서비스** | **딥브레인AI** — AI 매장 분석. **해외: Presto** — 드라이브스루 AI 분석. 모두 대형 체인 대상 |
| **MVP 판단** | **빼기.** 기술 난이도 극상 + 프라이버시 이슈 + 소상공인 체감 가치 낮음. 슈파 방향과 맞지 않음 |

### 2-3. AI 수요 예측

| 항목 | 내용 |
|------|------|
| **설명** | 날씨, 요일, 시즌, 주변 이벤트 등을 종합해 내일/이번 주 예상 매출과 손님 수를 예측. 식자재 발주량, 직원 배치에 활용 |
| **체감 가치** | **높음** — "내일 손님이 얼마나 올지"를 아는 것은 모든 운영 결정의 기초. 과잉 준비(폐기) vs 과소 준비(품절) 방지 |
| **기술 난이도** | ★★★★☆ (POS 매출 데이터 + 날씨 API + 지역 이벤트 데이터 + ML 모델) |
| **기존 서비스** | **해외: Tenzo, ClearCOGS** — 레스토랑 AI 수요 예측. **국내**: POS 업체에서 기초 통계는 제공하나 AI 예측은 거의 없음 |
| **MVP 판단** | **Phase 3.** POS 데이터 연동이 선행되어야 하므로 초기엔 어려움. 식자재 관리(기존 7번) 고도화 단계에서 자연스럽게 합류 |

### 2-4. AI 가격 최적화

| 항목 | 내용 |
|------|------|
| **설명** | 원가율, 경쟁 매장 가격, 수요 변동을 분석해 최적 메뉴 가격 제안. 시간대별/요일별 동적 가격(해피아워 등) 설계 |

| **기술 난이도** | ★★★★☆ (경쟁사 가격 크롤링 + 원가 데이터 + 가격탄력성 모델) |
| **기존 서비스** | **해외: eatOS** — AI 동적 가격. **국내**: 전무 |
| **MVP 판단** | **Phase 3.** 흥미롭지만 데이터 축적이 전제. "가격 인상 시 매출 영향 시뮬레이션" 정도는 Phase 2에서 간단히 제공 가능 |

## 3. 소상공인 생활 편의 (사장님 개인)

### 3-1. 사장님 건강 관리

