## 1. 슈파(SUPA) — AI 전환(AX) 서비스

- 박승현(기획) + 이승훈(현장 실행) + 김민호(AX 리드/방법론)
- **수원 로컬 현장 주도권** → 승훈님이 가장 주도적으로 할 수 있는 사업
- 현재: IR 기획서 v3 완료 → **현장 밀착형 실증 전략으로 피벗** (2026-03-26 확정)
- 슈파 = 식자재 공동구매 플랫폼에서 **AX(AI 전환) 서비스**로 진화

### 전략 피벗 (2026-03-26)

기존의 평면적 기능 제공(공고 알림, SNS 자동화, 메뉴판 등) → **현장 밀착형 실증**으로 전면 전환.
- 실제 매장에 상주하며 한 달간 AI 시스템을 직접 빌드업
- 상상이 아닌 현장 데이터 기반으로 개발
- 단순 SaaS가 아니라 **유통(식자재 공동구매)과 결합**하여 락인
- **데이터 신뢰성**이 핵심 가치 — 현장 스킨십을 통한 고빈도 반복 행동 데이터 수집

### 슈파 로드맵

| 단계 | 과업 | 산출물 |
|---|---|---|
| 1단계 (4월) | 행궁동 10개 팀 현장 밀착 AX 빌드업 | 매장별 AI 시스템 + 현장 인사이트 |
| 2단계 (4월 말~) | 실증 결과 기사화 (비로컬 등) + 확장 | 레퍼런스 + 대외 인지도 |
| 3단계 (하반기) | 슈파 웹 구현 + TIPS 신청 | 슈파 정식 MVP |

> 핵심: 남들은 상상으로 만들지만, 우리는 직접 붙어서 한 달간 빌드업한다. 이게 설득력이다.

### 미식가 AX 실증 사례

- 임대료 감면/CPC 광고비 지원 대신 → **브랜딩 중심 역제안**
- 입장포차 사례처럼 브랜드 스토리 강화에 투자
- 광고비는 마약 같은 리워드 — 브랜드 가치를 높이는 환경 개선에 우선 투입

## 2. 신도시 벤처스튜디오 (AC)

- 박승현 주도 + 엄수빈 재무 실무 + 이승훈 보조
- **4월 내 법인 전환 완료** — 정관 작성 + 사업 목적(소프트웨어 개발 등) 추가 (~4/15)
- 참여 인력의 **지분/급여 구조 최종 조율** (~4/30)
- 산하: 로컬벤처스튜디오(AC), 슈파(SUPA), 수원형 모두의창업, 락성연
- 목표: AC 등록 + 소상공인 발굴/육성 구조 만들기
- MVP 실증 데이터 → IR 자료 고도화 → 투자 유치(FI) 연결

### 모두의창업 확장

- 경기도 남부(수원, 화성, 오산, 평택) 창업 생태계 구축
- 노동부 KDT 사업 구조 개편에 맞춰 AI 실무 인재 교육/창업 지원금 확보
- 방산(드론 코딩), 약학(웨어러블) 등 1인 창업 멘토링 사례 확보 중

## 3. 공존공간

- 대규모 공사/부동산 정리 일단락 → **운영 효율화 + FSM 안착** 단계
- "끝"의 기준: ① 업무 흐름 문서화 ② 담당자 확정 ③ 최소 틀이 돌아감
- 공존공간 정리 완료 → 이승훈이 AX로 넘어감

### FSM 구체화 (2026-03-30 확정)

단순 노션 기록을 넘어, AI가 자동으로 데이터를 분석하는 시스템:
- **메일 포워딩**: 세무/회계 등 루틴 데이터는 수신 즉시 AI 분석 (~4/10)
- **로컬 폴더**: 직원은 파일 이동 등 최소 물리적 작업만 수행
- **대시보드**: FSM 운영 데이터가 자동 반영, 정보 비대칭 해소
- **깃허브 연동**: 다음 주부터 실제 운영 착수 (2026-03-30 결정)
- **실무자 업무 일지**: AI 분석으로 최적 시간 배분 모델 도출 → 성과 측정/급여 연동

## 4. 지역경영협의회 (신규, 2026-03-27)

- 기존 마을 만들기/도시재생의 한계 극복 → **지역경영** 틀로 통합
- AI + 투자가 결합된 모델로 정부 부처(정무수석실, 행안부, 고용노동부) 통합 대응
- **사단법인 설립**: 리젝트 → 보완 서류 작성 후 재투고 (즉시)
- 상권 기획자, 빈집 관리사 협회 등 유관 단체와 전략적 연대
- 슈파 10개 팀 실증 기사화 → TIPS + 정부 예산 확보 근거로 활용

- 김민호 담당: 코어 엔진 및 웹 MVP 빌드 착수
- 미결: LLM 기반 화자 분리 특허 전략 — 법률적 검토 필요

## AI 에이전트 전략 (참고)

- **풀 팅커벨**: 아이디에이션/전략 — 암묵지 이해, 방대한 배경 지식 기반 추론
- **라이트 팅커벨**: 실행/프로덕트 — 신속한 아웃풋 생성
- 추론 능력 ↔ 생산 속도는 트레이드오프 → 용도별 세션 분리로 해결
- 프론트매터(YAML) + 이벤트 소싱 기반 컨텍스트 관리 표준화
- HWPX 기반 공문서 자동 생성 워크플로우 구축 중

## 데이터 전략 (참고)

- 데이터의 양보다 **질(신뢰도)**에 집중
- 형용사/부사 기반 **취향 인덱스** 설계 (넷플릭스/왓챠 참조)
- 하이퍼레일(정성 데이터) + 슈퍼파트너(정량 데이터) 기술 연동
- 모든 데이터는 원본 보존 + YAML 프론트매터 마크다운으로 변환
