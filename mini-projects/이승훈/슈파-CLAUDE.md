## 1. 정체성

### 브랜드명
**SUPA (Super Partners, 슈퍼파트너스)** — "사장님의 슈퍼파트너, 슈파"

### ⚠️ 두 개의 정체성 정의가 공존함 (긴장 플래그)

슈파에는 현재 **대외용 정체성**과 **핵심 문제 정체성**이 병존한다. 둘은 모순되진 않지만 **강조점이 다르다.** 이건 숨길 게 아니라 플래그할 지점.

#### A. 대외용 (브랜드 페이지 최종본, OneDrive)
> 소상공인 사장님이 **장사에만 집중할 수 있도록, 나머지를 같이 하는 서비스**. 공존공간이 2년간 수원에서 사장님들과 직접 만나며 만들었다. **IT 회사가 만든 게 아니라 같이 장사하면서 시작한** 서비스다.
>
> — "사장님이 장사에만 집중할 수 있게, 나머지를 같이 합니다"

→ 강조: **운영 잡무 대체** (식자재 공동구매, 거래 정리, 리뷰 답변, SNS)

#### B. 핵심 문제 정의 (v5 확정, 2026-04-07)
> **"소상공인은 데이터도 있고 소스도 알려져 있다. 그런데 '그래서 우리 가게는 누구에게 뭘 팔아야 하는가?'라는 전략적 답을 내주는 사람이 없다. 사장님이 혼자서 5개 이상의 빅데이터를 뒤져 결론을 낼 수는 없다."**
>
> — "당신이 누구를 손님으로 삼을지, 데이터로 알려드립니다"
> — 출처: `공존공간-사업관리/research/2026-04-07-슈파-핵심문제-정의.md`

→ 강조: **AI 전략 참모** — 숫자 → 전략 → 실행을 연결

#### 두 정체성의 관계 (현재 해석)
B(AI 전략 참모)가 **슈파의 핵심 레이어/뇌**이고, A(4대 운영 서비스)가 **그 위에 얹힌 실행 도구**로 보임. 즉:
- AI 전략 참모 = "누굴 타겟할지" 결정
- 운영 서비스들 = 그 타겟에 맞춰 메뉴/마케팅/응대를 정렬
- 한 번 전략이 정해지면 네 가지 운영 서비스가 그 타겟에 맞게 작동 → **구조적 락인**

**[미결정]** 이 해석이 맞는지, 아니면 A와 B가 별개 방향인지 승훈님 확정 필요.

### 핵심 포지셔닝
- 운영 도구(POS, 캐시노트, 토스플레이스) ❌
- 대시보드 서비스 ❌
- 일회성 컨설팅 ❌
- **사장님의 AI 팀 + AI 전략 참모** ✅ — "AI를 배우지 마세요. 슈파가 대신 씁니다"

### 경쟁사 포지셔닝 맵

```
캐시노트 = 매출/데이터   → "숫자를 보여준다"
토스플레이스 = 결제/POS  → "돈을 받는다"
슈파 = 전략+실행        → "고객을 만든다" / "누굴 타겟할지 알려준다"
                         ← 이 포지션은 비어 있다
```

| 카테고리 | 대표 | 제공 가치 | 한계 |
|---|---|---|---|
| 운영 도구 | Square, Toast | 돈·사람·재고 굴리기 | "누구를 타겟하라" 없음 |
| 대시보드 | 캐시노트 | 숫자 보여줌 | 전략 공백 |
| 컨설팅 | 개별 컨설턴트 | 전략 답 | 비쌈 + 일회성 + 실행 단절 |
| **슈파** | — | **숫자 → 전략 → 실행 AI 연결** | — |

### 근거 데이터 (브랜드 페이지 footer)
- 외식업 식재료비 비중 **40.4%** (농림축산식품부, 2024)
- 외식업 월 영업이익 **221만원** (식품외식경제, 2024)
- 폐업 원인 중 원재료비 부담 **46%** (중소기업중앙회, 2025)
- 공존공간 소상공인 **142회 협업** (2023~2025)

## 6. 테스트베드 + 타겟 시장

### 첫 타겟 (확정, v5 핵심문제 정의)
- **상권 유형:** 목적형 약속상권 (행궁동)
- **업종:** 음식점
- **1차 테스트베드:** 미식가의주방 (공존공간 내, 네이버 예약 + OKPOS 데이터 존재)
- **2차 확장:** 대봉통닭 (동일 상권 유형)

### 왜 목적형 약속상권인가
재방문율 만능설의 반박. 상권 유형마다 핵심 지표가 다르다.

| 상권 유형 | 핵심 지표 | 재방문율 의미 |
|---|---|---|
| 동네/주거지 | 재방문율, 단골 | 매우 중요 |
| 오피스 점심 | 회전율 | 중요 |
| 역세권/유동 | 신규 유입 | 약함 |
| 관광지 | 리뷰, 노출 | 거의 무관 |
| **목적형 약속상권** | **약속 결정자, 추천 충성도, 그룹 사이즈** | **분기 단위로 봐야 의미 있음** |

**목적형 약속상권의 특이성:**
- 손님 = 모임의 "약속 결정자 한 명"이 N명(5~10명)을 데려옴
- "단골" 정의가 다름 — 자주 오는 사람 ≠ 약속마다 우리 가게 고르는 사람
- 월 단위 재방문율로는 단골조차 0%로 잡힘 → **기존 CRM 도구 무용지물**

### 미식가의주방 사장님의 3대 고통 (이승훈 관찰)
1. 돈이 안 남는다
2. 사람 구하기 힘들다
3. **손님이 안 온다** ← 뿌리 ("풀리면 나머지도 풀린다")

### "손님이 안 온다"의 근본 원인
네이버 플레이스 광고·인스타 피드 등 **모든 유입 수단 = 광고비 싸움 = 순위 경쟁.** 소상공인은 대기업 자본력에 밀림. **원천적 해결책 필요** — 이게 슈파 AI 전략 참모의 존재 이유.

### 행궁동 상권의 이질성 (슈파가 풀 문제)
- 화성 관광객 (일회성, 외지)
- 데이트/모임 (수원 거주)
- 카페 투어 (젊은 여성, SNS)
- 가족 나들이 (주말, 그룹)
- 직장인 회식 (평일 저녁)

→ **한 가게가 다 잡으려 하면 아무것도 못 잡는다.** 슈파가 "누굴 골라야 할지" 결정해준다.

### 확장 원칙 (v5 문서 + 리서치 기준)
- 같은 상권 유형(목적형 약속)부터 검증
- 음식점 업종부터
- 다른 업종/상권 유형 확장: `[미결정 — 시점과 기준]`

## 6-B. 가격 전략 (MVP 리서치 기준)

```
무료: 리뷰 답변 월 30건 + 지원사업 알림 + SNS 콘텐츠 월 10건
프리미엄: 월 9,900원 — 무제한 + 자동 모드 + 우선 알림
```

**중요 제약 조건 (리서치):**
- 70% 이상의 소상공인은 **월 1만원 이하 or 무료** 아니면 안 씀
- **3분 안에 쓸 수 있어야 함** (배울 시간 없음)
- **즉각적 가치 체감** ("이만큼 절약됐다")
- **모바일 우선**
- **50대도 쓸 수 있는 UX** (큰 글씨, 단순 흐름)
- **"돈을 아끼게 해주는 도구" > "돈을 벌게 해주는 도구"** (절약 프레이밍이 더 설득력)

## 6-D. 확장 패턴 (해외 올인원 도구 공통)

```
[1단계] Hook 기능 (무료/저비용, 매일 사용)
    ↓ 데이터 축적
[2단계] 데이터 활용 (분석, 인사이트)
    ↓ 신뢰 확보
[3단계] 돈 관련 기능 (결제, 급여, 대출)
    ↓ 금전적 락인
[4단계] 마케팅/성장 (고객 확보, 리뷰, 광고)
    ↓ 성장 의존
[5단계] AI 자동화 (위 모든 기능을 AI가 자동 운영)
```

**슈파 맥락 적용:**
- Hook 후보 1순위 = 리뷰 답변 AI (매일 사용, 고통 직결)
- 한국은 결제/POS가 이미 포화 → Square 공식 그대로 쓸 수 없음
- 한국 고유 pain point = **"누굴 타겟할지 모름"** = 슈파의 진짜 훅
