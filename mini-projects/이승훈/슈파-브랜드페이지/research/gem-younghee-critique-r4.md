### 2-3. AX 섹션 종합 판정

AX 섹션은 **숫자가 하나도 없다.** 이것은 의도적 선택이라면 괜찮다. 브랜드 페이지의 AX 섹션은 "설득"이 아니라 "포지셔닝"이기 때문이다. 다만 아래 2가지는 확인이 필요하다:

1. **"리뷰 답변, SNS 포스팅, 거래 분석"이 슈파의 실제 제공 기능인가?** 사업계획서v5의 3대 모듈(CLUB/SQUAD/CLEAN)에는 포함되어 있지 않다. 실제 제공하지 않는 기능을 브랜드 페이지에 적으면 과대 표현이다.

2. **"AI가 돌아가고 있지만"이 현재 시제인가?** 슈파가 현재 AI를 내장한 서비스를 운영 중인지, 계획 중인지. 현재 시제("돌아가고 있지만")를 쓰려면 실제 운영 중이어야 한다.

## 5. 강점 3개

1. **r1~r3 크리틱 중 "출처 없는 시간 수치 삭제"가 정확히 반영되었다.** 30분, 20분, 1시간 등 근거 없는 숫자가 모두 제거되었다. 남아 있는 숫자는 전부 출처가 있는 것들이다 (40.4%, 221만 원, 46%, 2700만 원 등). 이것 하나로 페이지의 수치 신뢰도 기반이 확보되었다.

2. **AX 섹션이 숫자를 한 개도 넣지 않은 것은 올바른 판단이다.** AX 섹션의 역할은 "AI 포지셔닝"이지 "AI 증거 제시"가 아니다. 숫자 없이 "대기업에는 AI 팀이 있습니다 / 소상공인에게는 그런 팀이 없습니다"라는 대비 구조만으로 메시지가 전달된다. 다만, 이것이 숫자 없이도 성립하는 이유는 주장이 자명한 사실이기 때문이다. 만약 "소상공인의 70%가 AI를 못 쓴다"같은 주장을 하려면 출처가 필요했을 것이다.

3. **Impact 시나리오의 산술 구조(2,700만 -> 40% -> 5% -> 55만 -> 660만)는 여전히 견고하다.** 각 단계의 숫자가 원본 출처와 일치하며, 독자가 스스로 검증할 수 있는 투명한 계산이다. 삭제 여부와 별개로, 수치 자체의 정확성은 유지되어 있다.

## 6. 약점 3개

1. **r1~r3에서 합의된 핵심 수정 6건이 최종본에 미반영되었다.** "절반 가까이"(40.4%), "200만 원"(221만 원), "절반이 재료비 때문"(46% 복수응답 3위) -- 이 3건은 3라운드에 걸쳐 반복 지적되었고, r3 최종 문서에서 "반영 완료"로 기록되었으나 실제 HTML에는 반영되지 않았다. 크리틱 프로세스의 실효성 문제다.

2. **"리뷰 답변, SNS 포스팅, 거래 분석"이 슈파의 실제 기능인지 확인되지 않았다.** 슈파 사업계획서v5의 3대 모듈은 CLUB(공동구매), SQUAD(외주 구조화), CLEAN(위생관리)이다. 리뷰 답변 자동화와 SNS 포스팅 자동화는 이 모듈에 포함되어 있지 않다. Service 섹션(line 508~509)과 AX 섹션(line 528)에서 동일 기능을 언급하고 있어, 이것이 미구현 기능이라면 페이지 전체의 신뢰가 흔들린다.

3. **Impact 시나리오의 "순이익 200만 원 기준, 25% 증가"는 산술 오류가 있다.** 55만/200만 = 27.5%이지 25%가 아니다. 원본 기준 221만 원을 쓰면 55만/221만 = 24.9% ≒ 25%로 정확해진다. 즉, 200만 원이라는 잘못된 기준치를 쓰면서 그에 맞지 않는 백분율(25%)을 적었다. 기준치를 221만 원으로 바꾸면 자연스럽게 해결된다.

## 7. 개선 제안 3개

### 제안 1: problem-list 수치를 r2 합의안대로 즉시 수정

이것은 "개선"이 아니라 "이미 합의된 내용의 실행"이다. 3라운드에 걸쳐 반복된 지적이므로 최우선으로 반영해야 한다.

변경 대상:

```
현재: 매출의 절반 가까이가 재료비로 나갑니다
수정: 매출의 40%가 재료비로 나갑니다

현재: 남는 건 월 200만 원 남짓
수정: 남는 건 월 220만 원 남짓

현재: 문 닫으신 사장님 절반이 "재료비 때문"이라고 하셨습니다
수정: 문 닫으신 사장님 46%가 "재료비 부담"을 꼽으셨습니다
```

Impact 시나리오 수정 (잔류 시):

```
현재: 순이익 200만 원 기준, 25% 증가
수정: 순이익 약 220만 원 기준, 25% 증가
```

이렇게 하면 55만/220만 = 25.0%로 산술도 맞고, 푸터의 221만 원과도 일치한다.

### 제안 2: "리뷰 답변, SNS 포스팅" 기능의 현재 상태를 확인하고 표현을 조정

확인 결과에 따라:

- **현재 운영 중이면**: 그대로 둔다.
- **개발 중/예정이면**: "리뷰 답변, SNS 포스팅, 거래 분석까지" -> "공동구매부터 거래 분석, 리뷰 답변까지" 순서를 바꿔서 실제 운영 중인 기능(공동구매)을 앞에, 예정 기능을 뒤에 배치한다. 또는 Service 카드에 badge-now/badge-soon 구분이 있으므로(line 283~290, CSS에만 존재하고 실제 사용 안 함) 이를 활용한다.

AX 섹션에서 특히 "뒤에서 AI가 돌아가고 있지만"(line 529)이 현재 시제인 점에 주의. 미구현이면 "돌아갈 예정"으로 바꾸거나, 현재 제공 중인 기능만 나열해야 한다.

### 제안 3: AX 섹션에 숫자 1개를 추가하면 설득력이 올라간다

현재 AX 섹션은 순수 카피라이팅이다. 여기에 숫자 1개만 넣으면 "주장"이 "팩트"가 된다.

추가 권고 수치: **"소상공인 10곳 중 7곳은 아직 AI를 쓰지 못하고 있습니다"**

근거: 중소기업 AI 활용률 28.7% (중소벤처기업부, 2025) -> 미활용 71.3% ≒ "10곳 중 7곳"

배치 예시:

```
현재:
  대기업에는 AI 팀이 있습니다.
  소상공인에게는 그런 팀이 없습니다.

수정:
  대기업에는 AI 팀이 있습니다.
  소상공인 10곳 중 7곳은 아직 AI를 쓰지 못하고 있습니다.
```

주의: 이 28.7%는 "중소기업" 대상 조사이지 "소상공인" 대상이 아니다. 소상공인의 AI 활용률은 이보다 낮을 가능성이 높다. 정확히 쓰려면 "중소기업 기준으로도"라는 한정이 필요하다. 또는 "(추정)" 표기를 한다. 출처 없는 숫자는 쓰지 않는다는 원칙을 지키되, 출처가 있는 숫자를 맥락에 맞게 번역하는 것은 허용 범위다.

## 8. 총평

최종본의 숫자 7건은 정확하다. 문제는 정확한 숫자를 부정확하게 표현한 3건("절반 가까이", "200만 원", "절반이")이 3라운드 크리틱을 거치고도 수정되지 않았다는 점이다. 이것은 데이터 문제가 아니라 프로세스 문제다.

AX 섹션은 데이터 없이도 성립하는 구조로 잘 설계되었으나, "리뷰 답변, SNS 포스팅"이 실제 제공 기능인지 반드시 확인이 필요하다. 미제공 기능을 제공 중인 것처럼 쓰면, 숫자 왜곡보다 심각한 신뢰 문제가 된다.

우선순위:
1. problem-list 3건 수치 수정 (즉시)
2. "리뷰 답변, SNS 포스팅" 기능 현황 확인 (즉시)
3. Impact 시나리오 "순이익 200만 원 -> 220만 원" 수정 (즉시)
4. AX 섹션에 수치 1개 추가 여부 검토 (선택)
