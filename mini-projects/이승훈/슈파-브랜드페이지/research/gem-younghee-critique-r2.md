## 2라운드 목적

미애(브랜드)가 제안한 구조 변경 -- Impact 섹션 삭제, Service+Vision 통합 -- 을 데이터 관점에서 수용하되, 숫자의 신뢰성을 훼손하지 않는 구체적 텍스트 수정안을 제시한다.

## 1. Impact 섹션을 걷어낸다면, 데이터를 어디서 어떻게 보여줄 것인가

### 판단

미애의 판정에 동의한다. 시뮬레이션(월 2,700만 원 -> 5% 절감 -> 연 660만 원)은 브랜드 페이지가 아니라 서비스 상세 페이지 또는 IR 자료의 콘텐츠다. 브랜드 페이지에서 ROI 계산이 나오면 "영업 페이지"로 인식되며, CTA의 부드러운 톤("가게 이야기, 편하게 들려주세요")과 충돌한다.

그러나 **숫자를 완전히 없애면 안 된다.** 숫자 없는 브랜드 페이지는 "좋은 말만 하는 페이지"가 된다. 핵심은 **계산을 시키지 않되, 규모감을 전달하는 것**이다.

### 수정안: Reality 섹션에 팩트 수치 2개를 집중 배치

Impact 섹션의 시뮬레이션을 삭제하고, 그 안에 있던 핵심 데이터를 Reality 섹션으로 옮긴다. Reality는 "문제의 크기"를 보여주는 섹션이므로, 숫자가 여기에 있는 게 맥락상 자연스럽다.

**배치할 수치 2개:**
- 식재료비 비중 40.4% (농림축산식품부, 2024)
- 월 영업이익 약 220만 원 (식품외식경제, 2024)

**배치하지 않을 수치:**
- 공동구매 5% 절감, 월 55만 원, 연 660만 원 -> 서비스 상세 페이지로 이동
- 순이익 25% 증가 -> 서비스 상세 페이지로 이동

이유: 절감 시뮬레이션은 "슈파를 쓰면 이만큼 아낍니다"라는 영업 메시지다. 브랜드 페이지에서는 "이 문제가 얼마나 큰가"만 보여주면 된다. "해결하면 얼마가 된다"는 다음 페이지의 역할이다.

## 2. 출처 없는 시간 수치 처리안

### 대상 수치 4개

| 현재 표현 | 출처 | 판정 |
|---|---|---|
| "가격 비교에 하루 1시간" | 없음 | 삭제 또는 전환 |
| "장부 정리에 매일 30분" | 없음 | 삭제 또는 전환 |
| "리뷰 답변에 매일 30분" | 없음 | 삭제 또는 전환 |
| "SNS에 매일 20분" | 없음 | 삭제 또는 전환 |

### 수정안: 구체적 시간을 삭제하고, 체감 표현으로 전환

출처 없는 숫자를 브랜드 페이지에 두는 것은 위험하다. "30분"이라는 숫자를 보고 "우리 가게는 10분이면 되는데?"라고 생각하는 사장님이 한 명이라도 있으면, 그 순간 페이지 전체의 숫자 신뢰가 무너진다.

**Reality 섹션 problem-list 수정안:**

현재:
```
- 가격 비교에 하루 1시간, 장부 정리에 매일 30분
- 리뷰가 쌓여도 답변 쓸 시간이 없습니다
```

수정 후:
```
- 가격 비교, 장부 정리, 리뷰 답변 -- 장사 외의 일이 장사 시간을 잡아먹습니다
```

하나로 합친다. 구체적 분(分) 수치를 빼는 대신, "장사 외의 일이 장사 시간을 잡아먹는다"는 체감을 전달한다. 사장님은 자기 가게 기준으로 알아서 대입한다.

**Impact 섹션 삭제 시, 아래 문장도 함께 삭제:**

현재:
```
리뷰 답변에 매일 30분, SNS에 매일 20분 쓰셨다면,
그 50분을 손님한테 쓰실 수 있습니다.
```

이 문장은 Impact 섹션과 함께 삭제된다. 만약 시간 절약 메시지를 남기고 싶다면, 통합된 Service+Vision 섹션에서 아래처럼 쓴다:

```
사장님이 잡무에 쓰던 시간이 돌아옵니다.
그 시간에 손님 한 분을 더 챙기실 수 있습니다.
```

숫자 없이, 체감만. 브랜드 페이지에서는 이것으로 충분하다.

## 3. "200만 원" vs "221만 원" 불일치 해결안

### 현황

| 위치 | 표현 | 원본 수치 |
|---|---|---|
| Reality problem-list | "남는 건 월 200만 원 남짓" | 221만 원 |
| Impact 시나리오 | "순이익 200만 원 기준" | 221만 원 |
| 푸터 출처 | "월 영업이익 221만원" | 221만 원 |

본문은 200만 원, 푸터는 221만 원. 같은 페이지 안에서 기준 수치가 다르다.

### 수정안

**"220만 원"으로 통일한다.**

Reality problem-list:
```
현재: 남는 건 월 200만 원 남짓
수정: 남는 건 월 220만 원 남짓
```

이유:
- 221만 원의 합리적 반올림은 220만 원이다 (200만 원이 아니라).
- "200만 원"은 21만 원(약 10%)을 내림한 것이어서, "남짓"이라는 표현으로도 커버하기 어렵다.
- 220만 원이면 푸터의 221만 원과 1만 원 차이. "남짓"으로 자연스럽게 포함된다.
- Impact 시나리오가 삭제되므로, 시나리오 내 "순이익 200만 원 기준, 25% 증가" 문장은 자동으로 사라진다. 불일치 해소.

**푸터 출처는 그대로 221만 원 유지.** 본문은 읽기 편한 반올림(220만 원), 출처는 정확한 원본(221만 원). 이 정도 차이는 "남짓"이 커버한다.

## 4. "절반" 표현 수정안

### 대상 2건

**(1) "매출의 절반 가까이가 재료비"**

원본: 40.4% (농림축산식품부, 2024)

현재: `매출의 절반 가까이가 재료비로 나갑니다`
수정: `매출의 40%가 재료비로 나갑니다`

이유: 40.4%와 50%는 약 10%p 차이다. 외식업에서 10%p는 흑자와 적자의 경계선이다. "절반 가까이"는 과장이다. 반면 "40%"는 그 자체로 충격적인 숫자다. "매출 10에 4가 재료비"라고 읽힌다. 과장할 필요가 없다.

인라인 출처 추가 권고: `매출의 40%가 재료비로 나갑니다` 뒤에 작은 글씨로 `(농림축산식품부, 2024)` 표기. 숫자 옆에 출처가 있으면 "주장"이 아니라 "팩트"가 된다.

**(2) "문 닫으신 사장님 절반이 '재료비 때문'"**

원본: 46.0%, 복수응답 3위 항목 (중소기업중앙회, 2025)

현재: `문 닫으신 사장님 절반이 "재료비 때문"이라고 하셨습니다`
수정: `문 닫으신 사장님 46%가 "재료비 부담"을 꼽으셨습니다`

이유:
- 46%를 "절반"이라 하면 4%p를 올린 것이다. 방향이 과장 쪽이다.
- 더 중요한 문제: 이 46%는 복수응답이며, 1위 고객 감소(52.2%), 2위 인건비(49.4%)에 이어 3위다. "절반이 재료비 때문"이라 하면 마치 재료비가 폐업의 최대 원인인 것처럼 읽힌다. 사실과 다르다.
- "46%가 꼽으셨습니다"로 바꾸면, (a) 정확하고 (b) "절반에 가깝지만 절반은 아닌" 수치를 왜곡 없이 전달하고 (c) "꼽으셨습니다"라는 동사가 복수응답임을 암시한다.

## 6. Impact 섹션 삭제 후, 통합 섹션에 남길 데이터 장치 1개

Impact 섹션의 시뮬레이션은 삭제하되, **"5%만 줄여도"라는 보수적 수치는 살릴 가치가 있다.** 이 수치는 references에 "업계 추정 15~20%" 대비 보수적으로 잡은 것이어서, 과장하지 않겠다는 태도를 보여주는 신뢰 장치다.

미애가 제안한 통합 섹션(Service+Vision -> "WHAT WE DO") 안에, 시뮬레이션이 아닌 한 줄 팩트로 삽입한다.

### 수정안

통합 섹션 본문 중, 공동구매 언급 부분:

```
식자재를 같이 사서 단가를 낮추고,
```

이것을 아래로:

```
식자재를 같이 사서 단가를 낮춥니다.
업계 평균 절감률은 15~20%. 보수적으로 5%만 잡아도 변화가 시작됩니다.
```

시뮬레이션(=계산 과정)은 없앴지만, "5%"와 "15~20%"라는 맥락은 남긴다. 독자가 스스로 "5%면 우리 가게는 얼마지?"를 떠올리게 하되, 계산을 강요하지 않는다.

단, 이 표현을 쓰려면 "업계 평균 절감률 15~20%"의 출처가 references에 명시되어야 한다. 슈파-식자재시장-데이터-2026.md의 주의사항에 "업계 추정"이라 되어 있으므로, 인라인에 `(업계 추정치)` 표기가 필요하다.

참고 수치 출처: 외식업 식재료비 비중 40.4%(농림축산식품부, 2024), 외식업 월 영업이익 221만원(식품외식경제, 2024),

- "고객 소비액, 만족도" 삭제 -- 본문에서 사용하지 않는 출처는 제거

## 요약: 강점 3개, 약점 3개, 개선 제안 3개

### 강점

1. **미애의 구조 제안(Impact 삭제, Service+Vision 통합)을 수용하면, 데이터의 배치가 오히려 강해진다.** 시뮬레이션을 빼면 Reality 섹션의 팩트 수치 3개(40%, 220만 원, 46%)가 페이지에서 유일한 숫자가 된다. 숫자가 적을수록 각 숫자의 임팩트가 커진다.

2. **"5%만"이라는 보수적 수치를 통합 섹션에 한 줄로 남기면, 시뮬레이션 없이도 설득력이 유지된다.** "업계 15~20% 중 5%만"이라는 맥락이 한 문장으로 전달되면, 독자는 스스로 계산한다. 강요된 계산보다 자발적 계산이 설득력이 높다.

3. **출처 없는 시간 수치 4개를 삭제하면, 남은 숫자 전체의 신뢰도가 올라간다.** 출처 있는 숫자 3개 + 출처 없는 숫자 4개 = 페이지 신뢰도 43%. 출처 없는 숫자를 빼면 100%. 숫자의 신뢰는 가장 약한 고리에 맞춰진다.

### 약점

1. **Reality 섹션의 problem-list를 3개로 줄이면, 전부 "돈 이야기"가 된다.** 40%, 220만 원, 46% -- 세 항목 모두 재료비/수익 관련이다. "시간"과 "감정"의 차원이 빠진다. 본문("가게 문 열기 전에 식자재 주문. 가게 문 닫고 나서 리뷰 답변...")이 이를 보완하지만, list와 본문의 메시지 균형을 의식적으로 확인할 필요가 있다.

2. **"업계 평균 절감률 15~20%"의 출처 강도가 약하다.** references에 "업계 추정"으로만 되어 있다. 구체적 조사 기관이나 보고서명이 없다. 본문에 쓰려면 "(업계 추정치)"라고 명시해야 하며, 이 표현이 신뢰를 높이는지 오히려 의심을 부르는지는 판단이 필요하다. 출처가 불확실하면 이 한 줄도 삭제하는 게 안전하다.

3. **인라인 출처 표기는 디자인과의 조율이 필요하다.** "40%(농림축산식품부, 2024)"를 problem-list 안에 넣으면 가독성이 떨어질 수 있다. 디자인적으로 작은 글씨 처리가 가능한지 확인 필요. 불가능하면 푸터 출처만으로 유지하되, problem-list의 숫자가 정확하면 신뢰는 유지된다.

### 개선 제안

1. **Reality problem-list를 위 5번 수정안대로 3개 항목으로 교체한다.** 정확한 숫자 3개가 모호한 표현 5개보다 강하다.

2. **Impact 섹션 전체를 삭제하고, 두 번째 인용 블록도 함께 삭제한다.** 미애 제안대로 사람의 목소리(사장님 또는 대표)로 교체하면, 데이터와 감성의 균형이 맞는다. 데이터는 Reality에, 감성은 인용에.

3. **통합 섹션(WHAT WE DO)에 "5%" 한 줄을 넣을지 여부는 "업계 추정 15~20%"의 출처 확보 가능 여부로 결정한다.** 출처를 확보할 수 있으면 넣는다. 확보할 수 없으면 넣지 않는다. 출처 없는 숫자는 쓰지 않는다는 원칙을 지킨다.
