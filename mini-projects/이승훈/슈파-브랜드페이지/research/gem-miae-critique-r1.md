## 크리틱 요약

이 페이지를 처음 읽는 사람은 "소상공인 동반자"라는 슈파의 진심을 충분히 느낄 수 있다. 다만 파일명(v2-장황.html)이 스스로 고백하듯, **같은 메시지를 여러 번 반복하면서 감정의 농도가 희석**되는 구간이 있다. "브랜드 소개 페이지"치고는 서비스 설명과 시뮬레이션이 너무 구체적이어서, 읽는 사람이 중반부터 "이거 영업 페이지 아닌가?"라는 느낌으로 전환될 수 있다.


## 상세 크리틱

### 1. 브랜드 톤의 일관성 -- "LIFESTYLE NEARBY"와의 결

**판정: 60% 정도 맞다.**

공존공간의 "LIFESTYLE NEARBY"는 세련되고 여유 있는 톤이다. 슈파 페이지의 "사장님, 장사 말고 해야 할 일이 너무 많지 않으십니까?" 같은 직접 화법은 따뜻하고 좋지만, 공존공간 모체 브랜드의 결과는 다른 궤도에 있다.

- 공존공간 = 미니멀, 감각적, 영문 키워드 중심
- 슈파 페이지 = 구어체, 공감형, 한국어 중심

이건 나쁜 게 아니라 **의도적 분리인지 비의도적 괴리인지**가 불분명하다는 게 문제다. 공존공간 홈페이지의 "브랜드" 메뉴 안에 들어가는 페이지라면, 방문자가 공존공간 -> 슈파로 넘어오면서 **톤이 갑자기 바뀌는 느낌**을 받을 수 있다.

**강점:**
- 다크톤, Montserrat+NanumSquareRound, 포인트컬러 #00B8FF -- 비주얼 시스템은 완벽하게 일치한다
- section-label에 영문 키워드(Reality, Origin, Service 등) 사용 -- 공존공간 스타일과 통일감 있다

**약점:**
- 본문 톤이 공존공간의 나머지 페이지와 다를 가능성이 높다. "사장님~하십니까?" 화법은 슈파 단독 랜딩페이지라면 완벽하지만, 공존공간 홈페이지 안의 하위 페이지로는 톤 점프가 크다.


### 2. 스토리 흐름 -- 섹션 전환의 자연스러움

**전체 구조:**
Hero -> Reality(공감) -> 인용 -> Origin(출발점) -> Service(서비스) -> Vision(비전) -> Impact(시나리오) -> 인용 -> Principle(원칙) -> CTA

**판정: 구조는 좋다. 그런데 중반에 늘어진다.**

- Hero -> Reality -> 인용 -> Origin: 이 흐름은 훌륭하다. "문제 인식 -> 감정적 전환 -> 우리가 왜 이걸 하는지"가 자연스럽게 연결된다.
- **Service -> Vision -> Impact 구간이 문제다.** Service에서 이미 4개 서비스를 소개하고, Vision에서 같은 서비스를 "하나의 흐름"으로 다시 설명하고, Impact에서 공동구매 시뮬레이션을 또 한다. 읽는 사람 입장에서 "이거 아까 봤는데?" 하는 순간이 온다.
- 두 번째 인용("사장님한테 필요한 건 최신 기술이 아니라...")은 Origin 섹션 끝에서 이미 거의 같은 문장이 나왔다. 반복이다.


### 3. 불필요하거나 과한 부분

**(1) Impact 섹션의 수치 시뮬레이션이 너무 구체적이다**

"월 매출 2,700만 원 가게라면 -> 식재료비 1,100만 원 -> 5% 절감 -> 연 660만 원" 이 계산은 IR 자료나 사업계획서에서는 킬러 콘텐츠다. 하지만 **브랜드 소개 페이지에서는 과하다.** 방문자 입장에서 "이 회사가 나를 설득하려고 하는구나"라는 방어막이 올라오는 순간이다.

브랜드 페이지의 역할은 "이 브랜드가 뭘 믿는가"를 느끼게 하는 것이지, "얼마를 아낄 수 있는가"를 증명하는 게 아니다.

**(2) Vision 섹션이 Service 섹션과 겹친다**

Service에서 공동구매/거래분석/리뷰답변/SNS를 소개한 뒤, Vision에서 "공동구매, 거래 분석, 리뷰 답변, SNS 포스팅. 따로따로 보이지만 하나의 흐름입니다"라고 다시 나열한다. 읽는 사람은 이미 안다. 이걸 다시 말해줄 필요가 없다.

**(3) 두 번째 인용 블록이 Origin 본문과 중복**

Origin 섹션 마지막 문장: "사장님한테 필요한 건 '최신 기술'이 아니라 '당장 도움 되는 것'이라는 걸."
두 번째 인용 블록: "사장님한테 필요한 건 '최신 기술'이 아니라 '당장 도움 되는 것'입니다."
거의 동일한 문장이 두 번 나온다. 강조가 아니라 실수처럼 보인다.

**(4) problem-list의 항목이 많다**

5개 항목이 나열되는데, 읽는 사람의 감정 곡선을 생각하면 3개로 충분하다. 항목이 많아지면 "데이터 나열"이 되고, 감정이 아니라 정보 처리 모드로 빠진다.


### 4. 빠진 스토리

**(1) "공존공간"이 뭔지에 대한 최소한의 맥락**

Origin 섹션에서 "슈파를 만든 건 공존공간입니다. 수원 행궁동에서 사장님들과 함께 일하는 지역경영회사입니다."라고 한 줄로 끝난다. 공존공간 홈페이지 안의 페이지이므로 이미 아는 사람이 보는 거라면 괜찮지만, 외부 링크로 직접 들어오는 경우를 감안하면 공존공간의 정체성을 한 호흡 더 줘도 좋다.

**(2) 사람의 얼굴이 없다**

브랜드 페이지에서 가장 기억에 남는 건 "누가 이걸 만들었는가"다. 대표의 한마디, 팀의 사진, 혹은 실제 사장님의 목소리 -- 이런 인간적 요소가 전혀 없다. 모든 게 설명문이다. 인용 블록이 2개 있지만 둘 다 슈파 자체의 메시지이지, 사람의 목소리가 아니다.

**(3) "LIFESTYLE NEARBY"와의 연결 고리**

공존공간의 "LIFESTYLE NEARBY" 철학이 어떻게 슈파로 이어지는지에 대한 브릿지가 없다. 공존공간 홈의 브랜드 메뉴에서 슈파 페이지로 들어오는 사람은 "이 회사의 여러 브랜드 중 하나"로 인식할 텐데, 그 연결이 없다.


### 5. "브랜드 소개 페이지" vs "영업 페이지"

**판정: 전반부는 브랜드 페이지, 후반부는 영업 페이지.**

Hero ~ Origin까지는 "우리가 왜 이걸 하는가"라는 브랜드 스토리다. 완벽하다. 그런데 Service부터 분위기가 바뀐다. 서비스 카드 4개, 수치 시뮬레이션, "먼저 결과를 보여드리겠습니다" -- 이건 전형적인 영업/랜딩 페이지 문법이다.

홈페이지에서 사람을 모으는 게 목적이 아니라는 브리프에 비추면, CTA 섹션은 이미 "가게 이야기, 편하게 들려주세요"로 적절하게 부드럽지만, 그 앞의 시뮬레이션과 서비스 카드가 그 톤과 충돌한다.


## 개선 제안

### 제안 1: Service + Vision을 하나로 합치고, 설명이 아니라 감각으로 보여주기

**현재:** Service 섹션에서 4개 서비스를 카드로 나열 -> Vision에서 같은 4개를 다시 "흐름"으로 설명
**문제:** 같은 이야기를 두 번 하고 있다.

**제안:** 서비스 카드를 없애고, Vision 섹션 하나로 통합한다. "슈파가 하는 일"을 항목이 아니라 한 편의 이야기로 전달한다.

변경 전:
```
(Service 섹션)
공동구매 / 거래 분석 / 리뷰 답변 / SNS 포스팅 [카드 4개]

(Vision 섹션)
공동구매, 거래 분석, 리뷰 답변, SNS 포스팅.
따로따로 보이지만, 하나의 흐름입니다.
사장님이 식자재를 주문하면, 그 데이터가 장부에 자동으로 쌓이고...
```

변경 후 (예시):
```
WHAT WE DO

사장님이 장사만 하셨는데,
가게 운영 전체가 돌아갑니다.

식자재를 같이 사서 단가를 낮추고,
거래 내역은 자동으로 정리되고,
손님 리뷰엔 사장님 말투 그대로 답변이 달리고,
가게 소식은 알아서 SNS에 올라갑니다.

큰 프랜차이즈에는 본사가 해주는 일들이 있습니다.
슈파는 동네 사장님한테 그 역할을 합니다.
```

카드형 나열 대신 **하나의 흐름으로 읽히게** 만들면, "아 이게 다 연결되는 거구나"를 구조가 아니라 리듬으로 느끼게 된다.


### 제안 2: Impact 섹션의 수치 시뮬레이션을 걷어내거나 대폭 축소

**현재:** 월 매출 2,700만 원 기준 시뮬레이션 (3단계 + 결과)
**문제:** 브랜드 소개 페이지에서 ROI 계산은 과하다. "설득하려는 느낌"이 생긴다.

**제안:** 숫자 대신 체감으로 바꾼다.

변경 전:
```
월 매출 2,700만 원 가게라면
1. 식재료비 약 1,100만 원 (매출의 40%)
2. 공동구매로 5%만 줄이면
3. 월 55만 원 절감, 연 660만 원
순이익 200만 원 기준, 25% 증가
```

변경 후 (예시):
```
가게 하나가 달라지는 모습

재료비가 조금 줄었습니다.
리뷰 답변에 쓰던 30분이 생겼습니다.
그 시간에 손님 한 분을 더 챙겼습니다.

작은 차이가 매일 쌓이면,
가게의 하루가 달라집니다.
```

구체적 수치가 필요하면 별도의 서비스 상세 페이지에서 다루면 된다. 브랜드 페이지에서는 "느낌"만 전달하는 게 맞다.


### 제안 3: 두 번째 인용 블록을 사람의 목소리로 교체

**현재:** "사장님한테 필요한 건 '최신 기술'이 아니라 '당장 도움 되는 것'입니다." (Origin 본문과 중복)
**문제:** 같은 문장 반복 + 브랜드 자체의 메시지만 있고 사람의 목소리가 없다.

**제안:** 실제 사장님의 한마디, 또는 대표의 철학을 넣는다.

변경 후 (예시 A -- 사장님 목소리):
```
"처음엔 또 뭐 팔려고 하나 했는데,
진짜 옆에서 같이 하더라고요."
-- 행궁동 OO식당
```

변경 후 (예시 B -- 대표 목소리):
```
"기술을 만드는 게 아니라
사장님 옆자리를 채우는 겁니다."
-- 공존공간 이승훈
```

인용 블록은 페이지에서 가장 눈에 띄는 요소다. 여기에 실제 사람의 목소리를 넣으면, 이 브랜드가 "진짜 현장에서 일하는 사람들의 것"이라는 신뢰감이 생긴다.

### 부록: 강점/약점 요약

**강점 3개:**
1. **비주얼 시스템이 공존공간과 완벽히 통일되어 있다.** 다크톤, 폰트, 포인트컬러, 섹션 라벨 -- 디자인 언어가 흔들리지 않는다.
2. **Reality-Origin 흐름이 감정적으로 탄탄하다.** "사장님 잘못이 아닙니다. 구조가 그렇습니다" -> "같이 하면 어떨까?" -> "주방 옆에서 시작했습니다" 이 세 줄의 연결이 이 페이지에서 가장 좋은 부분이다.
3. **CTA가 적절하게 부드럽다.** "가게 이야기, 편하게 들려주세요" -- 영업이 아니라 대화를 제안하는 톤. 브랜드 페이지에 맞다.

**약점 3개:**
1. **중반부(Service~Impact)에서 같은 메시지가 3번 반복된다.** 서비스 카드 -> 흐름 설명 -> 수치 시뮬레이션. 읽는 사람의 집중력이 떨어지는 구간이다.
2. **사람의 얼굴이 없다.** 인용 2개 모두 브랜드 자체 메시지. 실제 사장님이나 팀의 목소리가 없어서 "기업 소개서"처럼 읽힌다.
3. **후반부 톤이 영업 페이지로 기운다.** 수치 시뮬레이션, "먼저 결과를 보여드리겠습니다" 같은 표현은 전환율을 노리는 랜딩 페이지 문법이다. 브랜드 소개와 충돌한다.
