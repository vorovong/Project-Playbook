## 회사 정의
- (주)신도시 = "The First City OS Provider"
- 도시의 상업 활동을 관장하는 운영체제(OS)를 공급하는 로컬 테크 기업
- 핵심 솔루션: 슈파(SUPA) = 소상공인 AI 경영 파트너

## 구조: PropCo / OpCo 분리
- (주)공존공간 [PropCo] — 부동산 자산, 거버넌스, 파트너십
- (주)신도시 [OpCo] — 슈파 개발/운영, City OS, Asset-Light

## Problem
- 소상공인 790만 개 (전체 95.2%), 종사자 1,090만 명 (45.9%)
- 구조적 문제: 경영 비효율, 고비용 구조, 기존 지원 한계, 디지털 격차

## Solution — 슈파(SUPA)
- 한줄: AI로 경영 도와주고, 유저 모이면 비용까지 낮춰줌
- 핵심 공식: AI 운영비 = 0 → 수수료 극도로 낮게 → 무료/소액이 전략

## 3단계 로드맵

### Phase 1: 행궁동/수원 장악 (~27년 Q2)
- Cost Innovation: 공동구매로 프랜차이즈급 공급가, Shared Service(세무/법률/방역 N분의1)
- AI 경영 효율화: 대시보드, 세무자동정리, SNS자동생성, 구매몰, 고객관리, AI상담
- 완전 무료 + 프리미엄 쿠폰 전략
- 구매몰 수수료 3% (PG 1.5% 제외 시 실질 마진 1.5%)
- KPI: 행궁동 100개소 → 수원 1,000개소

### Phase 2: 국내 확장 + 수익화 (27년~)
- 구독 월 4,900원 (전환율 30%), 광고, 데이터 판매
- 상권 기금 선순환 (구독료 일부 → 공존 기금 → 상권 활성화 → 매출 증가)
- 데이터 판매처: 금융사, 유통사, 보험사, 지자체, 프랜차이즈 본사

### Phase 3: 피지컬 AI
- 배달/물류 로봇, 매장 자동화, 무인 주문결제, 재고 자동 관리
- 핵심: 현장 채널 + 데이터 + 현장 지식을 가진 곳은 우리뿐

## 기획서 활용 포인트
- "SaaS가 아니라 커머스 플랫폼" — 구독료가 아닌 거래 수수료가 핵심 매출
- "AI 운영비 = 0 → 무료가 전략" — 기존 SaaS와 근본적으로 다른 모델
- Toast(33조원) = 외식 POS → 금융/데이터 플랫폼 진화 = 글로벌 선례
- "SaaSpocalypse" 맥락 — 좌석당 구독 모델의 종말, 슈파는 이미 다른 길
- 공존 기금 = 경쟁사가 복제 불가한 해자(moat)
