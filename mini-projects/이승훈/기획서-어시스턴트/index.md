# 기획서 어시스턴트 — 자료 색인

이 파일을 먼저 읽고, 필요한 요약본만 찾아가세요.

---

## 사용법

1. 기획서 작업 요청이 오면 이 index.md를 먼저 읽는다
2. `rules/기본-작성규칙.md`는 **항상** 읽고 적용한다 (항목 번호, 표기법, 표현법, 띄어쓰기, 용어)
3. 공문서 요청 시 `rules/공문서-양식규칙.md`도 **추가로** 읽고 적용한다
4. 주제에 맞는 요약본만 읽는다 — 경로: `바탕 화면/DB/sources/`
5. 요약본으로 부족하면 그때 원본을 읽는다
6. profile.md는 항상 참고한다 (톤, 스타일, 포지셔닝)
7. 같은 주제의 자료가 여러 개면, 가장 최신 파일의 숫자를 쓴다

> **데이터 저장 경로**: `C:\Users\leeha\OneDrive\바탕 화면\DB\`
> - `DB/sources/` — 요약본 (기획서, 회의록 등)
> - `DB/references/` — 외부 참고자료 (통계, 벤치마크, 인사이트)

## 요약본 작성 규칙

새 자료가 들어오면 아래 4단계를 **모두** 완료한다:

1. **원본 정리** — 원본 파일을 `바탕 화면/DB/원본/{폴더명}/`에 복사한다.
   - **폴더명 결정 규칙:** 요약본의 제목/주제에서 핵심 키워드를 추출한다. `원본/` 하위에 같은 주제의 기존 폴더가 있으면 그 안에 넣고, 없으면 새 폴더를 만든다.
   - 예: "모두의창업 생태계 조성.pdf" → 기존 `원본/모두의창업/`에 추가. "AI별동대 지원서.docx" → `원본/AI별동대/` 새로 생성.
2. **요약본 생성** — 아래 양식에 맞춰 `바탕 화면/DB/sources/`에 요약본(.md)을 만든다.
3. **index.md 업데이트** — 자료 목록 + 원본 파일 목록 + 주제별 빠른 찾기 + 업데이트 기록을 갱신한다.
4. **저장** — git commit + push

### 회의록 양식

```
# 회의 요약: {제목}

- 날짜: YYYY-MM-DD
- 참석: 이름, 이름, ...
- 시간: N분
- 원본: {원본 파일 경로}

---

## 한줄 요약
(1~2문장)

## 기획서 활용 포인트

```
