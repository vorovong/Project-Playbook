# 기획서 어시스턴트 — 자료 색인

이 파일을 먼저 읽고, 필요한 요약본만 찾아가세요.

---

## 사용법

1. 기획서 작업 요청이 오면 이 index.md를 먼저 읽는다
2. 주제에 맞는 요약본(sources/)만 읽는다
3. 요약본으로 부족하면 그때 원본을 읽는다
4. profile.md는 항상 참고한다 (톤, 스타일, 포지셔닝)
5. 같은 주제의 자료가 여러 개면, 가장 최신 파일의 숫자를 쓴다

---

## 요약본 작성 규칙

새 자료가 들어오면 아래 양식에 맞춰 sources/에 요약본을 만들고, 자료 목록과 주제별 빠른 찾기를 업데이트한다.

### 회의록 양식

```
# 회의 요약: {제목}

- 날짜: YYYY-MM-DD
- 참석: 이름, 이름, ...
- 시간: N분
- 원본: {원본 파일 경로}

---

## 한줄 요약
(1~2문장)

## 핵심 결정
- 결정 사항 나열

## 기획서 활용 포인트
- 이 회의 내용을 기획서에 어떻게 쓸 수 있는지
```

### 문서(기획서/보고서/PDF) 양식

```
# {문서 제목}

- 원본: {원본 파일 경로}
- 작성일: YYYY-MM-DD (또는 추정 시기)
- 페이지: Np

---

## 핵심 내용
(문서의 구조를 살려서 정리)

## 핵심 숫자
(표로 정리)

## 기획서 활용 포인트
- 이 문서를 기획서에 어떻게 쓸 수 있는지
```
