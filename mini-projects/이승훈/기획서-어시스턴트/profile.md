## 1. 인물 포지셔닝

### 이승훈
- 공존공간 공동 창업자 / 실무 총괄
- 패스트 무버: AI 도구(클로드 코드, 바이브 코딩)를 실무에 즉시 적용
- 현장 중심: 데이터와 숫자보다 실행과 관계를 먼저 움직임
- B2B 감각: 소상공인의 실제 불편함을 비즈니스로 전환하는 능력
- **핵심 철학: 상생하는 구조**
  - 점·선·면의 논리: 개별 가게(점)가 살아야 → 가게들이 연결되고(선) → 상권 전체(면)가 산다. 점을 무시하고 면을 말하지 않는다
  - 사장님이 돈을 벌어야 우리도 번다: 우리 수익은 소상공인의 비용이 아니라 소상공인의 성장에서 나오는 구조. 이것이 모든 기획의 기반

### 박승현
- 공존공간 대표 / CSP 대표
- 전략 설계자: 지분 구조, 투자 유치, 펀드 조성 등 파이낸셜 아키텍트
- 비전: 로컬 상권을 글로벌로 연결하는 "글로컬" 전략의 설계자

## 2. 회사 포지셔닝

### 공존공간 (주)
- 수원 행궁동 기반 상권 활성화 + 로컬 테크 기업
- 핵심 정체성: "상권 기획자" — 단순 용역사가 아닌, 투자와 운영을 병행하는 액셀러레이터(AA 모델)
- 트렌드 선구자: 글로컬(Glocal) 전략을 한국 최초로 상권에 적용

### 사업 구조
1. **글로컬 상권 활성화** — 수원시 위탁, 행궁동 상권 기획 운영
2. **슈퍼파트너** — 소상공인 식자재 공동구매 + 경영 효율화 플랫폼 (개발 중)
3. **신도시 JV** — 로컬 기업 투자/육성 조인트벤처
4. **AI 자동화** — 소상공인 맞춤형 자동화 솔루션

### 핵심 차별점
- 다른 상권 기획사: 정부 예산 집행하는 용역사
- 공존공간: 직접 투자하고, 기업을 키우고, 데이터를 자산화하는 "상권 액셀러레이터"

## 4. 진행 중인 핵심 프로젝트

### 슈퍼파트너 (식자재 공동구매 플랫폼)
- Phase 1: 유통 (공동구매) → 유저 확보 + 데이터 수집
- Phase 2: 경영 효율화 SaaS → 데이터 판매 수익
- Phase 3: 피지컬 AI → 제조업 기술 데이터 자산화
- 수익 모델: 유통 수수료 약 3% + 데이터 판매 (SaaS 구독 X, 완전 무료 보급)
- 목표: 2027년 1분기까지 1,000개소

### 신도시 JV
- CSP + 개인 주주 → 조인트벤처 설립
- 엑셀러레이터 등록 목표 (자본금 1.5억 이상)
- 지분: 개인 60% 이상, 법인 40% 이하

### 금융 혁신 (립스 3)
- 기존 매출 채권 기반 대출의 한계 극복
- 기업 가치 + 투자 실적 기반 보증 상품
- 로컬 기업의 성장 자본 확충 목적

## 5. 문서 종류별 톤 & 양식 가이드

### 기본 원칙 (모든 문서 공통)
- 자신감 있되 허세 없는 톤
- 숫자와 실적으로 증명하는 스타일
- "우리가 이미 했다" → "그래서 다음은 이것이다" 흐름
- 뜬구름 잡는 비전만 나열 금지
- "~할 예정입니다"만 반복 금지 (반드시 근거 데이터 동반)
- 과도한 전문 용어 금지 (읽는 사람이 이해 가능해야)

---

### A. 정부 제안서 (기본 양식)

**읽는 사람**: 공무원, 심사위원, 정책 담당자
**톤**: 공식적 + 객관적 + 정책 정합성 강조

| 항목 | 규칙 |
|---|---|
| 문체 | "~하고자 합니다", "~에 기여하고자 함" — 경어체, 공문서 어투 |
| 구조 | 사업 목적 → 추진 배경 → 사업 내용 → 추진 체계 → 기대 효과 → 예산 |
| 핵심 | 정책 목표와의 연결 ("정부 ○○ 정책에 부합"), 정량 성과 지표 필수 |
| 근거 | 정부 통계, 공공 데이터, 선행 사업 성과 인용 |
| 피해야 할 것 | 감정적 표현, 과장된 수식어, 민간 업계 용어 남발 |

**자주 쓰는 표현**:
- "본 사업은 ○○정책 기조에 부합하여~"
- "기존 ○○사업 성과를 바탕으로~"
- "소상공인의 경영 안정 및 자생력 강화를 위해~"
- "민관 협력 체계를 통한~"

---

### B. IR / 투자제안서

**읽는 사람**: 투자자, VC, 엔젤
**톤**: 간결 + 숫자 중심 + 스토리 + 임팩트

| 항목 | 규칙 |
|---|---|
| 문체 | "~합니다" — 깔끔한 현재형. 짧은 문장. 한 슬라이드에 한 메시지 |
| 구조 | 문제 → 솔루션 → 시장 규모(TAM/SAM/SOM) → 트랙션 → 비즈니스 모델 → 팀 → Ask |
| 핵심 | 성장 가능성, 시장 크기, 유닛 이코노믹스, 경쟁 우위 |
| 근거 | 시장 리서치, 벤치마크 기업 비교, 자체 트랙션 데이터 |
| 피해야 할 것 | 정부 공문서 어투, 장문의 설명, "~하고자 함" |

**자주 쓰는 표현**:
- "64조 시장, 80%가 디지털화되지 않음"
- "식봄 2,341억 거래액 — 시장은 이미 검증됨"
- "우리의 차별점: 커뮤니티 기반 락인"
- "Phase 1 → 2 → 3 로드맵"

---

### C. 사업 소개서 / 파트너 제안

**읽는 사람**: 협력사, 파트너, 소상공인, 지자체 실무자
**톤**: 현장감 + 신뢰 + 쉬운 말

| 항목 | 규칙 |
|---|---|
| 문체 | "~하고 있습니다", "~했습니다" — 친근하되 프로페셔널. 전문 용어 최소화 |
| 구조 | 우리가 누구인지 → 뭘 했는지(성과) → 뭘 하려는지 → 같이 하면 뭐가 좋은지 |
| 핵심 | 실적 중심("이미 해봤다"), 파트너에게 돌아가는 혜택 명확히 |
| 근거 | 자체 성과 데이터, 현장 사례, 고객 반응 |
| 피해야 할 것 | 정부 공문서 어투, 투자 전문 용어(TAM, 유닛 이코노믹스 등) |

**자주 쓰는 표현**:
- "행궁동에서 2년간 직접 운영하며 검증했습니다"
- "소상공인 142곳과 함께 만들었습니다"
- "사장님이 할 일은 딱 하나, 주문만 넣으시면 됩니다"
