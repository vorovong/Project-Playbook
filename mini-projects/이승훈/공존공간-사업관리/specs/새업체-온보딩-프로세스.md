# 새 업체 온보딩 프로세스

새 업체를 월정산 시스템에 추가할 때 따라가는 표준 프로세스.
`businesses/<업체명>/` 폴더에 config·collectors·sheets 3개 파일을 만들면 `monthly-report.js`가 자동으로 인식한다.

---

## Phase 1: 업체 파악 (대화)

새 업체가 정해지면 아래 항목을 대화로 파악한다. 모르는 건 `[미결정]`으로 남기고 추측하지 않는다.

### 필수 파악 항목

| # | 항목 | 질문 | 예시 |
|---|---|---|---|
| 1 | 업체명 | 정확한 상호명 | 미식가의주방, 포토인더박스 |
| 2 | 사업 유형 | 뭐 하는 곳? | 음식점, 포토부스, 워크샵 |
| 3 | 정산 유형 | 어떻게 돈이 오가는지? | 아래 유형 분류표 참고 |
| 4 | 정산 주기 | 얼마나 자주 정산? | 주간, 월간 |
| 5 | 데이터 소스 | 매출 데이터가 어디에 있는지? | OKPOS, 엑셀, 수기장부, PG사 |
| 6 | 리포트 항목 | 월보고서에 뭐가 들어가야 하는지? | 매출, 공과금, 인건비, 재고 등 |
| 7 | 노션 위치 | 월정산 페이지가 어디에? | parentPageId |

### 정산 유형 분류

| 유형 | 구조 | 핵심 계산 | 기존 예시 |
|---|---|---|---|
| **수수료형** | 매출 × 수수료율 = 공존 수입 | 카드/현금 수수료 + 부가세 | 미식가의주방 (카드15%, 현금13%) |
| **매출-지출형** | 총매출 - 운영비 = 순매출 | 운영수수료, 임대료, 재고 차감 | 포토인더박스 (운영수수료15%) |
| **클래스형** | 건당 매출 - 원가 = 이익 | 재료비, 강사비, 플랫폼 수수료 | 재생전술 |
| **월세형** | 고정 월세 + 관리비 + 공과금 | 수금 여부 관리 | (아직 없음) |
| **복합형** | 위 유형 조합 | 유형별로 분리 계산 | 공존공간 = 수수료+매출지출+클래스 |

## Phase 2: 노션 DB 세팅

### 2-1. 필요한 DB 결정

정산 유형별로 필요한 DB가 다르다:

| 정산 유형 | 필요 DB |
|---|---|
| 수수료형 | 매출 DB (일별) |
| 매출-지출형 | 매출 DB + 재고 DB |
| 클래스형 | 클래스 DB (건별) |
| 월세형 | 수금 DB (월별) |
| 공통 | 공과금 DB, 구매내역 DB (필요시) |

### 2-2. DB 프로퍼티 설계

기존 gongzon DB 구조를 참고해서, 해당 업체에 맞는 프로퍼티를 정한다.
- 타이틀(제목) 필드: 보통 날짜 또는 월
- 숫자 필드: 금액, 수량 등
- 셀렉트 필드: 종류, 구분, 채널 등
- 날짜 필드: 기간 필터용

### 2-3. .env 업데이트

```
# 새업체
NOTION_새업체_SALES_DB=xxx
NOTION_새업체_UTILITY_DB=xxx
```

### 2-4. 데이터 입력 방식

| 방식 | 언제 | 설명 |
|---|---|---|
| 수동 입력 | 첫 단계 | 노션에 직접 입력 (가장 빠른 시작) |
| 엑셀 → 노션 | 기존 엑셀이 있을 때 | 스크립트로 변환 |
| POS 연동 | 자동화 단계 | okpos-sales.js 같은 수집기 |
| 메일 파싱 | 고지서 등 | scan-mail.js 같은 파서 |

## Phase 3: config.js 작성

`businesses/gongzon/config.js`를 복사해서 시작한다.

```javascript
// businesses/새업체/config.js
module.exports = {
  name: '업체명',
  parentPageId: 'xxx', // 노션 월정산 페이지 ID

  db: {
    sales: process.env.NOTION_새업체_SALES_DB,
    // 필요한 DB만 추가
  },

  // 업체별 설정값 (정산 유형에 따라 다름)
  // 수수료형: 수수료율
  // 매출-지출형: 운영수수료율, 임대료
  // 클래스형: (config 없이 DB에서 가져옴)
  // 월세형: 월세, 관리비
};
```

**체크포인트:** `node -e "require('./businesses/새업체/config')"`

## Phase 4: collectors.js 작성

`businesses/gongzon/collectors.js`를 참고한다.

```javascript
// businesses/새업체/collectors.js
const { queryAll } = require('../../lib/notion');

async function fetch매출(config, month, range) {
  // 노션 DB에서 해당 월 데이터 조회
  // queryAll(config.db.sales, { filter })
  // 정산 유형에 맞게 계산
  return { /* 계산된 데이터 */ };
}

module.exports = {
  collectors: [
    { key: 'sales', label: '매출', fn: fetch매출, needsRange: true },
    // 필요한 수집기 추가
  ],
};
```

### 수집기 작성 팁

- `queryAll`은 페이지네이션 자동 처리 (100개 이상도 OK)
- `needsRange: true`면 fn(config, month, range)로 호출됨
- `needsRange` 없으면 fn(config, month)로 호출됨
- range 객체: `{ start, end, weeks: [{ num, start, end }] }`

**체크포인트:** `node -e "require('./businesses/새업체/collectors')"`

```javascript
  ],

```

### 블록 빌더 사용법 (lib/blocks.js)

| 함수 | 용도 | 예시 |
|---|---|---|
| `heading2(text)` | 섹션 제목 | `heading2('매출 요약')` |
| `heading3(text)` | 소제목 | `heading3('1주차')` |
| `table(rows, colCount)` | 표 | `table([row(['항목','금액'])], 2)` |
| `row(cells)` | 표 행 | `row(['매출', fmt(1000000)])` |
| `fmt(number)` | 숫자 포맷 | `fmt(1500000)` → `'1,500,000'` |
| `text(content)` | 텍스트 | `text('데이터 없음')` |
| `divider()` | 구분선 | |

**체크포인트:** `node -e "require('./businesses/새업체/sheets')"`

## Phase 6: 테스트 & 검증

### 6-1. 모듈 로드 확인

```bash
node -e "require('./businesses/새업체/config')"
node -e "require('./businesses/새업체/collectors')"
node -e "require('./businesses/새업체/sheets')"
```

3개 모두 에러 없으면 통과.

### 6-2. 실행

```bash
node monthly-report.js 새업체 2026-03
```

### 6-3. 결과 확인

- [ ] 노션에 메인 페이지(월 결산) 생성됨
- [ ] 서브페이지들 생성됨
- [ ] 숫자가 원본 데이터와 일치함
- [ ] 표 레이아웃이 깨지지 않음
