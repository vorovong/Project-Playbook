# 소상공인 매출 보전 보험 — 심층 분석

조사일: 2026-04-06

---

## 요약 (핵심 발견 5가지)

1. **해외에는 이미 존재한다** — 파라메트릭(Parametric) 보험이라는 이름으로 미국, 영국, 유럽에서 소상공인 대상 상품이 빠르게 확산 중이다. 글로벌 시장 규모 $21B(2025) → $24B(2026), 연 13% 성장률.

2. **한국에는 상용화된 상품이 없다** — 2026년 4월 현재, 한국에 소상공인 매출 감소를 직접 보상하는 보험 상품은 존재하지 않는다. 풍수해보험(재물 손해)과 자영업자 고용보험(폐업 시 실업급여)이 가장 가까운 대체재다.

3. **기술 인프라는 이미 갖춰져 있다** — 캐시노트(한국신용데이터)가 130만+ 사업장의 실시간 매출 데이터를 보유하고 있으며, 이미 29개 금융사와 연동 중이다. 보험 설계에 필요한 데이터 파이프라인이 사실상 완성되어 있다.

4. **핵심 걸림돌은 규제와 리스크 모델 부재다** — 한국 보험업법은 "손해 발생 → 손해 보상" 구조를 요구하며, 파라메트릭(지수 기반 자동 지급) 방식을 보험으로 인정하는 법적 근거가 불명확하다. 일본은 1999년에 이미 보험사의 파생상품 취급을 허용했다.

5. **코로나 손실보전금(23조원)이 수요를 증명했다** — 정부가 371만 소상공인에게 매출 감소 기준으로 600~1,000만원을 지급한 전례가 있다. 이것은 사실상 "정부가 보험자 역할을 한 것"이며, 민간 보험 상품화의 가능성과 수요를 동시에 증명한다.

## 1. 해외 사례

### 1-1. 파라메트릭 보험이란?

전통 보험이 "실제 손해액을 산정해서 보상"하는 구조라면, 파라메트릭 보험은 **사전에 합의한 객관적 지표(트리거)가 충족되면 약정 금액을 자동 지급**하는 구조다.

- 손해 사정 불필요 → 수일 내 지급 (전통 보험은 수주~수개월)
- 트리거 예시: 풍속 X km/h 초과, 수위 X cm 초과, 기온 X도 이하
- 간접 손실(매출 감소, 영업 중단)도 보상 가능

### 1-3. 소상공인/요식업 관련 사례

- **Lockton (글로벌 보험중개사)**: 소매업 대상 파라메트릭 솔루션 제공. "과도한 강우나 이상 난동으로 매출이 감소하면 사전 합의 조건에 따라 보상"하는 구조를 소개
- **InsTech 분석**: "호텔 예약률의 전년 동기 대비 변화" 또는 "보행자 통행량(footfall)" 같은 지표를 트리거로 사용하는 모델이 등장
- **Business Continuity Indicator**: 특정 지역/업종의 "예상 소비액 vs 실제 소비액"을 비교해 자동 보상하는 지수가 개발 중
- **NEXT Insurance 조사**: 미국 레스토랑 오너의 40% 가까이가 극한 기상에 대한 보험 미가입 상태

### 1-4. 보험료 수준 [추정 포함]

- FloodFlash 기준: 보장액 대비 약 **3.2%~10.9%** (트리거 조건에 따라 변동)
- 일반적 파라메트릭 보험: 전통 보험 대비 **20~50% 저렴**하다는 분석이 있으나, 보상 범위가 다르므로 단순 비교는 어려움
- [추정] 소상공인 매출 보전 보험의 경우 월 매출의 **1~3%** 수준의 보험료가 시장 수용 가능 범위로 보임

## 2. 한국 현황

### 2-1. 현재 한국에서 가장 가까운 상품들

| 상품/제도 | 운영 주체 | 보상 대상 | 한계 |
|-----------|-----------|-----------|------|
| **풍수해·지진재해보험** | 정부(중소벤처기업부) + 7개 민영보험사 | 태풍·홍수·대설 등에 의한 재물 손해 | **매출 감소는 보상 불가**. 물리적 피해만 대상 |
| **자영업자 고용보험** | 정부(고용노동부) | 비자발적 폐업 시 실업급여 | **폐업해야만** 보상. 영업 중 매출 감소는 해당 없음 |
| **소상공인 손실보전금** (2022) | 정부 재정 | 코로나 방역 조치로 인한 매출 감소 | **일회성 재정 지출**. 23조원 투입, 371만명 대상 |
| **날씨보험** | 기상청 + 보험업계 협의체 | 강수량·기온 기반 행사 취소 등 | 야외 행사·스키장·워터파크 등 **한정된 업종** |
| **농업재해보험** | 정부(농림부) + 농협손보 등 | 자연재해로 인한 농작물 피해 | 농업 전용. 일반 소상공인 해당 없음 |

### 2-2. 보험사/인슈어테크 동향

- **한국공제보험신문 전문가 좌담 (2024)**: "한국에는 아직 상용화된 파라메트릭 보험 상품이 없다"고 명시. 규제 문제가 핵심 장벽.
- **기상청 협의체**: 2018년부터 보험업계·기상업계 참여 협의체 운영. 날씨보험 개발 추진 중이나, 소상공인 매출 보전까지는 확장되지 않음.
- **캐시노트(한국신용데이터)**: 2023년 롯데손해보험과 보험 판매 MOU, 리치앤코와 소상공인 보험 인프라 MOU 체결. 그러나 매출 데이터 기반 "파라메트릭 보험 상품 개발"까지는 진행되지 않은 것으로 보임.
- **삼성화재**: 캐시노트에 투자자로 참여 (1,600억원 누적 투자 라운드 중). [추정] 매출 데이터 활용 보험 상품에 관심이 있을 가능성.

### 2-3. 규제 환경

- **보험업법의 벽**: 한국 보험업법은 "실제 손해 발생 → 손해액 산정 → 보상"의 실손보상 원칙이 기본. 파라메트릭 방식(지표 충족 시 정액 지급)이 "보험"으로 인정되는지 법적 해석이 불분명.
- **일본과의 차이**: 일본은 1999년부터 보험사가 파생상품(날씨 파생 등)을 취급할 수 있도록 규제 완화. 이것이 파라메트릭 보험 발전의 기반이 됨.
- **금융규제 샌드박스**: 2026년부터 '기획형 규제 샌드박스' 도입 예정. 정부가 먼저 규제 합리화 방안을 기획한 뒤 사업자를 모집하는 방식. 파라메트릭 보험이 이 경로로 진입할 수 있는 가능성 존재.
- **인슈어테크 라이선스 문제**: 보험 비교·추천 서비스 제공 시 보험대리점업 라이선스가 필요. 중소 인슈어테크 기업들이 규제 장벽에 직면 중.

### 2-4. 코로나 손실보전금이 남긴 교훈

2022년 정부가 실행한 소상공인 손실보전금은 사실상 **정부가 보험자 역할을 한 것**이다:
- 트리거: 매출 감소율 (전년 대비)
- 보상: 매출 규모·감소율에 따라 9개 구간, 600만~1,000만원
- 대상: 371만 소상공인
- 총 규모: 23조원

이 경험은 두 가지를 증명한다:
1. **수요가 있다** — 매출 감소에 대한 보상 니즈가 압도적
2. **재정으로는 지속 불가능** — 23조원을 세금으로 부담하는 것은 반복 불가능한 구조

## 3. 기술적 가능성

### 3-1. 데이터 인프라 현황

| 플랫폼 | 데이터 유형 | 사업장 수 | 실시간성 | 보험 연계 가능성 |
|---------|-------------|-----------|----------|------------------|
| **캐시노트(KCD)** | 카드 매출, POS 매출, 결제 건수, 시간대별·업종별 분석 | 130만+ | 거의 실시간 | **매우 높음** — 이미 29개 금융사 연동, 신용평가사(한국평가정보) 운영 |
| **토스 (비바리퍼블리카)** | 계좌 거래, 카드 매출, 소비 패턴 | [추정] 수십만 사업장 | 실시간 | 높음 — 금융 슈퍼앱으로 보험 판매 채널 보유 |
| **배민/요기요** | 배달 주문·매출 | 해당 플랫폼 입점 업체 | 실시간 | 중간 — 배달 업종에 한정 |
| **카드사(BC, 삼성, 신한 등)** | 전체 카드 결제 | 전 가맹점 | T+1~2일 | 높음 — 가장 광범위한 매출 데이터 |
| **국세청 현금영수증** | 현금 매출 포함 전체 매출 | 전 사업자 | T+1일 | 중간 — 정부 데이터라 민간 활용에 제약 |

### 3-2. POS 데이터 기반 자동 심사/보상 구조

```
[1] 가입 단계
    사업장 매출 데이터 연동 (캐시노트/POS API)
    → 업종별 매출 패턴 분석 (계절성, 요일별 편차, 성장률)
    → 보험료 자동 산출 (리스크 모델 기반)
    → 보장 범위·트리거 합의

[2] 모니터링 단계
    매일/매주 매출 데이터 자동 수집
    → 기준 매출 대비 실적 추적
    → 이상 신호 감지 (급격한 매출 하락)

[3] 보상 단계
    트리거 조건 충족 (예: 전년 동기 대비 30% 이상 매출 감소가 2주 연속)
    → 외부 요인 확인 (날씨 데이터, 정부 방역 조치, 지역 재난 등)
    → 자동 보험금 산출 및 지급 (수일 내)
```

### 3-3. 필요한 데이터와 리스크 모델

**보험 설계에 필요한 데이터:**
- 업종별 월별 매출 분포 (최소 3년치)
- 매출 변동의 원인 분해: 계절성 / 경기 / 외부 이벤트(재난·방역)
- 폐업률, 업종 전환률
- 지역별 소비 패턴 변화

**리스크 모델 핵심 요소:**
- 손실 빈도(frequency): 특정 업종에서 매출 X% 이상 감소가 연 몇 회 발생하는가
- 손실 심도(severity): 감소 발생 시 평균 감소폭과 지속 기간
- 상관관계(correlation): 동일 지역·업종 내 동시 발생 확률 (대규모 재난 시 전체가 영향)
- 기저 위험(basis risk): 트리거 지표와 실제 손실 간 괴리

**캐시노트가 이 모델에 줄 수 있는 것:**
캐시노트는 130만 사업장의 수년간 매출 데이터를 보유하고 있다. 이것은 업종별·지역별·시간대별 매출 분포를 구축하기에 **세계적으로도 드문 수준의 데이터셋**이다. [추정] 이 데이터만으로도 상당히 정교한 리스크 모델 구축이 가능할 것이다.

## 4. 사업 모델 초안

### 4-1. 보험자 구조 (3가지 시나리오)

| 시나리오 | 보험자 | 장점 | 단점 |
|----------|--------|------|------|
| **A. 기존 보험사 + 플랫폼** | 삼성화재/DB손보 등이 인수, 캐시노트/토스가 판매·데이터 | 라이선스 문제 해결, 재보험 접근 용이 | 보험사의 보수적 문화, 상품 개발 속도 느림 |
| **B. 정부 주도 공영보험** | 풍수해보험처럼 정부가 보험료 일부 지원 | 역선택 최소화(전 업종 대상), 사회적 안전망 | 재정 부담, 정치적 변수, 모럴 해저드 |
| **C. 인슈어테크 MGA** | 인슈어테크가 상품 설계·판매, 보험사가 리스크 인수 | 빠른 혁신, 데이터 활용 극대화 | 규제 샌드박스 필요, 초기 자본 조달 어려움 |

**[추정] 가장 현실적 경로: A+C 하이브리드** — 캐시노트 같은 플랫폼이 MGA 역할을 하고, 기존 보험사(삼성화재 등 이미 투자자)가 리스크를 인수하는 구조. 규제 샌드박스를 활용해 시범 운영 후 확대.

### 4-2. 보험료 구조

```
[기본 설계]
- 보험료 = 월 매출의 1~3% (업종·지역·리스크 등급에 따라 차등)
- 예시: 월 매출 3,000만원 음식점 → 월 보험료 30~90만원
  (연 360~1,080만원)

[보장 내용]
- 트리거 충족 시 월 매출의 50~80%를 최대 3개월간 보전
- 예시: 월 3,000만원 → 월 1,500~2,400만원 × 3개월 = 최대 7,200만원

[보험료 할인 요인]
- 캐시노트 연동 (데이터 투명성) → 10~20% 할인
- 다년 계약 → 5~10% 할인
- 복수 사업장 → 단체 할인
```

### 4-3. 보상 트리거 설계 (3가지 유형)

| 트리거 유형 | 조건 예시 | 장점 | 단점 |
|-------------|-----------|------|------|
| **외부 지표형** | 기상청 폭우 경보 발령 + 해당 지역 | 객관적, 조작 불가 | 매출 감소와 직접 연결이 약할 수 있음 (기저 위험) |
| **매출 데이터형** | 전년 동기 대비 매출 30%+ 감소가 2주 연속 | 실제 손실과 직접 연동 | 모럴 해저드 우려 (의도적 매출 축소) |
| **복합형 (권장)** | 외부 이벤트 발생 **+** 매출 데이터상 감소 확인 | 두 조건 동시 충족으로 정확도 높음 | 설계 복잡, 보상 지연 가능성 |

**[추정] 복합형이 가장 실현 가능** — "정부 재난 선포 또는 기상 이벤트 발생" + "캐시노트 데이터상 매출 X% 감소 확인" 두 조건을 모두 충족해야 보상.

### 4-4. 모럴 해저드(도덕적 해이) 방지

| 위험 | 대응 방안 |
|------|-----------|
| 의도적 영업 축소 | POS 데이터로 영업시간·주문건수 모니터링. 영업시간 단축 시 보상 제외 |
| 매출 과소 보고 | 카드 매출 + 현금영수증 교차 검증. 캐시노트 연동 의무화 |
| 고위험 사업장만 가입 | 업종별 풀링(pooling), 단체 가입 유도, 의무가입 구간 설정 |
| 가입 후 사업장 관리 소홀 | 보상한도를 매출의 일부(50~80%)로 제한하여 자기부담 유지 |
| 허위 사업장 등록 | 사업자등록증 + 실제 결제 데이터 3개월 이상 확인 후 가입 허용 |

### 4-5. 타겟 고객 (우선순위)

| 순위 | 업종 | 이유 |
|------|------|------|
| 1 | **음식점 (외식업)** | 날씨·계절·재난에 매출 민감도 최고. 코로나 시 최대 피해 업종. 전국 70만+ 사업장 |
| 2 | **카페·베이커리** | 음식점과 유사한 리스크 프로필. 유동인구 의존도 높음 |
| 3 | **소매업 (의류·잡화)** | 경기 변동·계절·날씨 민감. 온라인 전환 리스크 추가 |
| 4 | **숙박업 (모텔·펜션)** | 관광 수요 변동, 자연재해 직접 영향 |
| 5 | **학원·교육서비스** | 전염병 시 직접 영업 제한 대상 |

## 5. 리스크/걸림돌

### 5-1. 왜 아직 아무도 안 만들었는가?

| 이유 | 상세 |
|------|------|
| **규제 불확실성** | 파라메트릭 보험이 한국 보험업법상 "보험"인지 "파생상품"인지 불명확. 보험으로 하려면 실손보상 원칙과 충돌, 파생상품으로 하려면 다른 라이선스 필요 |
| **리스크 모델 부재** | 소상공인 매출 데이터 기반의 보험 수리 모델(actuarial model)을 구축한 곳이 없음. 캐시노트 데이터는 존재하나 보험 목적의 분석은 미수행 |
| **상관 리스크(correlated risk)** | 코로나·경기침체 같은 시스템 리스크 발생 시 전체 가입자가 동시에 청구 → 보험사 지급 능력 초과 위험. 재보험 확보 필수 |
| **기저 위험(basis risk)** | 외부 지표 트리거 사용 시, 트리거는 충족되었으나 실제 매출 감소가 없는 경우(또는 그 반대) 발생 |
| **시장 규모 불확실** | 소상공인이 월 매출의 1~3%를 보험료로 낼 의사가 있는지 미검증. 마진율이 낮은 업종에서는 추가 비용 부담이 클 수 있음 |
| **보험사의 보수적 문화** | 한국 보험사들이 익숙하지 않은 상품 구조에 대한 투자 의사결정이 느림 |

### 5-2. 보험업 라이선스 문제

- 보험상품 판매: 보험업 허가 또는 보험대리점 등록 필요
- 파라메트릭이 "보험"이 아닌 "파생상품"으로 분류될 경우: 자본시장법 적용, 금융투자업 라이선스 필요
- **현실적 해결책**: 기존 보험사(라이선스 보유)와 협력하여 MGA(Managing General Agent) 구조로 진입 → 규제 샌드박스로 시범 운영

### 5-3. 역선택(Adverse Selection) 문제

- 매출이 이미 하락 추세인 사업장만 가입하려는 경향
- **대응**: 가입 시 최근 6~12개월 매출 데이터 제출 의무화 + 가입 후 대기 기간(예: 3개월) 설정 + 업종별 풀링으로 리스크 분산

### 5-4. 데이터 신뢰성 문제

- 현금 매출 누락 가능성 (카드 매출만으로는 전체 매출 파악 불완전)
- POS 데이터 조작 가능성 (비정상 할인, 취소 처리 등)
- **대응**: 카드 매출 + POS + 현금영수증 교차 검증, 이상 거래 탐지 알고리즘 적용

### 5-5. 거시 리스크

- 팬데믹·경기침체 같은 시스템 리스크 발생 시 보험금 청구가 폭발적으로 증가
- 일반 보험의 "대수의 법칙"이 작동하지 않음 (개별 사업장의 매출 감소가 독립적이지 않음)
- **대응**: 재보험 의무화 + 정부 재보험 풀 + 보상 한도 설정 + catastrophe bond 발행

## 6. 결론: 기회인가, 함정인가?

### 기회인 이유

1. **검증된 수요**: 코로나 손실보전금 23조원은 정부가 "이 보험이 있었으면 좋겠다"를 스스로 증명한 것이다.
2. **데이터 인프라 완비**: 캐시노트 130만+ 사업장 데이터는 세계적으로도 드문 소상공인 매출 데이터베이스다. 보험 설계의 원재료가 이미 있다.
3. **글로벌 트렌드 합류**: 파라메트릭 보험 시장은 연 13% 성장 중이며, 소상공인으로 확장하는 것이 글로벌 방향이다.
4. **경쟁 부재**: 한국에 이 상품이 없다는 것은 선점 기회를 의미한다.
5. **정책 환경 개선**: 기획형 규제 샌드박스 도입으로 혁신 상품 시험 가능성이 열리고 있다.

### 함정인 이유

1. **규제 돌파에 1~3년 소요** [추정]: 보험업법 해석, 샌드박스 신청, 시범 운영까지 빠르지 않다.
2. **시스템 리스크 해결 필수**: 코로나급 이벤트 시 보험사가 감당할 수 없다. 정부 재보험 없이는 상품 자체가 성립하지 않을 수 있다.
3. **소상공인의 보험료 지불 능력**: 이미 마진이 낮은 소상공인에게 매출의 1~3%는 부담이다. 정부 보험료 지원(풍수해보험처럼) 없이는 가입률이 낮을 수 있다.
4. **모럴 해저드 완전 방지 불가**: 매출 데이터 기반이므로 조작 유인이 존재한다.

### 최종 판단

**기회이되, 단독으로는 어렵다.**

가장 현실적인 경로:
```
캐시노트(데이터+판매) + 기존 손보사(리스크 인수+라이선스) + 정부(보험료 일부 지원+재보험 풀)
→ 규제 샌드박스로 음식점 업종 시범 운영
→ 성과 검증 후 업종 확대
```

이 3자 협력 모델이 성립하면, 한국은 세계 최초로 **실시간 매출 데이터 기반 소상공인 파라메트릭 보험**을 만들 수 있다. 캐시노트의 데이터 밀도는 해외 어디에도 없는 수준이기 때문이다.

첫 단계로 필요한 것: **캐시노트 데이터를 활용한 업종별 매출 변동 리스크 분석 보고서** — 이것이 보험 수리적 타당성을 증명하는 출발점이다.
