# 슈파 핵심 문제 정의 (v5, 확정)

> 작성일: 2026-04-07
> 방법: 이승훈 ↔ 김비서 탐색 대화 (Q1~Q11)
> 목적: 슈파가 풀 핵심 문제 정의 — "소상공인에게 처음 건네는 한마디"

---

## 1. 핵심 문제 정의 (확정)

> **"소상공인은 데이터도 있고 소스도 알려져 있다. 그런데 '그래서 우리 가게는 누구에게 뭘 팔아야 하는가?'라는 전략적 답을 내주는 사람이 없다. 사장님이 혼자서 5개 이상의 빅데이터를 뒤져 결론을 낼 수는 없다."**

---

## 2. 가설의 진화 과정 (4단계)

대화를 진행하면서 문제 정의가 4번 진화했다. 각 단계는 이전 단계의 "표면 증상"을 넘어 한 단계 더 깊은 원인을 드러냈다.

| 버전 | 가설 | 한계 / 반박 |
|---|---|---|
| v1 | "측정이 안 된다" | 너무 막연. 뭘 측정하려는지 불분명 |
| v2 | "데이터가 흩어져 있어 측정 불가" | 합치는 도구는 이미 있음 (캐시노트, 토스플레이스 등) |
| v3 | "목적형 약속상권은 약속 결정자를 식별 못한다" | 식별해도 "왜 이 사람이 오는지"는 여전히 모름 |
| v4 | "누구를 타겟해야 할지부터 모른다" | 타겟 정의 → 측정 → 액션 순서가 필요 |
| **v5 (확정)** | **"전략적 답을 내주는 사람이 없다"** | — |

---

## 3. 대화에서 나온 핵심 발견

### 3-1. 미식가의주방 사장님의 3대 고통 (이승훈 관찰)
1. 돈이 안 남는다
2. 사람 구하기 힘들다
3. **손님이 안 온다** ← 이것이 뿌리 ("풀리면 나머지도 풀린다")

### 3-2. "손님이 안 온다"의 근본 원인
네이버 플레이스 광고·인스타 피드 등 모든 유입 수단이 **"광고비 싸움 = 순위 경쟁"**으로 귀결되며, 소상공인은 대기업 자본력에 밀림. **원천적 해결책 필요.**

### 3-3. 데이터는 이미 있다 (흩어져 있을 뿐)
미식가의주방이 이미 보유한 데이터:
| 데이터 | 소스 |
|---|---|
| 고객 전화번호 | 네이버 예약 |
| 방문/노쇼 이력 | 네이버 예약 |
| 검색 키워드·유입 | 네이버 플레이스 광고 |
| 매출 (카드/현금/메뉴) | OKPOS |

→ **합쳐서 보는 사람이 없을 뿐, 데이터 자체는 존재.**

### 3-4. 상권 유형에 따라 핵심 지표가 다르다 (이승훈의 중요한 지적)
재방문율 만능설의 반박. 미식가·대봉 모두 **"목적형 약속상권 (행궁동)"**.

| 상권 유형 | 핵심 지표 | 재방문율 의미 |
|---|---|---|
| 동네/주거지 | 재방문율, 단골 | 매우 중요 |
| 오피스 점심 | 회전율, 직장인 단골 | 중요 |
| 역세권/유동 | 신규 유입, 가시성 | 약함 |
| 관광지 | 리뷰, 검색 노출, 객단가 | 거의 무관 |
| **목적형 약속상권** | **약속 결정자, 추천 충성도, 그룹 사이즈** | **분기 단위로 봐야 의미 있음** |

### 3-5. 목적형 약속상권의 특이성
- 손님 = 모임의 **"약속 결정자 한 명"**이 N명(5~10명)을 데려옴
- "단골"의 정의가 다름: 자주 오는 사람 ≠ 약속이 생길 때마다 우리 가게를 고르는 사람
- 월 단위 재방문율로는 단골조차 0%로 잡힘 → 기존 CRM 도구 무용지물
- 일반 CRM의 "30일 이탈" 정의가 목적형 상권에선 오작동

### 3-6. 진짜 틈새 — "오는 사람 다 받는다" 상태
- 이승훈 답변: 미식가·대봉 모두 **명확한 타겟 없음**
- 타겟이 없으니 메뉴·가격·서비스·마케팅이 전부 "모든 사람용"
- 결국 어떤 것도 특별하지 않음 → 광고비 싸움으로 회귀

### 3-7. 데이터 소스는 이미 접근 가능 (이승훈 직접 경험)
이승훈이 직접 사용해본 5개 소스 — 모두 쓸만함:
1. KT 빅데이터 / SKT 데이터스퀘어 (유동인구, 거주/직장 인구, 시간대)
2. 신한카드 빅데이터 (카드 결제 기반 소비자 분석)
3. 소상공인시장진흥공단 상권정보시스템 (업종별 매출, 경쟁점)
4. 오픈업 (KB국민카드) (상권 분석, 경쟁 강도)
5. 네이버 데이터랩 (키워드 검색 트렌드, 지역 검색)

→ **접근성은 문제가 아니다. 해석·통합·전략화가 문제다.**

## 4. 슈파의 포지셔닝 (경쟁사 대비)

| 카테고리 | 대표 | 제공 가치 | 한계 |
|---|---|---|---|
| 운영 도구 | Square, Toast | 돈·사람·재고를 잘 굴리게 함 | "누구를 타겟하라"는 답 없음 |
| 대시보드 | 캐시노트 | 숫자를 보여줌 | 숫자 → 전략으로 넘어가는 부분 공백 |
| 컨설팅 | 개별 컨설턴트 | 전략적 답 제공 | 비쌈 + 일회성 + 실행 연계 안 됨 |
| **슈파** | — | **숫자 → 전략 → 실행을 AI가 연결** | — |

**한국 시장에서 이 포지션은 비어 있음.** (캐시노트가 일부 상권 리포트를 제공하지만 "당신 가게는 누굴 타겟하세요"까지는 가지 않음.)

## 5. 슈파의 핵심 기능 흐름 (가설)

```
[외부 빅데이터 5종]      [내부 데이터]
 KT/SKT 유동인구          네이버 예약 (전화번호, 방문이력)
 신한카드 소비자          OKPOS (카드/현금/메뉴 매출)
 소진공 상권정보          네이버 플레이스 광고 (유입 키워드)
 오픈업 경쟁강도
 네이버 데이터랩 검색
        ↓                      ↓
        └──────── 통합 ────────┘
                  ↓
        [AI 전략 참모 레이어]
                  ↓
        "이 상권에 오는 OOO 세그먼트를 타겟으로 잡으세요.
         근거: XYZ. 메뉴는 A, 가격대는 B, 마케팅은 C."
                  ↓
        [실행 액션 제안]
        - 메뉴 조정
        - 타겟 광고 방향
        - 응대/서비스 가이드
        - 결과 추적 → 타겟 재조정
```

### 슈파의 차별화된 가치 제안 (한 줄)

> **"당신이 누구를 손님으로 삼을지, 데이터로 알려드립니다."**

### Square "제품 4개 락인" 공식과의 연결
한 번 이 전략을 받으면 메뉴·마케팅·응대·가격이 전부 그 타겟에 정렬됨 → 슈파 없이는 되돌아갈 수 없음 → 구조적 락인.

## 6. 테스트베드 & 첫 타겟

| | 상권 | 상태 |
|---|---|---|
| **테스트베드 (현재)** | 미식가의주방 (공존공간, 행궁동) | 네이버 예약 + OKPOS 데이터 존재, 직접 관찰 가능 |
| **확장 1호 (미래)** | 대봉통닭 (목적형 약속상권, 유사) | 동일한 상권 유형 → 검증 타당 |

**행궁동 상권의 이질성 (슈파가 풀 구체적 문제):**
- 화성 관광객 (일회성, 외지)
- 데이트/모임 (수원 거주)
- 카페 투어 (젊은 여성, SNS)
- 가족 나들이 (주말, 그룹)
- 직장인 회식 (평일 저녁)

→ **한 가게가 다 잡으려 하면 아무것도 못 잡는다.** 슈파가 "누굴 골라야 할지" 결정해준다.
