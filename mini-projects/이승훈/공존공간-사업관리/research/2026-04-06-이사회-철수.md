# 로하 이사회 1라운드 — 철수(밸류) 브리핑

> 안건: "공존공간을 건물주-임차인 양면 슈퍼앱 플랫폼으로 확장할 것인가?"
> 날짜: 2026-04-06
> 렌즈: 돈이 되는가? 리스크는?

---

## 결론부터

**확장한다. 단, 조건부다.**

건물주-임차인 양면 플랫폼은 글로벌에도 없는 포지션이다. 블루오션이 맞다. 그러나 블루오션이라는 것은 "아무도 안 했다"는 뜻이기도 하고, "수요가 없을 수 있다"는 뜻이기도 하다.

투자자가 이걸 보면 첫 질문은 이것이다:

> **"지금 월정산 자동화를 쓰는 건물주가 몇 명이고, 그 중 돈을 내겠다는 사람이 몇 명인가?"**

이 숫자가 없으면, 모든 확장 논의는 공상이다.

---

## 1. 사업성 평가 — Phase별 수익 모델

### Phase 1: 월정산 자동화 (현재, 0~12개월)

| 항목 | 수치 |
|------|------|
| 현재 상태 | 공존공간 1곳에서 자체 사용 중 |
| 제품 형태 | Node.js 스크립트 + 노션 |
| 적용 소요시간 | 새 업체당 4~6시간 (모듈화 완료) |
| 수익 모델 | SaaS 구독 (월 5~15만원/건물) 또는 정산 건당 과금 |
| ARPU 추정 | 월 10만원 x 12개월 = 연 120만원/건물 |
| BEP 필요 고객 수 | 50~100건물 (월 500~1,000만원, 1인 운영 기준) |
| TAM | 한국 상가건물 약 40만동 (5층 이하 근생 기준) |
| SAM | 임차인 3개 이상 상가건물 약 10만동 |
| SOM (3년) | 500~1,000건물 (SAM의 0.5~1%) |

**판정: 이것만으로는 큰 돈이 안 된다.** 연 120만원 ARPU는 너무 낮다. 그러나 이것은 "카드리더기"다 — Square도 카드리더기로 돈을 번 게 아니라, 카드리더기로 고객을 확보한 뒤 금융으로 돈을 벌었다.

### Phase 2: 데이터 리포팅 (6~18개월)

| 항목 | 수치 |
|------|------|
| 제품 형태 | 건물 수익률 대시보드, 임차인 매출 트렌드, 공실 조기경보 |
| 수익 모델 | 프리미엄 구독 (월 20~50만원) |
| ARPU 추정 | 월 30만원 x 12개월 = 연 360만원/건물 |
| 핵심 가치 | "내 건물이 얼마나 벌고 있는지 실시간으로 안다" |

**판정: 건물주에게 돈을 더 받을 수 있는 첫 번째 기회.** 건물주 입장에서 공실 1개 = 연 1,000~3,000만원 손실이다. 공실 1건만 예방해도 구독료 100배 가치다.

### Phase 3: 임차인 운영 도구 (12~24개월)

| 항목 | 수치 |
|------|------|
| 제품 형태 | 간편장부, 급여 자동계산, 세무 연동, 공동구매 |
| 수익 모델 | 임차인 직접 구독 (월 3~10만원) 또는 건물주 번들 |
| ARPU 추정 | 임차인당 월 5만원, 건물당 평균 5개 임차인 = 월 25만원 추가 |
| 비교 | 캐시노트 무료~5만원, 얼마에요 유료, 삼쩜삼 건당 과금 |

**판정: 여기서 양면 네트워크가 작동하기 시작한다.** 건물주가 플랫폼을 쓰면 임차인이 자동으로 들어온다. 캐시노트는 임차인 한 명 한 명을 모아야 하지만, 공존공간은 건물주 1명 = 임차인 5~10명이 묶음으로 온다. 이것이 유일한 구조적 우위다.

### Phase 4: 금융 서비스 (24~36개월)

| 항목 | 수치 |
|------|------|
| 제품 형태 | 매출 데이터 기반 대출 연계, 매출보전보험, 보증금 보험 |
| 수익 모델 | 금융사 연계 수수료 (대출액의 0.5~2%, 보험료의 10~20%) |
| 시장 규모 | 소상공인 대출 시장 500조원+, 매출보전보험 잠재시장 2~5조원 |
| ARPU 추정 | 금융 연계 건당 수수료 50~200만원, 보험 수수료 월 1~5만원/임차인 |

**판정: 진짜 돈은 여기에 있다.** Square 매출의 33%가 금융, Shopify의 75%가 Merchant Solutions(금융 포함)이다. 정산 데이터가 곧 신용 데이터다. 한국에 소상공인 매출보전보험은 아직 없고, 규제 샌드박스가 2026년부터 열린다. 타이밍이 있다.

### Phase 5: 생태계/마켓플레이스 (36개월+)

| 항목 | 수치 |
|------|------|
| 제품 형태 | 세무사, 인테리어, 설비, 보험사 연결 마켓플레이스 |
| 수익 모델 | 연결 수수료, 광고비, 구독 |
| ARPU 추정 | 건물당 월 10~50만원 추가 |

**판정: Phase 4까지 살아남으면 자연스럽게 열린다.** 지금 논의할 단계는 아니다.

### 한 줄 피치

> "상가건물 월정산에서 시작해, 건물주-임차인 양면 데이터를 독점하고, 금융 서비스로 수익화하는 버티컬 SaaS"

## 3. 강점 3개

1. **글로벌에 없는 포지션**: 건물주-임차인 양면 플랫폼은 Square(임차인만), HqO(오피스 빌딩만) 어디에도 없다. Bidrento(유럽)가 가장 가깝지만 소상공인 운영 도구까지는 가지 않는다.

2. **번들 고객 획득 구조**: 건물주 1명 영업 = 임차인 5~10명 자동 온보딩. CAC가 구조적으로 낮다. 캐시노트가 210만 소상공인을 1명씩 모은 것과 정반대의 접근이다.

3. **금융 서비스의 독점적 데이터 기반**: 건물주의 정산 데이터(임대료, 관리비, 공과금) + 임차인의 매출 데이터를 동시에 보유하면, 대출 심사/보험 언더라이팅에서 캐시노트보다 정확한 리스크 평가가 가능하다.

---

## 4. 약점 3개

1. **PMF 미검증**: 외부 고객 0명. 자체 건물 1곳에서만 사용 중. "내가 만들고 내가 쓰는" 상태에서 "남이 돈을 내고 쓰는" 상태로의 전환이 검증되지 않았다. 이것이 가장 치명적이다.

2. **1인 팀의 한계**: 기술, 영업, 운영, 재무를 1명이 감당한다. Phase 2부터는 전담 엔지니어 없이 불가능하다. 건물주 영업과 임차인 온보딩을 동시에 하려면 최소 3명이 필요하다.

3. **기술 장벽이 낮다**: Node.js + 노션 API + Puppeteer. 캐시노트나 토스플레이스가 "건물주 모듈"을 하나 추가하면 6개월 안에 따라잡을 수 있다. 코드가 모트가 아니라 데이터와 네트워크가 모트인데, 데이터가 아직 1건물밖에 없다.

## 5. 개선 제안 3개

### 제안 1: 6개월 안에 유료 고객 10곳을 만들어라

지금 가장 급한 건 확장 로드맵이 아니라 **PMF 검증**이다.
- 이승훈 대표의 지인/네트워크 중 상가건물 소유주를 10명 찾는다
- 월 10만원에 정산 자동화를 제공한다 (적자여도 괜찮다)
- 3개월 사용 후 "계속 쓰겠는가?"를 묻는다
- 10명 중 7명 이상이 "예"라고 하면 PMF 확인. 5명 이하면 피벗.

### 제안 2: Phase 1~2에서 죽지 마라

Phase 4(금융)가 진짜 돈이지만, 거기까지 가려면 3년이 필요하다. 그 3년 동안의 현금흐름이 문제다.
- **수익화 1순위**: 월정산 자동화 구독 (월 10~15만원/건물)
- **수익화 2순위**: 건물주 컨설팅 (공실 분석, 임대료 최적화) — 건당 50~100만원
- **수익화 3순위**: 임차인 정산 대행 — 건당 수수료 3~5%
- 목표: **Phase 2 끝날 때까지 월 1,000만원 매출** (건물 50~100곳)

### 제안 3: 캐시노트/토스와 싸우지 말고, 그들이 안 하는 곳을 파라

캐시노트 210만, 토스플레이스 수십만. 정면 대결은 자살이다.

공존공간이 이길 수 있는 유일한 싸움:
- **건물주 진입점**: 캐시노트/토스는 임차인에서 시작한다. 건물주를 고객으로 보지 않는다. 이 비대칭이 기회다.
- **복합 상가건물**: 임차인 5개 이상인 상가건물의 통합 정산은 캐시노트가 못 한다. 캐시노트는 가게 1개씩 본다.
- **매출 연동 임대료**: Percentage Rent 모델을 기술로 구현한 곳이 전세계에 없다. 이것을 Phase 2에서 실험하면 차별화 스토리가 된다.

## 6. 리스크 시나리오 분석

### 시나리오 A: 캐시노트가 "건물주 모드"를 출시한다

- **확률**: 중 (30~50%). 캐시노트는 이미 보험사와 MOU를 맺었고, 데이터 기반 금융으로 가고 있다. 건물주 시장은 아직 안 보고 있지만, 공존공간이 성과를 내면 빠르게 따라올 수 있다.
- **대응**: 속도. 건물 100곳을 먼저 확보하면 데이터 모트가 생긴다. 100곳 이전에 캐시노트가 들어오면 진다.
- **심각도**: 치명적. 캐시노트가 210만 임차인 데이터 + 건물주 모듈을 합치면 공존공간은 경쟁 불가.

### 시나리오 B: 건물주가 돈을 안 낸다

- **확률**: 중-고 (40~60%). 건물주는 보수적이고, 월 10만원도 아까워하는 사람이 많다. "관리 프로그램? 엑셀이면 되는데?"
- **대응**: (1) 무료 체험 후 전환, (2) 공실 방지 ROI를 숫자로 증명, (3) 건물주가 아닌 관리업체(PM사)를 타겟으로 피벗
- **심각도**: 높음. 이 시나리오가 현실이면 사업 모델 자체를 재검토해야 한다.

### 시나리오 C: Phase 4(금융)까지 현금이 버티지 못한다

- **확률**: 고 (60%+). 3년간 월 1,000만원 미만 매출로 버텨야 할 수 있다.
- **대응**: (1) 투자 유치 (Seed 3~5억), (2) 정산 대행 수수료로 현금흐름 확보, (3) 건물주 컨설팅으로 추가 수익
- **심각도**: 치명적. 대부분의 스타트업이 여기서 죽는다.

### 시나리오 D: 규제가 매출보전보험을 막는다

- **확률**: 중 (30~40%). 한국 보험업법은 파라메트릭 보험에 대한 법적 근거가 불분명하다.
- **대응**: (1) 보험이 아닌 "매출 보전 공제" 형태로 우회, (2) 금융규제 샌드박스 활용 (2026년 기획형 샌드박스 시작), (3) 기존 보험사와 파트너십 (삼성화재가 캐시노트에 이미 투자)
- **심각도**: 중간. Phase 4의 한 축이 막히지만, 대출 연계는 여전히 가능.

## 7. 3년 생존 전략

### Year 1 (2026.04 ~ 2027.03): "증명의 해"

| 분기 | 목표 | 핵심 지표 |
|------|------|-----------|
| Q1 | 베타 고객 5곳 확보 (무료~할인) | 건물 수, 활성 사용률 |
| Q2 | 유료 전환 시작 (월 10만원) | MRR 50만원, 유료 전환율 |
| Q3 | 고객 20곳, 데이터 리포팅 MVP | MRR 200만원, 이탈률 |

## 9. 철수의 최종 판단

**투자 가능하다. 단, 3가지 조건 하에서.**

### 조건 1: 6개월 내 유료 고객 10곳

자기 건물에서 잘 되는 것과 남의 건물에서 돈을 받는 것은 완전히 다른 게임이다. 10곳이 돈을 내면 사업이다. 0곳이면 취미다.

### 조건 2: 1인 체제를 벗어날 계획

이승훈 대표가 코딩하면서 영업하면서 정산하면서 전략까지 짜는 구조는 12개월 이상 지속 불가능하다. 최소 엔지니어 1명이 필요하다.

### 조건 3: Phase 4(금융) 전까지 죽지 않을 현금흐름

3년 뒤 금융 서비스가 진짜 돈이라는 건 맞다. 그러나 3년 동안 살아있어야 한다. Year 1에 MRR 500만원을 못 만들면, 투자 유치도 안 되고, 팀 확장도 안 되고, Phase 2로 넘어갈 수 없다.

---

> "착한 비전은 착한 돈만으로는 지킬 수 없다. 압도적인 실력과 숫자가 필요하다."
>
> 글로벌에 없는 포지션이라는 건 좋은 소식이다.
> 그러나 포지션이 있다는 것과 돈이 된다는 것은 다르다.
> 6개월 뒤, 유료 고객 10곳의 숫자로 다시 이야기하자.
