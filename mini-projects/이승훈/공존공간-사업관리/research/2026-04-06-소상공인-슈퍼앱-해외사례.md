## 2. 주요 서비스 상세 분석

### 2-1. Square (Block Inc.)

**개요**
- 2009년 잭 도시가 창업. 작은 카드리더기 하나로 시작하여 소상공인의 "운영체제"로 진화
- 미국 소상공인 54%가 결제에 Square를 사용

**기능 확장 경로**
```
카드리더기(2009) → POS(2013) → 인보이싱 → 급여(Square Payroll)
→ 비즈니스 대출(Square Capital) → 온라인스토어 → 뱅킹(Square Checking)
→ 통합 구독 플랜(2025) → AI 인사이트(2026)
```

**수익 구조**
- 결제 수수료: 대면 2.6%+10c, 온라인 2.9%+30c
- 구독: Free / Plus / Premium 플랜 (2025년 10월 통합)
- 구독+서비스 매출이 전체의 약 33%
- Square Capital: 가맹점 매출 데이터 기반 소액 대출

**락인 전략**
- 결제 → 급여 → 뱅킹 → 대출이 하나의 자금 흐름으로 연결
- Square Checking에 매출이 즉시 입금 → 다른 은행 불필요
- ACH 결제 시 수수료 0% (Square Checking 사용 시)
- 제품 4개 사용 시 이탈률 ~20% (1개 사용 시 ~70%)
- 2026년: AI가 매출 트렌드/재고 예측 → 의사결정 도구 락인

**핵심 교훈**: 결제라는 "돈이 흐르는 지점"에서 시작해 양방향으로 확장 (매출 전 = 마케팅/재고, 매출 후 = 정산/급여/세무/대출)

### 2-2. Toast

**개요**
- 2012년 설립. 음식점에만 집중하는 버티컬 SaaS의 대표 사례
- 2025년 한 해에만 3만 개 순증, 총 15.6만 매장
- ARR $2B+ 돌파 (2025 Q3)

**기능 범위**
| 카테고리 | 기능 |
|---------|------|
| POS/주문 | 테이블 주문, 테이크아웃, 배달, 온라인 주문, 키오스크 |
| 결제 | Toast Payments (자체 결제 필수) |
| 직원관리 | 스케줄링, 타임카드, 급여, 팁 분배, 채용 |
| 마케팅 | 이메일캠페인, 로열티, 기프트카드 |
| 운영 | 재고관리, 메뉴관리, 레시피 원가 |
| AI | Toast IQ: 자연어 질의 기반 실시간 인사이트 (2026) |

**수익 구조**
- 소프트웨어 구독: $0~$69+/월
- 결제 수수료: 2.49%~3.69%
- 하드웨어: $494~$1,034 (터미널, KDS 등)
- 금융 서비스: Toast Capital (가맹점 대출)

**락인 전략 (매우 강력)**
- **전용 하드웨어**: $5,000~$15,000 투자한 전용 장비가 다른 POS와 호환 불가
- **자체 결제 필수**: 외부 결제사 사용 불가, Toast Payments만 가능
- **장기 계약**: 2~3년 약정 + 자동 갱신
- **조기 해지 수수료**: $1,000~$5,000+
- **계약 중 수수료 인상 가능**: 30일 사전 통보로 수수료 인상 가능

**핵심 교훈**: 업종 특화(음식점)로 깊이 있는 문제 해결 → 높은 전환비용으로 이탈 방지. 단, 강압적 락인은 불만도 큼

### 2-3. Shopify

**개요**
- 2006년 설립. 온라인 스토어 빌더에서 시작하여 오프라인 POS, 금융, 물류까지 확장
- 560만+ 활성 스토어, 2025년 GMV $300B+ 처리

**기능 확장 경로**
```
온라인스토어 빌더(2006) → 앱스토어(2009) → Shopify Payments(2013)
→ Shopify POS(오프라인, 2013) → Shopify Capital(대출, 2016)
→ Shopify Shipping → Shopify Balance(뱅킹) → Shopify Markets(글로벌)
```

**수익 구조**
- 구독: Basic $39/월, Shopify $105/월, Advanced $399/월
- Merchant Solutions(결제, 배송, 대출 등): 전체 매출의 75% (YoY 37% 성장)
- 앱스토어: 12,000+ 앱, 2,580만+ 설치

**락인 전략**
- **앱 생태계**: 12,000개 앱으로 맞춤화 → 재구축 비용 막대
- **통합 자금 흐름**: 결제 → Balance → Capital이 하나의 파이프라인
- **옴니채널 통합**: 온라인+오프라인+소셜+마켓플레이스 재고/주문 통합
- **전환 비용**: 80%가 서드파티 앱 사용 → 이전 시 재설정 필요

**핵심 교훈**: 개방형 생태계(앱스토어)로 확장 부담 없이 기능 확대. Merchant Solutions가 핵심 수익원이 됨

### 2-4. SumUp

**개요**
- 2012년 런던 설립. 유럽 소상공인 대상 카드리더기로 시작
- 400만+ 가맹점, 비즈니스 계좌 예치금 10억 유로 돌파 (2025.12)

**기능 범위**
- 카드리더기/Tap to Pay → POS → 인보이싱 → 비즈니스 계좌 → 대출(Merchant Cash Advance) → 온라인 스토어 → 투자(머니마켓펀드, 2026 독일 시작)

**수익 구조**
- 결제 수수료: 1.69% (유럽 대면)
- 비즈니스 계좌: 무료 기본 / Plus 유료
- 대출: Merchant Cash Advance 이자
- 하드웨어 판매

**핵심 교훈**: 유럽판 Square. "은행이 더 이상 필요 없게" 만드는 전략 — 결제 수익 → 금융 서비스 확장

### 2-5. Grab / GoTo (Gojek) — 동남아 슈퍼앱

**Grab Merchant**
- 라이드헤일링에서 시작 → 배달 → GrabPay → 대출/보험
- 동남아 라이드헤일링 70% 점유율
- 가맹점에게: 배달 플랫폼 + 결제 + 타겟 마케팅 + 소액 대출

**GoTo (Gojek + Tokopedia)**
- Moka POS를 $130M에 인수(2020) → 오프라인 POS + 온라인 배달 통합
- Moka: 35,000+ 매장에 POS/결제/재고/직원관리/대출 제공
- GoPay 결제 데이터로 신용 평가 → P2P 대출 연결
- GoBiz: 50만+ 가맹점 파트너

**핵심 교훈**: 소비자 트래픽(배달)을 무기로 가맹점 확보 → POS/금융으로 락인. 거래 데이터가 대출 심사의 핵심 자산

### 2-6. 울트라-버티컬 SaaS (업종 초특화)

**Slice (피자 전문)**
- 온라인 주문 플랫폼으로 시작 → POS → 소비자앱(900만 유저) → 공급망(피자박스, 배달)
- "프랜차이즈의 해체": 독립 피자집에 프랜차이즈급 인프라 제공
- 1.5만+ 피자집, 매출 $100M+
- 수익: 주문 수수료 + 공급망 마진

**Dripos (커피숍 전문)**
- 커피숍이 10개 이상의 소프트웨어를 쓰는 문제 해결
- 올인원 POS + 주문 + 마케팅 + 직원관리 + 재고
- 비용 60% 절감 주장

**Arryved (브루어리/바 전문)**
- 양조장 특화 POS + 탭룸 관리

**핵심 교훈**: 업종을 극도로 좁히면 → 그 업종의 고유 워크플로우를 깊이 해결 → 범용 POS 대비 높은 만족도와 전환율

## 3. 건물주-임차인 플랫폼 사례

### 3-1. 테넌트 경험 플랫폼 (Tenant Experience Platform)

| 서비스 | 핵심 기능 | 타겟 | 특징 |

| **HqO** | 앱 기반 테넌트 경험, 편의시설 예약, 커뮤니티, IoT 통합 | 대형 오피스 빌딩 | 빌딩 기술 통합 "커맨드 센터" |

### 3-2. 건물주가 임차인에게 "비즈니스 운영 플랫폼"을 제공하는 사례

**현재 시장 상황:**
- 대부분의 테넌트 플랫폼은 **오피스 빌딩 입주사(기업)** 대상
- 기능도 편의시설 예약, 커뮤니케이션, 방문자 관리 등 **빌딩 생활** 중심
- **소상공인 임차인에게 사업 운영 도구(POS, 정산, 급여, 세무)를 제공하는 모델은 사실상 없음** [추정]

**가장 가까운 사례:**
- 일부 상업용 부동산 관리 소프트웨어(Yardi, AppFolio)가 테넌트 포탈에서 임대료 결제, 매출 보고, 유지보수 요청 기능을 제공
- 그러나 이는 **건물주 관점의 관리 도구**이지, 임차인의 사업 운영을 돕는 플랫폼이 아님

### 3-3. 시사점: 비어 있는 시장

```
현재 시장 구조:

[건물주 도구]                    [소상공인 도구]
  Yardi, Buildium                 Square, Toast, 캐시노트
  AppFolio, HqO                   토스플레이스, Shopify
        ↓                              ↓
  임대관리, 수금                   POS, 정산, 급여
  유지보수, 시설관리               마케팅, 세무

           ← 이 사이에 연결고리가 없음 →
```

**건물주가 임차인에게 운영 플랫폼을 제공하여 양쪽을 연결하는 서비스는 글로벌 시장에서도 발견되지 않음.** 이것이 공존공간의 기회.

### 4-2. 한국의 빈 공간 (아직 아무도 안 하는 것)

1. **급여 + 세무 통합**: Square처럼 POS 데이터 → 급여 자동 계산 → 세무 신고까지 원스톱은 없음
2. **소상공인 뱅킹/대출 통합**: SumUp이나 Square처럼 POS 매출 기반 즉시 대출 + 비즈니스 계좌는 토스가 시도 중이나 아직 초기
3. **건물주-임차인 연결 플랫폼**: 전혀 없음. 건물주 입장에서 임차인의 사업 데이터를 보고 지원할 수 있는 도구 부재
4. **업종 초특화 버티컬**: Slice(피자), Dripos(커피)처럼 한국의 치킨집, 카페, 분식집 등에 특화된 올인원은 없음
5. **공급망 통합**: Slice처럼 소상공인 공동구매/공급망 최적화를 POS와 연결한 서비스 부재
6. **임차인 매출 기반 임대료 최적화**: 매출 연동 임대료(Percentage Rent) 모델을 기술로 구현한 곳 없음

## 5. 공존공간 적용 시사점

### 5-1. 공존공간의 독특한 포지션

```
일반 POS 회사:     소상공인 → [플랫폼] → 소상공인
일반 프롭테크:     건물주   → [플랫폼] → 건물주

공존공간 가능성:   건물주 ←→ [플랫폼] ←→ 소상공인(임차인)
                          ↕
                    양방향 데이터 흐름
```

**이것이 글로벌 시장에서도 없는 모델.** Square는 소상공인만 보고, HqO는 빌딩 오피스만 본다. "건물주 + 소상공인 임차인"을 동시에 서빙하는 플랫폼은 블루오션.

### 5-2. 해외 사례에서 배울 핵심 전략

| 전략 | 사례 | 공존공간 적용 |
|------|------|-------------|
| **돈이 흐르는 지점에서 시작** | Square: 결제에서 시작 | 월정산(임대료+관리비+공과금)이 공존공간의 "결제 지점" |
| **데이터가 다음 서비스를 만든다** | Gojek/Grab: 거래 데이터 → 대출 심사 | 임차인 매출/정산 데이터 → 임대료 최적화, 공실 예측 |
| **제품 수 = 이탈 방지** | Square: 4개 제품 사용 시 이탈률 20% | 정산 + 공과금 + 유지보수 + 매출리포트 = 4개 서비스 |
| **업종 특화가 깊이를 만든다** | Toast: 음식점만 → 팁분배, 메뉴관리 | 상가건물 임차인만 → 임대차 특화 기능 |
| **플랫폼에서 생태계로** | Shopify: 앱스토어 12,000개 | 세무사, 인테리어, 보험 등 파트너 연결 |
| **금융이 진짜 수익원** | SumUp: 결제 → 뱅킹 → 대출 → 투자 | 정산 데이터 → 보증금 대출 연계, 매출보전보험 |

### 5-3. 추천 시작점 (Phase 순서)

**Phase 1: 월정산 자동화 (= Square의 "카드리더기")**
- 임대료 + 관리비 + 공과금 정산을 자동화
- 이것이 "돈이 흐르는 지점"이자 건물주-임차인 양쪽이 매월 반드시 겪는 접점
- 근거: Square도 결제(=돈의 흐름)에서 시작. 공존공간은 정산(=돈의 흐름)에서 시작

**Phase 2: 데이터 축적 → 리포팅**
- 정산 데이터로 임차인 매출 트렌드, 공실 위험, 건물 수익률 시각화
- 근거: Toast IQ, Square AI가 데이터 축적 후 인사이트 제공으로 락인

**Phase 3: 임차인 운영 도구 확장**
- 간편장부/세무 연동, 직원 급여, 공동구매
- 근거: Toast가 POS → 급여 → 마케팅으로 확장한 경로

**Phase 4: 금융 서비스 연결**
- 매출 데이터 기반 대출 연계, 매출보전보험, 보증금 보험
- 근거: Grab/SumUp/Square 모두 거래 데이터 → 금융이 가장 큰 수익원

**Phase 5: 생태계/마켓플레이스**
- 세무사, 인테리어, 설비, 보험사 등을 연결하는 파트너 마켓플레이스
- 근거: Shopify 앱스토어, Slice 공급망 통합

### 5-4. 공존공간만의 차별화 무기

1. **양면 네트워크 효과**: 건물주가 많아지면 임차인 데이터가 풍부 → 더 좋은 서비스 → 더 많은 건물주 유입
2. **Percentage Rent 모델 가능성**: 실시간 매출 데이터로 매출 연동 임대료 모델 구현 (글로벌에서도 기술적으로 구현한 곳 없음)
3. **건물 단위 공동체**: 한 건물 내 임차인들의 공동구매, 공동 마케팅, 상생 프로그램
4. **공실 예방 = 건물주 최대 Pain Point**: 임차인 매출 모니터링 → 조기 경보 → 선제 대응

## 7. 결론: 공존공간의 전략적 시사점 한 줄 요약

> **"월정산"이라는 돈의 흐름에서 시작하고, 데이터를 축적하며, 건물주-임차인 양면 플랫폼이라는 글로벌에도 없는 포지션을 점유하라.**

Square가 카드리더기 하나에서 $24B 기업이 되고, Toast가 레스토랑 POS에서 $6B 기업이 된 것처럼 — 공존공간은 "상가건물 월정산"이라는 극도로 좁고 깊은 진입점에서 시작하되, 양면(건물주+임차인) 데이터의 가치를 극대화하는 방향으로 확장해야 한다.
