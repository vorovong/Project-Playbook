# 월정산 자동화 — 소상공인 가이드

> 매달 엑셀/구글시트로 하던 정산을 스크립트 한 줄로 자동 생성
> 카페, 식당, 미용실, 임대업, 복합매장 등 어떤 가게든 적용 가능

---

## 1. 이 시스템이 하는 일

매달 반복하는 이 작업을:
- POS에서 매출 뽑고
- 카드/현금 정리하고
- 공과금 확인하고
- 인건비, 재료비, 구매내역 정리하고
- 엑셀에 숫자 넣고

**스크립트 한 줄로 끝낸다.**

```
node monthly-report.js 내가게 2026-04
→ 노션에 "4월 운영현황" 페이지 자동 생성 (표, 계산 전부 포함)
```

### 요약

기존 서비스는 **"매출 조회"**까지는 잘 해준다.
하지만 **"매출 + 지출 + 수수료 + 인건비 + 공과금을 합쳐서 월 정산 보고서를 만드는 것"**은 안 해준다.

이 시스템은:
- 기존 서비스가 하는 것 (매출 조회, 리포트) **다 하고**
- 거기에 **지출 통합, 수수료 정산, 커스텀 보고서, 복합 사업체 관리**까지 한다
- 형식도 **내가 원하는 대로** (구글시트 형식 그대로 노션에)

## 3. 우리 가게는 어떤 유형인가?

### 유형 A: 직영 가게

> 카페, 식당, 미용실, 빵집, 옷가게 등 — 내가 직접 운영

```
손님 → 결제 → 내 매출
```

- **매출 = POS 총매출** (단순함)
- 지출 = 인건비 + 재료비 + 임대료 + 공과금
- 배달앱 쓰면 배달 수수료 차감

**예시: 동네 카페**
| 항목 | 금액 |
|---|---|
| 카드매출 | 800만 |
| 현금매출 | 200만 |
| 배달앱매출 | 150만 |
| → 총매출 | 1,150만 |
| 카드수수료(1.5%) | -12만 |
| 배달수수료(15%) | -22.5만 |
| → 실수령 | 1,115.5만 |

### 유형 B: 임대업

> 건물주, 상가 임대, 공유오피스 등 — 임차인에게 수수료/임대료 받음

```
임차인 매출 → 수수료/임대료 → 내 매출
```

- **매출 = 수수료 + 임대료** (임차인 총매출 ≠ 내 매출)
- 지출 = 관리비 + 대출이자 + 인건비

**예시: 상가 임대 (공존공간)**
| 항목 | 금액 |
|---|---|
| 미식가 카드수수료(15%) | 600만 |
| 미식가 현금수수료(13%) | 70만 |
| 포토박스 운영수수료 | 45만 |
| 고정 임대료 | 763만 |
| → 내 매출 (수수료) | 715만 |
| → 내 수입 (임대료 포함) | 1,478만 |

### 유형 C: 복합형 (직영 + 임차)

> 일부는 직접 운영, 일부는 임대

- 직영 파트: 총매출이 매출
- 임차 파트: 수수료만 매출
- **각각 따로 계산해서 합산**

### 유형 D: 온라인/배달 중심

> 배민, 쿠팡이츠, 스마트스토어 등 — 플랫폼 매출이 대부분

```
플랫폼 주문 → 수수료 차감 → 정산금 입금
```

- **매출 = 플랫폼 총 주문액**
- 실수령 = 매출 - 플랫폼수수료 - 배달비 - 부가세
- 플랫폼별로 정산 주기가 다름 (주간/월간)

## 4. 매출 정리: POS가 뭔가에 따라 다르다

### POS 종류별 데이터 수집

| POS | 수집 방법 | 난이도 | 비고 |
|---|---|---|---|
| OKPOS | 웹 크롤링 (Puppeteer) | 중 | 공존공간에서 검증됨 |
| 키움POS | 웹 크롤링 | 중 | 웹 관리자 페이지 있음 |
| 토스POS | API 또는 크롤링 | 중 | 토스 비즈니스 앱 |
| 배달의민족 셀프서비스 | 웹 크롤링 | 중 | 사장님 사이트 |
| 쿠팡이츠 파트너 | 웹 크롤링 | 중 | 파트너 사이트 |
| 카드사 직접 조회 | 웹 크롤링 | 상 | 여신금융협회 |
| 엑셀 수동 | 수동 입력 | 하 | 노션 DB에 직접 |
| POS 없음 (수기) | 수동 입력 | 하 | 노션 DB에 직접 |

### POS가 없거나 수동인 경우

POS가 없어도 된다. 노션 DB에 직접 입력하면 똑같이 동작한다.

```
매일 노션 앱에서:
  일자: 2026-04-01
  카드매출: 350,000
  현금매출: 80,000
  → 저장

월말에: node monthly-report.js 내가게 2026-04
  → 자동으로 월 정산 생성
```

### 수집해야 하는 매출 데이터

| 필드 | 필수 | 설명 |
|---|---|---|
| 일자 | ✅ | 날짜 |
| 카드매출 | ✅ | 카드 결제 총액 |
| 현금매출 | ✅ | 현금 결제 총액 |
| 총매출 | ✅ | 카드 + 현금 + 기타 |
| 배달앱매출 | 해당 시 | 배민/쿠팡 등 |
| 간편결제 | 해당 시 | 카카오페이, 네이버페이 등 |
| 할인액 | 선택 | 쿠폰/이벤트 할인 |
| 결제건수 | 선택 | 분석용 |

## 5. 지출 정리: 가게마다 다른 항목들

### 지출 유형 분류

#### A. 인건비

| 상황 | 설정 방법 |
|---|---|
| 사장 혼자 운영 | 인건비 항목 없음 (또는 사장 급여만) |
| 알바 1~2명 | 시급 × 시간 또는 월 고정 |
| 정직원 있음 | 월 고정급 + 4대보험 |
| 일용직 | 일당 × 일수 |

```javascript
// config.js 예시 — 카페
인건비: {
  매장: [
    { 이름: '김알바', 금액: 1_200_000 },  // 주 5일
    { 이름: '이알바', 금액: 800_000 },     // 주 3일
  ],
},
```

#### B. 고정비 (매달 같은 금액)

| 항목 | 있는 가게 | 없는 가게 |
|---|---|---|
| 임대료 | 대부분 | 자가 |
| 대출이자 | 대출 있으면 | |
| 통신요금 | 대부분 | |
| 보험료 | 대부분 | |
| 정수기/렌탈 | 해당 시 | |
| 기장료(세무사) | 대부분 | 직접 신고 시 |
| POS 이용료 | 일부 | |
| 배달앱 월정액 | 배달 가게 | |

```javascript
// config.js 예시 — 카페
고정비: {
  임대료: 2_000_000,
  대출이자: 500_000,
  통신요금: 50_000,
  보험료: 150_000,
  기장료: 110_000,
  POS이용료: 30_000,

```

#### C. 변동비 (매달 다른 금액)

| 항목 | 수집 방법 | 설명 |
|---|---|---|
| 공과금 (전기/가스/수도) | 메일 자동 또는 수동 | 고지서 금액 |
| 재료비/원자재 | 수동 입력 | 식재료, 원두, 부자재 |
| 소모품 구매 | 수동 입력 | 컵, 빨대, 냅킨 등 |
| 배달수수료 | 자동 또는 수동 | 배민/쿠팡 정산서 |
| 카드수수료 | 자동 계산 | 매출 × 수수료율 |
| 수선/수리비 | 수동 입력 | 간헐적 |

#### D. 수수료 (결제수단별)

| 결제수단 | 일반 수수료 | 비고 |
|---|---|---|
| 신용카드 | 1.5~2.5% | 연매출 3억 이하 우대 |
| 체크카드 | 1.0~1.5% | |
| 현금 | 0% | |
| 배달의민족 | 6.8~12.5% | 중개수수료 |
| 쿠팡이츠 | 9.8~15% | |
| 카카오페이 | 1.5~2.0% | |
| 네이버페이 | 1.5~3.0% | |
| 스마트스토어 | 2~6% | 카테고리별 |

## 6. 공과금: 자동 수집 vs 수동

### 자동 수집 가능한 것

| 공과금 | 방법 | 조건 |
|---|---|---|
| 전기세 | 한전 이메일 파싱 | 이메일 고지서 설정 |
| 가스비 | 도시가스 이메일 파싱 | 이메일 고지서 설정 |
| 수도세 | 지자체 웹 크롤링 | 지자체마다 다름 |

### 수동 입력이 필요한 것

| 공과금 | 이유 |
|---|---|
| 관리비 | 건물마다 형식 다름 |
| 도시가스 (일부) | 이메일 고지서 미지원 |
| 기타 (인터넷 등) | 자동화 대비 비용 안 맞음 |

### 공과금 DB 필드 (공통)

| 필드 | 타입 | 설명 |
|---|---|---|
| 구분 | title | "지하1층", "매장" 등 |
| 종류 | select | 전기/가스/수도/관리비 |
| 청구월 | text | 2026-04 |
| 청구금액 | number | 고지서 금액 |
| 사용량 | number | kWh, ㎥ (선택) |

## 7. 정산 주기: 가게마다 다르다

| 유형 | 설명 | 적용 |
|---|---|---|
| 달력 기준 월 | 1일~말일 | 대부분의 가게 (가장 단순) |
| 주차 기준 월 | 월~일 4주 = 1개월, 나머지 이월 | 주 단위 정산하는 곳 |
| 플랫폼 정산 주기 | 배민: 주간, 쿠팡: 주간 | 배달 중심 가게 |
| 회계연도 기준 | 분기별 | 법인 |

### 달력 기준 (대부분)

```javascript
// 가장 단순 — lib/weeks.js 대신 이거 쓰면 됨
function getMonthRange(month) {
  const [y, m] = month.split('-').map(Number);
  const lastDay = new Date(y, m, 0).getDate();
  return {
    start: `${month}-01`,
    end: `${month}-${String(lastDay).padStart(2, '0')}`,
  };
}
```

### 주차 기준 (공존공간)

```
3월: 3/2(월) ~ 3/29(일) = 28일
4월: 3/30(월) ~ 4/26(일) = 28일
→ 29~31일은 다음 달로 이월
```

## 8. 시트(보고서) 구성: 유형별 추천

### 유형 A: 직영 가게 (카페/식당)

| 시트 | 내용 |
|---|---|
| **월 결산** | 매출 vs 지출 요약, 순이익 |
| **일별 매출** | 날짜별 카드/현금/배달 + 합계 |
| **인건비** | 직원별 급여 |
| **공과금** | 전기/가스/수도/관리비 |
| **재료비/구매** | 식재료, 소모품 등 |
| **배달 정산** | 배민/쿠팡별 매출, 수수료, 실수령 (해당 시) |

### 유형 B: 임대업

| 시트 | 내용 |
|---|---|
| **월 결산** | 수수료수입 + 임대료 vs 지출 |
| **임차인별 매출** | 임차인 총매출, 수수료, 정산 |
| **공과금** | 정산율 적용 |
| **인건비** | 관리인건비 |
| **고정비** | 대출이자, 관리비 등 |

### 유형 C: 복합형

유형 A + B 합산. 직영 파트와 임차 파트 분리해서 각각 계산.

### 유형 D: 온라인/배달 중심

| 시트 | 내용 |
|---|---|
| **월 결산** | 총 주문액 vs 실수령 vs 지출 |
| **플랫폼별 매출** | 배민/쿠팡/자체 주문 분리 |
| **수수료 상세** | 플랫폼별 수수료, 배달비, VAT |
| **인건비** | 직원 + 라이더 (자체 배달 시) |
| **재료비** | 식재료 원가 |

## 12. 적용 순서 (5단계)

### 1단계: 내 가게 분석 (30분)

아래 질문에 답하기:

| 질문 | 답 |
|---|---|
| 어떤 유형? | 직영 / 임대 / 복합 / 배달 |
| POS가 뭔가? | OKPOS / 키움 / 토스 / 없음 |
| 결제수단은? | 카드 / 현금 / 배달앱 / 간편결제 |
| 직원은? | 혼자 / 알바 / 정직원 |
| 고정 지출은? | 임대료, 대출, 통신, 보험, 세무사... |
| 변동 지출은? | 공과금, 재료비, 소모품... |
| 배달앱 쓰나? | 배민 / 쿠팡 / 안 씀 |
| 정산 주기는? | 달력 기준 / 주차 기준 |

### 2단계: 업체 폴더 생성 (30분)

```bash
cp -r businesses/gongzon businesses/내가게

# config.js 수정 — 위 질문 답변 기반으로 값 채우기
# .env에 노션 DB ID 추가
```

### 3단계: 노션 DB 만들기 (30분)

create-operation-dbs.js를 복사해서 내 가게에 맞는 DB 생성.

최소 필요 DB:
- 일별 매출 DB (POS 자동 or 수동)
- 공과금 DB (메일 자동 or 수동)
- 구매내역 DB (수동)

선택 DB:
- 배달 정산 DB (배달 가게)
- 임차인 매출 DB (임대업)
- 클래스/이벤트 DB (체험/교육)

### 4단계: 데이터 수집 설정 (0~3시간)

| 상황 | 할 일 | 시간 |
|---|---|---|
| POS 없음, 수동만 | 없음 (노션에 직접 입력) | 0분 |
| 같은 POS (OKPOS) | 기존 스크립트 복사, ID/PW만 변경 | 30분 |
| 다른 POS | 새 크롤러 작성 | 2~3시간 |
| 공과금 이메일 자동 | 메일 파서 설정 | 1시간 |
| 공과금 수동 | 없음 (노션에 직접 입력) | 0분 |

### 5단계: 테스트 + 운영 (1시간)

```bash
# 1개월치 데이터 입력 (수동이면 노션에서)
# 실행
node monthly-report.js 내가게 2026-04

# 기존 엑셀/시트와 숫자 대조
# 매월: 수동 데이터 입력 → 스크립트 실행
```

## 14. FAQ

### Q. 노션 안 쓰는데 다른 데 저장할 수 있나?
구글시트, 에어테이블 등도 API가 있어서 가능하다. 다만 현재 코드는 노션 기반이라 연동 모듈(lib/notion.js)을 교체해야 한다.

### Q. 코딩 모르는데 쓸 수 있나?
설정(config.js)은 숫자만 바꾸면 된다. POS 크롤러 같은 건 개발자 도움이 필요하지만, 수동 입력 방식이면 설정만으로 가능하다.

### Q. 매일 뭘 해야 하나?
- 자동 수집(POS, 메일): 아무것도 안 해도 됨 (스케줄러 설정 시)
- 수동 입력: 매일 5분 (노션 앱에서 매출/지출 입력)
- 월말: `node monthly-report.js 내가게 YYYY-MM` 한 줄 실행

### Q. 여러 가게를 하나로 관리할 수 있나?
된다. businesses/ 폴더에 가게별로 만들면 된다.
```
businesses/
  카페A/
  식당B/
  미용실C/
```

### Q. 배달앱 정산은 어떻게?
배달의민족 사장님 사이트에서 주간 정산 내역을 크롤링하거나, 정산 문자/메일을 파싱할 수 있다. 또는 노션 DB에 주간 정산 금액을 수동 입력해도 된다.

### Q. 세금 신고에 쓸 수 있나?
이 시스템은 **운영 현황 파악**용이다. 세금 신고는 세무사가 별도로 하며, 이 데이터를 세무사에게 전달하면 기장료를 줄일 수 있다.
