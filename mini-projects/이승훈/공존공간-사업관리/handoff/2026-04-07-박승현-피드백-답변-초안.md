## ⑤ 공과금 데이터 소스

### [확인됨]

| 항목 | 어디로 오는지 | 누가 확인 | 현황 |
|---|---|---|---|
| 전기 | 한전 → 메일 (네이버 + Gmail 혼용) | `utility-autofill.js` 자동 파싱 → 노션 저장 | 자동화 코드 있음 |
| 가스 | 빌코리아(삼천리) → 메일 | `utility-autofill.js` 자동 파싱 → 노션 저장 | 자동화 코드 있음 |
| 수도 | 수원시 사이트 (메일 X) | `water-bill.js` puppeteer로 고객번호 입력해 자동조회 → 노션 저장 | 자동화 코드 있음 |
| 태양광 발전 | REMS 사이트 (sqi-658 계정) | `rems-solar.js` 자동조회 → 노션 저장 | 자동화 코드 있음 |

### [모순 — 점검 필요]

박승현 대표 문서: "정산서에 공과금이 빠져 있음, 빈 채로 정산서가 나가고 있다"
↔ 코드: 4종 자동화 스크립트 모두 작성돼 있음

→ 가능성:
1. **스크립트가 정기 실행이 안 되고 있다** (작업 스케줄러 미등록 또는 중단)
2. **노션 저장은 되는데 정산서(`monthly-report.js`)에 반영되는 경로가 끊어져 있다**
3. **임차인별 배분 로직이 없어서**, 노션에 들어와도 "누구한테 얼마"가 안 정해져서 정산서엔 못 박힘

→ **김비서가 진단 가능:** 이번 주 안에 4개 스크립트 실제 1회 실행 + 노션 저장 결과 + monthly-report 실행 결과 비교해서 어디서 끊겼는지 추적 가능.

### [확인 필요]

| 항목 | 확인 질문 |
|---|---|
| 임차인별 배분 기준 | 면적 비율? 고정 분담금? 실사용량? 아예 안 나누고 임대인 부담? |
| 메일 수신 계정 | 한전/가스 메일이 네이버/Gmail 중 어느 쪽? (둘 다라면 계정 목록) |
| 수원시 고객번호 | 어디에 보관? `.env`인지 별도인지 |

## 정리: 회신을 위해 이승훈이 채워야 할 것 (우선순위)

| 우선 | 항목 | 작업 | 예상 소요 |
|:---:|---|---|---|

| 🟡 4 | ⑤ 공과금 메일 계정 + 배분 기준 | 답변만 | 10분 |

### 김비서가 추가로 진단할 것 (이승훈 승인 시)

1. 공과금 4개 스크립트 1회 실행 → 어디서 끊겼는지 추적 (⑤번 모순 해결)
2. 노션 DB 6개 query → 컬럼 사용현황 자동 표 생성 (③번)
3. 노션 DB 사업별 데이터 건수 분류 → 종료/활성 분류 (⑥번)
