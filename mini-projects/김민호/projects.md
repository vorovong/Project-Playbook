## 1. 회의 비서

### 배경과 의도

현재 공존공간 회의는 아이폰으로 녹음하고, 승훈이형이 네이버 클로바로 텍스트 변환 후 구글 문서로 정리해주고 있다. 문제는 이 과정이 사람 손을 타야 한다는 것과, 정리된 결과도 단순 회의록 수준이라는 점이다.

최종적으로 원하는 건 "쓸모없는 잡담은 걸러내고, 업무별로 맥락에 맞게 정리하며, 새로운 주제가 나오면 자동으로 카테고리를 생성하고, 그 카테고리가 다음 회의록 정리에 반영되는" 학습형 회의 비서다.

그리고 최종 결과물은 GitHub에 md로 모으고 싶다.

### 미니 프로젝트 범위

녹음 파일(m4a)을 넣으면 텍스트를 추출하는 최소 자동화.
OpenAI Whisper(오픈소스)를 로컬에서 돌려서, LLM API 비용 없이 스크립트만으로 STT를 처리한다.

### TODO

- [ ] Whisper 로컬 설치 및 한국어 STT 테스트
- [ ] 녹음 파일(m4a) → 텍스트 추출 스크립트 작성
- [ ] 추출된 텍스트를 md 파일로 저장

### 이후 계획

미니 달성 후 풀 프로젝트로 확장:
- 추출된 텍스트를 업무별로 자동 분류 (LLM 활용)
- 잡담 필터링
- 카테고리 자동 생성 + 누적 학습 (다음 회의에 반영)
- GitHub 자동 푸시

## 2. 공존공간 AX 아카이빙

### 배경과 의도

공존공간의 문서 자산이 구글 드라이브에 구글 문서와 txt 파일로 흩어져 있다. 폴더도 많고 파일도 여기저기 뿌려져 있어서, 수작업으로 정리하는 건 비현실적이다.

이걸 GitHub 레포지토리로 전환해서 체계적으로 아카이빙하고 싶다. 구글 문서와 txt를 md로 변환하고, 각 파일 헤더에 YAML 프론트매터를 넣어서 AI나 클로드 코드가 검색하기 좋은 구조를 만들려고 한다.

전체 폴더를 클로드 코드가 다 읽으면 토큰 사용량이 너무 크기 때문에, 폴더 트리 복사와 md 변환은 스킬/스크립트로 처리하고 토큰 사용을 최적화해야 한다. 단순 문서 변환은 저렴한 모델(Haiku, Sonnet)로 분산하고, YAML 프론트매터의 핵심 내용 작성처럼 판단이 필요한 부분만 Opus가 담당하는 에이전트 구조가 필요하다.

### 미니 프로젝트 범위

구글 드라이브 폴더 트리를 자동으로 긁어오고, txt 파일을 md로 변환하는 스크립트까지.

### TODO

- [ ] 구글 드라이브 API 또는 MCP 연동으로 폴더 트리 추출
- [ ] 폴더 트리를 로컬에 동일 구조로 복제
- [ ] txt → md 자동 변환 스크립트 작성

### 이후 계획

풀 프로젝트로 확장:
- 구글 문서 → md 변환 자동화
- YAML 프론트매터 자동 생성 (Opus가 핵심 내용 작성)
- 모델 분산 에이전트 구축 (단순 변환 = Haiku/Sonnet, 판단 = Opus)
- 토큰 최적화 스킬 제작
- GitHub 레포지토리 구조 설계 및 자동 푸시

## 3. 텔레그램 알림 봇

### 배경과 의도

현재 팀 소통은 카카오톡으로 하고 있는데, 카카오톡은 API가 막혀 있어서 클로드 코드에서 자동으로 메시지를 보낼 수 없다. 텔레그램은 봇 API가 완전 무료이고 그룹 채팅에 봇을 추가할 수 있어서, 클로드 코드에서 "git push 했으니 pull 받으세요" 같은 알림을 자동으로 보내는 게 가능하다.

### 미니 프로젝트 범위

텔레그램 그룹 채팅방을 만들고, 봇이 메시지 하나를 보내는 최소 자동화.

### TODO

- [ ] 텔레그램 BotFather로 봇 생성
- [ ] 팀 그룹 채팅방 생성 + 봇 추가
- [ ] 클로드 코드에서 봇으로 메시지 보내는 스크립트 작성

### 이후 계획

- 클로드 코드 커스텀 스킬로 만들기 (`/notify` 같은 명령어)
- git push 시 자동 알림 (hooks 연동)
- 팀원들 각자의 클로드 코드에서도 사용 가능하게 확장
