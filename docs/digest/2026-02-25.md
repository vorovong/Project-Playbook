## 팀 패턴

### 공통으로 배운 것
- **시스템 분리 사고**: 박승현(GIRIN FSM ↔ GONGZONE FSM 분리), 이승훈(자동 연동 ↔ 수동 입력 분리), 김민호(progress 관리 체계 정비) — 각자 "하나로 합치지 않고 역할별로 나누는" 패턴을 적용

### 공통으로 어려워한 것
- **세션 간 연속성**: 박승현은 이전 대화를 기억 못 하는 문제 발견 → MD 파일 저장으로 해결. 김민호는 메모리 동기화 문제를 CHANGELOG로 추적 중. 세션 간 맥락 유지가 팀 공통 과제.

## 서로 도울 수 있는 포인트

- 이승훈의 **KAMIS API 연동 경험** (API 키 신청, 품목별 적합성 판단) → 박승현이 GONGZONE FSM에서 외부 데이터 연동할 때 참고
- 박승현의 **커스텀 명령어 패턴** (commands.md로 분리) → 이승훈도 반복 작업을 명령어화할 때 참고
- 김민호의 **progress 관리 패턴** (날짜 내림차순, 파일번호 규칙) → 팀 전체 문서 일관성 유지

## 플레이북 개선 제안

- 세션 간 맥락 유지 문제가 반복 → 커스텀 명령어(commands.md) 접근법을 팀 가이드에 추가 검토
- KAMIS 같은 외부 API 키 신청 과정 기록 → 향후 API 연동 가이드 재료로 활용
