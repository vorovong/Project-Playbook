## 엣지케이스 기록

> **엣지케이스(Edge Case)란?** 정상적인 흐름에서는 안 만나지만, 특수한 조건에서 발생하는 상황. "보통은 괜찮은데 이럴 때만 문제가 된다"는 것. 예를 들어 우산은 비 올 때만 필요한데, 비를 안 만나봤으면 우산의 필요성을 모름. **엣지케이스를 기록해두는 이유는, 한 번 겪은 특수 상황을 팀 전체가 공유하면 같은 삽질을 반복하지 않기 때문.**

### git pull로 해결 안 되는 변경이 있다

**상황**: 박승현의 commands.md "잘자" 규칙을 수정해서 push했지만, 박승현의 클로드 코드 메모리는 로컬(`.claude/`)에 저장되므로 git pull로 바뀌지 않음.

**왜 엣지케이스인가**: 보통은 파일을 고치고 push하면 상대가 pull해서 끝. 하지만 클로드 코드 메모리는 git 대상이 아니라서 이 "보통"이 안 통하는 특수 상황.

**해결**: CHANGELOG "아직 반영 안 된 것" 섹션에 기록 → 아침에 확인하고 직접 전달. 상대방이 클로드 코드에게 "메모리 업데이트해줘"라고 말하면 그때 반영됨.

**팀 규칙**: 엣지케이스는 매일 발생하지 않으므로 일과 루틴에 넣지 않고, 발생할 때마다 CHANGELOG + 다이제스트에 기록하여 팀 전체가 공유한다.
