# 케이스 스터디 — 코딩 제로에서 AI 뉴스 자동화까지

> 비개발자가 클로드 코드로 이틀 만에 개인 AI 뉴스 허브를 만든 이야기

---

## 한줄 요약

터미널을 한 번도 열어본 적 없는 사람이 클로드 코드와 대화하며 **이틀 만에 21개 소스에서 AI 뉴스를 자동 수집·요약·배포하는 시스템**을 완성했다.

---

## 시작 전 상태

- **코딩 경험**: 제로. Python, HTML, JavaScript 전부 처음.
- **터미널**: 생전 처음 열어봄. `cd`, `ls` 같은 기본 명령어도 모르는 상태.
- **목표**: "매일 반복하는 일을 자동화하고 싶다."
- **도구**: MacBook + 클로드 코드 (Claude Code CLI)

---

## 타임라인

### Day 1 — 처음부터 배포까지 (v1.0)

**에피소드 1: "뭘 만들지?"**

클로드 코드가 물었다: "매일 반복하는 일 중에, 자동으로 되면 좋겠다고 생각한 적 있어?"

대답: "AI 뉴스를 매일 여기저기서 찾아보는데, 한 곳에서 정리해서 보고 싶다."

이 한 마디가 프로젝트의 시작이었다.

**에피소드 2: 첫 번째 자동화**

클로드 코드가 하나씩 설명하면서 만들었다:
- `sources.json` — 신뢰하는 뉴스 소스 목록 (17개)
- `collect_news.py` — RSS에서 뉴스를 가져오고, Gemini AI로 요약
- GitHub Actions — 매일 아침 8시에 자동 실행
- GitHub Pages — 결과물을 웹사이트로 자동 배포

**결과**: 자는 동안 뉴스가 수집되고, 아침에 내 사이트에서 볼 수 있게 됨.

이 순간이 "Ignition Moment"였다. **"내가 이걸 만들었다!"**

**에피소드 3: 첫 번째 문제 발견**

사이트를 열어보니 문제가 보였다:
- 요약이 너무 짧아서 무슨 내용인지 모르겠음
- 어제 뉴스가 사라짐 (매일 덮어쓰기)
- AI 개발 뉴스만 많고, 이미지·영상 AI 소스가 부족

→ 사용자가 직접 문제를 발견하고 지적한 첫 번째 순간.

---

### Day 2 — 문제 해결의 연속 (v2.0 → v3.0)

**에피소드 4: 안정성과 깊이 (v2.0)**

Day 1의 문제를 하나씩 해결:
- Gemini API 에러 시 자동 재시도 (2회)
- 날짜별 아카이브 저장 (어제 뉴스도 볼 수 있음)
- 기사 원문 크롤링 + 유튜브 자막 추출 → 깊이 있는 요약
- 소스 4개 추가 (Hugging Face, Matt Wolfe, The Rundown AI, Stability AI)

**에피소드 5: 타임존 버그 (v2.2)**

"왜 오늘 날짜가 아니라 어제 날짜로 뜨지?"

원인: GitHub Actions 서버가 미국 시간(UTC) 기준. 한국시간(KST)으로 변환 필요.

→ 버그를 발견하고, 원인을 이해하고, 수정하는 과정을 함께 경험.

**에피소드 6: 사용자 피드백 폭발 (v3.0)**

실제로 사이트를 쓰면서 구체적인 피드백이 쏟아짐:
- "모바일에서 글씨가 너무 작아"
- "요약이 문장체인데, 개조식이 더 읽기 편해"
- "'왜 중요한가'가 기사 내용 반복이야. 나한테 어떤 영향인지 알려줘"
- "아카이브에도 카테고리 필터가 있어야 해"
- "Anthropic, Midjourney 같은 공식 소스가 빠져있어"

모든 피드백을 반영:
- 개조식 명사구(-음/-임)로 전체 변경
- "나에게 미치는 영향(my_impact)"으로 교체
- 모바일 UI를 Apple HIG 기준으로 개선
- 아카이브 카테고리 필터 추가

**에피소드 7: 한계 인식과 풀 프로젝트 전환**

v3.0을 쓰면서 핵심 깨달음:

> "뉴스를 모아주는 건 좋은데, 나한테 맞는 걸 골라주는 **편집장**이 필요해."

15개 기사 중 진짜 관심 있는 건 5개. 수준도 안 맞고, 깊이도 부족.
→ 단일 파이프라인의 한계를 직접 경험하고, 에이전트 기반 아키텍처로의 전환을 결정.

## 최종 결과물

### ai-news-hub v3.0
- **사이트**: https://vorovong.github.io/my-ai-hub/
- **소스**: 21개 (16개 자동, 5개 수동)
- **자동화**: 매일 아침 8시 자동 실행 (GitHub Actions)
- **기능**: AI 뉴스 수집 → Gemini 요약 → 카테고리 분류 → 아카이브 → 모바일 반응형

### 코드 규모
- `collect_news.py`: ~800줄 (RSS 수집 + 원문 크롤링 + Gemini 요약 + HTML 생성)
- `sources.json`: 소스 21개 등록
- `daily-update.yml`: GitHub Actions 워크플로우

### 기술 스택
- Python + Gemini 2.5 Flash + GitHub Actions + GitHub Pages
- trafilatura (원문 크롤링) + youtube-transcript-api (자막 추출)

## 핵심 인사이트 5개

### 1. "맥락이 결과의 질을 결정한다"

같은 AI에게 같은 질문을 해도, 얼마나 구체적으로 맥락을 전달하느냐에 따라 결과가 완전히 달라진다.

- v1.0 프롬프트: "AI 뉴스를 요약해줘" → 밋밋한 결과
- v3.0 프롬프트: 사용자 프로필(관심 분야, 불필요한 것, 선호 문체) + 소스 신뢰도 + 예시까지 포함 → 훨씬 정확한 결과

비유: "같은 요리사에게 '맛있는 거 해주세요'라고 하는 것과 '매운 걸 좋아하고, 오늘 피곤해서 따뜻한 국물이 먹고 싶어요'라고 하는 것의 차이."

### 2. "실수해도 되돌릴 수 있다"

이틀 동안 수많은 에러를 만났다:
- Gemini가 JSON을 깨뜨림 → 재시도 로직 추가
- 타임존 버그로 어제 날짜 표시 → KST 변환
- 유튜브 API가 바뀌어서 자막 추출 실패 → 새 API 호출 방식 적용
- 모바일에서 태그가 밀림 → CSS 수정

모든 실수는 "이렇게 하면 안 되는구나"를 배우는 기회였다. 파일을 잘못 만들어도 지우면 되고, 코드가 안 되면 다시 고치면 된다.

### 3. "직접 써보면서 개선하는 게 최고의 방법"

기획서를 100장 쓰는 것보다, v1.0을 빨리 만들고 직접 써보는 게 10배 효과적이었다.

- v1.0을 열어본 순간 "요약이 너무 짧다"를 깨달음
- v2.0을 써보고 "모바일에서 안 보인다"를 발견
- v3.0을 쓰면서 "편집장이 필요하다"는 핵심 니즈를 파악

**만들고 → 쓰고 → 고치고**의 반복. 이것이 가장 빠른 학습이다.

### 4. "챗봇은 물고기를 주고, CLI는 낚싯대를 만들어준다"

ChatGPT에게 "오늘 AI 뉴스 알려줘"라고 물으면 한 번 답을 준다. 내일 다시 물어야 한다.

클로드 코드에게 같은 질문을 하면, **매일 자동으로 뉴스를 수집하는 시스템**을 만들어준다. 한 번 만들면 매일 새벽에 자동으로 돌아간다.

- 챗봇: 결과를 준다 (일회성)
- 클로드 코드: 결과를 만드는 **시스템**을 만든다 (반복 가능)

### 5. "복잡해지면 문서부터"

v3.0까지는 "대화 → 코드 → 배포"로 빠르게 진행했다. 하지만 "편집장 에이전트"를 만들려니 구조가 복잡해졌다.

이때 배운 것: **복잡한 프로젝트는 코드 전에 문서를 먼저 쓴다.**

- CLAUDE.md (프로젝트 인수인계서)
- PRD (무엇을, 왜 만드는가)
- Spec (어떻게 만드는가)

이 문서들이 있으면, 새 대화를 시작해도 클로드 코드가 문서를 읽고 맥락을 즉시 파악한다. 매번 처음부터 설명할 필요가 없다.

## 동료를 위한 시작 가이드

### Step 1: 환경 준비 (10분)

1. 터미널 열기 (Mac: Spotlight에서 "터미널" 검색)
2. 클로드 코드 설치: `npm install -g @anthropic-ai/claude-code`
3. 프로젝트 폴더 이동: `cd ~/projects/Project_Playbook`
4. 클로드 코드 실행: `claude`

### Step 2: 첫 대화 (5분)

클로드 코드가 인사하면, 이렇게 말해보세요:

> "매일 반복하는 일 중에 자동화하고 싶은 게 있어."

구체적일수록 좋다:
- "매주 같은 형식의 보고서를 만들어야 해"
- "이메일에서 특정 정보를 반복적으로 정리해야 해"
- "매일 같은 데이터를 확인하고 요약해야 해"

### Step 3: 첫 미니 프로젝트 (30분)

클로드 코드가 대화를 통해 요구사항을 파악하고, 간단한 자동화를 함께 만들어줄 거야.
모든 과정을 설명해주니까, 모르는 게 나오면 바로 물어봐.

**기억할 것:**
- 실수해도 괜찮아. 파일 지우면 되고, 코드 고치면 돼.
- 완벽할 필요 없어. v1.0은 30%만 되어도 OK. 나중에 고치면 돼.
- 질문은 많이 할수록 좋아. "이게 뭐야?" "왜 이렇게 해?" 다 물어봐.

### Step 4: 반복하고 성장하기

첫 미니 프로젝트가 끝나면:
1. 직접 결과물을 써본다
2. "이것도 되면 좋겠다"를 발견한다
3. 클로드 코드에게 개선을 요청한다
4. 이 과정을 반복한다

미니 프로젝트 2~3개를 진행하면, 어느새 CLAUDE.md를 직접 작성하고, 풀 프로젝트를 기획하게 된다.

## FAQ

**Q: 코딩을 아예 모르는데 가능해?**
A: 이 케이스 스터디의 주인공이 코딩 경험 제로였다. 클로드 코드가 모든 코드를 작성하고, 왜 그렇게 하는지 설명해준다. 필요한 건 "무엇을 만들고 싶은지" 말하는 것뿐이다.

**Q: 어떤 자동화를 만들 수 있어?**
A: 파일을 다루는 거의 모든 반복 작업. 뉴스 수집, 보고서 생성, 데이터 정리, 이메일 분류, 일정 관리 등. 단, 회사 내부 시스템 접근이 필요한 작업은 제한될 수 있다.

**Q: 비용이 얼마나 들어?**
A: 클로드 코드 구독료(Claude Pro/Max)만 있으면 된다. GitHub, GitHub Pages는 무료. Gemini API도 무료 티어로 충분하다.

**Q: 영어를 잘 못해도 되나?**
A: 클로드 코드와 한국어로 대화하면 된다. 코드는 영어지만, 클로드 코드가 무슨 뜻인지 설명해준다.

**Q: 만든 자동화가 멈추면 어떻게 해?**
A: 클로드 코드에게 "자동화가 멈췄는데, 뭐가 문제야?"라고 물어보면 된다. 에러 로그를 보고 원인을 알려준다.
