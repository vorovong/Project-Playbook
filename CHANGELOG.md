## 아직 반영 안 된 것

### 박승현에게 전달 필요 (2026-02-25 아침)

- **commands.md "잘자" 명령어 변경됨**: progress에는 템플릿 양식 진행 기록만, 대화 원문/요약은 미니프로젝트 폴더에 넣도록 수정
- **이유**: 기존 "잘자"가 대화 원문 + 요약을 별도 파일로 progress에 올렸는데, progress 양식 규칙에 안 맞음
- **형이 해야 할 것**: 클로드 코드에게 "잘자 규칙 메모리 업데이트해줘. commands.md 읽고 반영해"라고 말하기
- **참고**: commands.md 파일은 git pull로 자동 반영되지만, 클로드 코드 메모리는 로컬이라 형이 직접 업데이트 요청해야 적용됨

## 2026-02-23 (2차)

### 바뀐 것

- **CLAUDE.md 다이어트**: 226줄 → 153줄로 36% 줄임. 원칙과 판단 기준만 남기고, 비유·대사·예시 같은 상세 내용은 별도 문서로 분리
- **교육 레퍼런스 문서 신규**: `docs/guides/교육-레퍼런스.md` — CLAUDE.md에서 빠진 비유 모음, 넛지 대사, 첫 세션 가이드를 여기에 모음
- **사용자 식별 로직 추가**: 세션 시작 시 이름을 물어보고, 폴더 이력 기반으로 Phase를 자동 판단하도록 변경
- **Phase를 사용자별로 개별 적용**: 기존에는 전체 Phase였는데, 이제 멤버마다 다른 Phase 가능
- **스킬(Skill) 성장 경로 추가**: Phase 1에서 CLAUDE.md로 원리 체득 → Phase 2에서 개념 연결 → Phase 3에서 스킬 설계로 이어지는 흐름
- **설치 가이드 변경**: 순정 터미널(PowerShell/Terminal) 대신 **Warp** 터미널을 먼저 설치하도록 변경. 순정 터미널 안내는 접힌 섹션으로 보존
- **.gitignore 추가**: `.env`, `__pycache__/`, `*.pyc` 파일이 실수로 올라가지 않도록 설정

### 없어진 것

- **CLAUDE.md 작성법 학습 (점진적)** 섹션 제거 — 실제로는 대화 흐름에서 자연스럽게 진행되므로 별도 단계가 불필요
