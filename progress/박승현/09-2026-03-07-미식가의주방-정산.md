### 배운 개발 지식

- **Data Entry Form 패턴** — 입력 폼 하나 + 전송 버튼으로 데이터를 쌓고, 요약 시트가 자동 계산하는 구조. 식당 주문서처럼 한 장 쓰고 넘기면 주방에서 알아서 정리하는 것.
- **SUMPRODUCT** — 여러 조건을 동시에 만족하는 셀만 골라 합산하는 수식. "3월이면서 1주차인 행의 카드매출만 합쳐줘" 같은 요청을 처리.
- **Apps Script** — 구글시트에 붙이는 자동화 코드. 전송 버튼 클릭 → 검증 → 저장 → 초기화를 한 번에 처리.

- 📐 유저 인터페이스 설계 (UI Design) ⚖️
  시트1을 누가 입력하는가? 비개발자 동료.
  선택지: ①가로형 테이블(정보 밀도 높음) ②세로형 폼(입력 직관적) ③단일 폼+전송(반복 없음)
  판단: 12월에 #50번 폼까지 스크롤하는 건 사용자 고문. 폼 하나+전송이 가장 직관적.
  = 사용자가 하는 부분은 사용자 눈높이에 맞춰야 한다. (ATM에서 버튼 3개면 충분한데 50개 만들 이유 없음)

- 📐 데이터 타입 인테그리티 (Data Type Integrity) 🔒
  Excel→구글시트 변환 후 SUMPRODUCT가 0을 반환.
  선택지: ①수식만 수정(미봉) ②Apps Script에서 Number() 강제 변환(입력 단) ③setupFormulas()로 구글시트 네이티브 수식 재생성(근본)
  판단: ②+③ 동시 적용. 입력도 숫자로, 수식도 구글시트에서 직접 생성.
  = 데이터는 들어갈 때 타입을 확정해야 한다. (우편함에 넣을 때 봉투 규격 맞춰야 자동 분류기가 작동하는 것)
