### 배운 개발 지식

- **Hook (훅)** — 특정 이벤트 발생 시 자동으로 실행되는 스크립트. 문 앞의 경비원처럼 AI가 잊어도 무조건 작동. Claude Code에 17개 이벤트 지원.

- **SessionStart / PreToolUse / PostToolUse / PreCompact** — Hook이 발동하는 4가지 시점. 각각 세션 시작, 도구 실행 전, 도구 실행 후, 컨텍스트 압축 전.

- **Bypass 메커니즘** — Hook이 파일 수정을 차단했을 때, 임시 파일(bypass)을 만들어 5분간 잠금 해제하는 방식. 완전 차단이 아니라 "승인 후 통과" 구조.

- 📐 이벤트 드리븐 아키텍처 (Event-Driven Architecture) 🔒
  "부탁"으로 규칙을 지키게 하면 4회 실패. "시스템"으로 강제하면 0회 실패.
  선택지: ①규칙만 작성(간단하지만 AI가 무시 가능) ②Hook으로 강제(설계 필요하지만 확실)
  판단: 로봇 모드 4회 반복 = 규칙의 한계 증명. 시스템으로 전환.
  = 교통 법규만으로 사고가 안 줄면, 신호등(시스템)을 설치해야 한다.

- 📐 가드레일 패턴 (Guardrail Pattern) ⚖️
  프론티어(OpenAI, Claude Code)가 공통으로 쓰는 패턴. 도구 실행 전에 검증 → 차단 또는 허용.
  선택지: ①완전 차단(안전하지만 유연성 없음) ②경고만(유연하지만 무시 가능) ③차단+bypass(균형)
  판단: 불약 파일은 차단+bypass, 일반 경고(PDF, progress)는 경고만.
  = 금고는 열쇠 필요, 서류함은 "확인하세요" 스티커.

- 📐 설계와 구현의 순서 (Design Before Implementation) 🔒
  주문 체계화(메뉴판 정리)와 Skill 전환(버튼 연결)을 동시에 하려 했음.
  선택지: ①설계와 구현 동시(빠르지만 메뉴 바뀌면 버튼 재작업) ②설계 먼저, 구현 나중(느리지만 한 번에 완성)
  판단: 메뉴판이 확정 안 됐는데 버튼 만들면, 메뉴 바뀔 때마다 버튼도 다시 만들어야 함.
  = 건축 설계도 확정 전에 벽돌 쌓기 시작하면, 설계 바뀔 때 벽 허물어야 한다.

- 📐 암묵지 외재화 시스템 (Tacit Knowledge Externalization System) ⚖️

  메인 AI가 직접 파일을 읽으면 컨텍스트에 쌓임. 심부름꾼(서브에이전트)이 대신 읽고 요약만 반환하면 메인은 가벼움.
  선택지: ①메인이 직접 읽기(정확하지만 무거움) ②서브에이전트 요약(가볍지만 요약 품질 의존)
  판단: 브리핑 수집은 정확한 원문보다 핵심 요약이 중요. 심부름꾼이 적합.
  = 사장이 모든 서류를 직접 읽는 것 vs 비서가 요약 메모 한 장 올려주는 것.

- 📐 엠시피 (MCP — Model Context Protocol) ⚖️
  외부 서비스(Notion, Gamma 등)를 AI에 연결하는 플러그인. 연결만 해도 도구 설명서가 항상 컨텍스트에 로드됨.
  선택지: ①항상 연결(편하지만 무거움) ②필요할 때만 켜기(가볍지만 번거로움)
  판단: 지금 안 쓰는 도구는 꺼두면 ~10%+ 절약 가능. 나중에 필요하면 다시 켜면 됨.
  = 스마트폰에 앱 100개 설치하면 느려지는 것. 자주 쓰는 앱만 남기고 나머지는 삭제.
