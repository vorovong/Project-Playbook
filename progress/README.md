# progress/ — 진행 기록

이 폴더에는 **미니 프로젝트의 일일 진행 기록**만 저장합니다.

## 폴더 구조

```
progress/
  김민호/
    01-2026-02-23-팀-협업-시스템.md
  박승현/
    01-2026-02-23-GIRIN-마스터-md-만들기.md
  이승훈/
    01-2026-02-23-슈파-데이터분석.md
```

## 규칙

1. **자기 이름으로 폴더**를 만드세요
2. **파일명**: `[번호]-[시작일]-[프로젝트이름].md` (예: `01-2026-02-23-이메일-자동화.md`)
3. **양식**: `docs/templates/progress-template.md` 의 형식을 **반드시** 따를 것
4. 미니 프로젝트가 끝나면 `docs/case-studies/` 폴더에 케이스 스터디도 올려주세요

## 동료가 별도로 작성하는 문서는 2종류뿐

| 문서 | 위치 | 시점 |
|---|---|---|
| **진행 기록** | `progress/{이름}/` | 매 세션 종료 시 |
| **케이스 스터디** | `docs/case-studies/` | 미니 프로젝트 완료 시 |

그 외 모든 결과물(기획서, 시나리오, 코드, 설계 문서 등)은 **해당 미니 프로젝트 폴더 안에** 생성됩니다:
`mini-projects/{이름}/{프로젝트명}/`

## 핵심 형식 요약

```markdown
## 기본 정보
- **작성자**: / **시작일**: / **한줄 목표**: / **상태**:

## YYYY-MM-DD          ← "Day N" 형식 사용 금지
### 대화 흐름           ← "오늘의 대화 흐름" 등 수식어 금지
### 결과               ← "오늘의 결과" 등 수식어 금지
### 배운 개발 지식
### 어려웠던 것 / 다음에 할 것
```

상세 규칙은 `docs/templates/progress-template.md` 하단의 **작성 규칙 (필수)** 참조.
