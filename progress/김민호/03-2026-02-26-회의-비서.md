### 배운 개발 지식

- **구독 vs API** — 구독(Google AI Pro)은 웹 채팅 접근권, API는 코드에서 호출하는 것. 별개 과금. (식당 멤버십 vs 식재료 납품 계약)

- 📐 하네스 엔지니어링 (Harness Engineering) 🔄
  풀 프로젝트 전환 시 코드부터 짜면 방향을 잃기 쉬움.
  선택지: ①바로 코딩 ②CLAUDE.md만 작성 ③하네스 세트(CLAUDE.md+spec.md+planning.md+템플릿+설정) 생성
  판단: 6개 파일로 하네스를 완성하면, 어떤 세션에서든 에이전트가 맥락을 즉시 파악하고 실행 가능.
  = 코드를 직접 짜는 대신, AI가 잘 짤 수 있는 환경을 설계하는 것. (말 탈 사람이 말을 훈련하는 게 아니라, 말이 잘 달릴 수 있는 코스를 설계하는 것)

- 📐 멀티모달 파이프라인 (Multimodal Pipeline) ⚖️
  회의 녹음 → 텍스트 변환 → 회의록 생성이 2단계로 나뉘면 비용과 복잡도 증가.
  선택지: ①Whisper STT + LLM 정리 (2단계) ②Gemini 오디오 직접 입력 (1단계)
  판단: Gemini가 오디오를 직접 이해하므로 중간 변환 불필요 → 파이프라인 단순화 + 맥락 손실 방지.
  = 번역가에게 원문을 직접 주는 것 vs 다른 사람이 요약한 걸 주는 것. 원문이 더 정확함.
