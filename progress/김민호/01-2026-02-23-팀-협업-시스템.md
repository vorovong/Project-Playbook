  선택지: ①쉬운 한국어만 ②영어만 ③한국어 발음 메인 + 영어 괄호

  = 전문 용어를 쓰면 AI가 더 정확한 답을 준다. (의사에게 "배 아파요" vs "우하복부 통증")

### 배운 개발 지식

- **3단계 방어선 패턴** — 규칙을 한 곳에만 쓰면 놓치기 쉬움. 진입 경로마다 규칙을 배치하면 어떤 방향으로 접근해도 규칙을 만나게 됨. 건물의 화재 안내도를 층마다 붙이는 것과 같음.
- **git rename (mv)** — 파일 이름을 바꾸거나 위치를 옮기는 것. git이 "삭제 + 생성"이 아닌 "이동"으로 인식해서 히스토리가 유지됨.
- **공유 파일 vs 로컬 메모리의 한계** — git으로 공유되는 파일(CLAUDE.md, commands.md)은 pull로 동기화되지만, 각자의 클로드 코드 메모리(`.claude/`)는 로컬이라 git 대상이 아님. 규칙을 바꿔도 상대방이 직접 메모리를 업데이트해야 적용됨. 공유 문서함은 바꿀 수 있지만, 상대방 머릿속 메모는 본인만 바꿀 수 있는 것과 같음.
- **CHANGELOG "미반영 사항" 패턴** — git pull만으로 해결 안 되는 변경(메모리 업데이트, 수동 설정 등)을 CHANGELOG에 명시해두면, 아침 브리핑 때 빠짐없이 전달 가능. 팀 협업에서 "자동 반영"과 "수동 전달"을 구분하는 운영 체계.

### 배운 개발 지식

- **git pull** — GitHub에 새로 올라온 것만 내 컴퓨터로 가져오기. 공유 폴더에서 새 파일만 받아오는 것과 같음.
- **git commit vs git push** — commit은 내 컴퓨터에만 저장, push는 GitHub에 업로드. 편지를 쓰는 것(commit)과 우체통에 넣는 것(push)의 차이.
- **Collaborator 권한** — GitHub 저장소에 다른 사람이 파일을 올릴 수 있게 초대하는 것. 공유 폴더에 접근 권한을 주는 것과 같음.
- **gh auth login** — GitHub CLI로 인증하기. 비밀번호 대신 브라우저 로그인으로 안전하게 연결.
- **Skills (스킬)** — Claude에게 맥락과 지침을 줘서 행동을 바꾸는 것의 총칭. CLAUDE.md 자체가 이미 "글로벌 스킬"이고, SKILL.md는 그걸 모듈화한 것.
- **참조형 vs 작업형 스킬** — 참조형은 "이런 규칙을 따라"(매뉴얼), 작업형은 "이 단계대로 실행해"(체크리스트).
- **블랙박스화** — 상세 내용을 별도 파일로 분리하고 포인터만 남기는 것. CLAUDE.md처럼 항상 로드되는 파일은 가볍게 유지해야 효율적.
