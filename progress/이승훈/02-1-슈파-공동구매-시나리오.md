## 시나리오 0: 고객 — 첫 주문 시 정보 등록 (회원가입 대체)

```
[언제] 고객이 처음 챗봇에서 주문할 때 (1회만)
[어디서] 카카오톡 → 슈파 챗봇

고객이 처음으로 [주문하기]를 누른다
  → 챗봇: "첫 주문이시네요! 배송을 위해 정보를 등록해주세요"
      이름: ___
      연락처: ___
      배송지 주소: ___
  → 고객이 정보를 입력한다
  → 챗봇: "✅ 등록 완료! 이제 바로 주문하실 수 있어요"
  → 이후부터는 정보 입력 없이 바로 주문 가능
  → (카카오 고유 ID로 자동 식별 — 매번 로그인 불필요)
```

## 시나리오 1: 운영자 — 상품 등록

```
[언제] 새로운 공동구매 상품을 올릴 때
[어디서] 관리자 웹 화면

승훈이 관리자 화면에 접속한다
  → "상품 등록" 버튼을 누른다
  → 상품 정보를 입력한다:
      상품명: 청송 사과
      용량: 10kg
      패키지 단위: 1박스
      가격: 25,000원
      배송비: 직접배송
      공급사: ○○농장
      주문 마감: 매일 15:00
  → "등록" 버튼을 누른다
  → 상품이 목록에 추가된다
  → (선택) 카카오톡 챗봇에 자동 반영됨
```

## 시나리오 2: 고객 — 카카오톡으로 주문하기

```
[언제] 고객이 공동구매 상품을 주문하고 싶을 때
[어디서] 카카오톡 → 슈파 챗봇

고객이 슈파 카카오톡 채널에 들어온다
  → 챗봇: "안녕하세요! 슈파 공동구매입니다"
  → 챗봇: 메뉴 버튼 표시
      [상품 보기] [재주문] [내 주문 확인] [상품 요청] [배송 조회]

고객이 [상품 보기]를 누른다
  → 챗봇: 현재 판매중인 상품 목록 표시
      ┌─────────────────────────┐
      │ 🍎 청송 사과 10kg (1박스) │
      │ 25,000원 | 직접배송       │
      │ ⏰ 15:00 전 주문 → 당일출고│
      │ [주문하기]               │
      ├─────────────────────────┤
      │ 🥔 감자 5kg (1박스)      │
      │ 12,000원 | 배송비 3,000원 │
      │ ⏰ 15:00 전 주문 → 당일출고│
      │ [주문하기]               │
      └─────────────────────────┘

고객이 사과 [주문하기]를 누른다
  → (첫 주문이면 시나리오 0 → 정보 등록 먼저)
  → 챗봇: "청송 사과 10kg — 몇 박스 주문하시겠어요?"
  → 고객: "3"
  → 챗봇: 주문 확인 카드 표시
      ┌─────────────────────────┐
      │ 📋 주문 확인              │
      │ 청송 사과 10kg × 3박스    │
      │ 상품금액: 75,000원        │
      │ 배송비: 직접배송           │
      │ 총 결제금액: 75,000원     │
      │                          │
      │ [카드결제] [주별정산]      │
      └─────────────────────────┘
```
