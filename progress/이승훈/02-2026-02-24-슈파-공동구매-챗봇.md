## 프로젝트 개요

### 만들려는 것
카카오톡 챗봇을 통해 고객이 공동구매 상품을 주문하고, 운영자(승훈)가 관리 화면에서 주문을 확인·수집하는 시스템

### 사용자
- **운영자 (승훈)**: 상품 등록, 주문 확인, 집계
- **고객 (주문자)**: 카카오톡에서 챗봇과 대화하여 주문

---

## 전체 구조 (3가지 부품)

```
고객 (카카오톡)          서버 (뒷방)              운영자 (관리 화면)
┌──────────────┐      ┌──────────────────┐      ┌──────────────────┐
│ 카카오톡 앱   │ ──→  │  주문 접수·저장    │  ←── │  주문 확인·관리    │
│ 챗봇과 대화   │ ←──  │  상품 정보 제공    │  ──→ │  집계·내보내기     │
└──────────────┘      └──────────────────┘      └──────────────────┘
  카카오 채널            Node.js + DB             웹 대시보드
```

## 단계별 계획

### Phase 1 — 관리자 화면 + 서버 기본 (먼저!)
> 챗봇 연결 전에, 뒷방(서버)부터 만들기

- [ ] 상품 등록/관리 화면
- [ ] 주문 목록 보기 화면
- [ ] 발주서 생성 (공급사별 엑셀)
- [ ] 배송 안내 발송
- [ ] 주별 정산 관리
- [ ] 상품 요청 관리
- [ ] 품질 클레임 관리
- [ ] 이탈 고객 감지
- [ ] 서버 만들기 (Node.js + 데이터베이스)
- [ ] 기본 API

**왜 이걸 먼저?**
→ 챗봇이 주문을 저장할 "뒷방"이 없으면 챗봇을 만들어도 소용없음.
→ 관리자 화면이 있어야 테스트할 수 있음.

### Phase 2 — 카카오톡 챗봇 연결
> 고객이 카카오톡으로 주문할 수 있게 연결

- [ ] 카카오 비즈니스 채널 개설
- [ ] 카카오 i 오픈빌더에서 챗봇 시나리오 설계
- [ ] 챗봇 ↔ 서버 연결 (Webhook)
- [ ] 주문 흐름: 상품 보기 → 수량 → 결제
- [ ] 재주문 기능
- [ ] 상품 요청 기능
- [ ] 품질 클레임 접수
- [ ] 주문 합산 기능

**챗봇 대화 예시:**
```
챗봇: 안녕하세요! 슈파 공동구매입니다 🛒
      오늘의 상품을 확인하시겠어요?

고객: 상품 보기

챗봇: 📦 이번 주 공동구매 상품
      1. 사과 10kg — 25,000원 (마감: 2/28)
      2. 감자 5kg — 12,000원 (마감: 2/28)

      주문할 상품 번호를 눌러주세요!

고객: 1

챗봇: 사과 10kg (25,000원)
      몇 개 주문하시겠어요?

고객: 3

챗봇: ✅ 주문 완료!
      사과 10kg × 3개 = 75,000원
      마감일(2/28)에 배송 안내 드릴게요!
```

### 배운 개발 지식
- **서버** — 카페의 주방 같은 것. 손님(고객)은 카운터(카카오톡)에서 주문하고, 주방(서버)에서 접수·처리하고, 사장님(운영자)은 주방 모니터(관리화면)로 확인.
- **API** — 주방과 카운터 사이의 "전달 창구". 카운터에서 "아메리카노 1잔" 전표를 넣으면, 주방에서 그걸 받아서 처리하는 구조.
- **데이터베이스(DB)** — 주문 장부. 모든 주문을 기록해두는 노트. 컴퓨터가 꺼져도 남아있음.
- **시나리오 먼저 쓰기** — 코드를 만들기 전에 "누가 언제 무엇을 하는지" 흐름을 먼저 그리면, 나중에 빠지는 것 없이 만들 수 있음. 설계도 없이 집을 짓지 않는 것과 같음.
- **Node.js** — 서버(주방)를 만들기 위한 도구. 컴퓨터에 설치해야 사용 가능.

### 핵심 의사결정 기록
- 후기/별점 방식 X → **품질 클레임 접수** 방식 (B2B 소상공인 대상이므로 불량만 관리)
- 이탈 고객에게 자동 쿠폰 X → **운영자가 직접 1:1 대화** (관계 기반 운영)
- 마감 알림 X → 15시 전 주문 = 당일출고, 이후 = 익일출고 (안내만)
- 회원가입 X → **첫 주문 시 정보 등록** (이름+연락처+주소만, 1회)

- **Node.js + Express** — 서버 (주방)

- **multer** — 이미지 업로드 처리
- **HTML + CSS + JavaScript** — 관리자 화면 (주방 모니터)

### 배운 개발 지식
- **npm install** — 필요한 도구를 내 프로젝트에 가져오는 명령어. 마트에서 재료를 사오는 것과 비슷.
- **npm start** — 서버를 켜는 명령어. 가게 문을 여는 것.
- **localhost:3000** — 내 컴퓨터에서만 접속 가능한 주소. 아직 인터넷에 올리지 않은 상태.
- **FormData** — 이미지 같은 파일을 서버에 보낼 때 쓰는 방식. 일반 텍스트와 파일을 같이 담는 택배 상자.
