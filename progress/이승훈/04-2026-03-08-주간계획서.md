### 배운 개발 지식

- **Apps Script 승인** — 구글 시트에 붙은 자동화 코드를 처음 실행할 때 "이 코드가 내 시트를 건드려도 괜찮아?"라고 허락받는 과정. 한 번만 하면 됨.
- **xlsx vs exceljs** — xlsx는 기본 엑셀 읽기/쓰기, exceljs는 색상/테두리/스타일까지 세밀하게 제어 가능. 예쁘게 만들려면 exceljs.
- **드롭다운(데이터 유효성 검사)** — 칸을 클릭하면 미리 정한 목록에서 선택하게 하는 기능. 직접 타이핑할 필요 없어서 반복 입력이 편해짐.
- 📐 인터페이스 디자인 (Interface Design) ⚖️
  노션 vs 구글 시트 vs 엑셀 중 선택.
  선택지: ①노션(체크박스 강점, 스프레드시트 약함) ②구글 시트(표+자동화 강점) ③엑셀(로컬, 공유 불편)
  판단: 가로형 한눈에 보기 + 팀 공유 + 자동화(Apps Script) 모두 가능한 구글 시트.
  = 도구는 목적에 맞는 걸 고르는 거지, 유행을 따르는 게 아님 (카페에서 에스프레소 머신으로 국 끓이지 않는 것처럼)

### 어려웠던 것 / 다음에 할 것

- 노션 템플릿은 스프레드시트보다 쓰기 불편했음 → 구글 시트로 확정
- 세로형 레이아웃은 스크롤이 길어짐 → 가로형이 한눈에 보여서 좋음
- xlsx 패키지는 스타일 제한 → exceljs로 교체해서 해결
- **사이드바(도구 패널)** — 구글 시트 오른쪽에 항상 떠 있는 작은 창. 큰 버튼을 넣어서 메뉴를 찾지 않아도 바로 저장/초기화 가능.
- **설치형 트리거(Installable Trigger)** — 시트를 열 때마다 자동으로 뭔가를 실행하게 하는 설정. 최초 1회 설정하면 다음부터 시트 열 때 도구 패널이 자동으로 뜸.
