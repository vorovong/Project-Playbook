# 팀 협업 가이드 — GitHub에 진행 기록 올리기

---

## Part 1: 리더가 할 일 (민호)

동료들이 이 저장소에 파일을 올릴 수 있도록 **권한을 부여**해야 합니다.

### 1단계: 동료의 GitHub 아이디 받기

동료들에게 GitHub 가입 후 **아이디(username)** 를 알려달라고 하세요.
(아이디는 github.com/[여기 나오는 것] 입니다)

### 2단계: Collaborator로 초대하기

1. 브라우저에서 저장소 페이지로 이동합니다
   - https://github.com/vorovong/Project-Playbook
2. 상단 탭에서 **Settings** 클릭
3. 왼쪽 메뉴에서 **Collaborators** 클릭
4. 비밀번호 확인이 나오면 입력
5. **"Add people"** 버튼 클릭
6. 동료의 GitHub 아이디 또는 이메일 입력 → **선택** → **Add**
7. 동료에게 이메일로 초대장이 갑니다

> 동료가 초대 이메일에서 **"Accept invitation"** 을 클릭해야 권한이 생깁니다.
> 이메일이 안 오면 https://github.com/notifications 에서도 확인할 수 있습니다.

---

## Part 2: 동료가 할 일

### 준비: GitHub 계정이 없는 경우

1. https://github.com 에 접속
2. **"Sign up"** 클릭
3. 이메일, 비밀번호, 사용자 이름(아이디)을 입력
   - 사용자 이름은 영문으로, 기억하기 쉬운 걸로 하세요
   - 예: `hong-gildong`, `jimin-kim`
4. 이메일 인증을 완료하세요
5. 완료되면 **아이디를 민호에게 알려주세요** (초대를 보내야 합니다)
6. 민호가 초대를 보내면 **이메일 또는 https://github.com/notifications** 에서 수락하세요

### 진행 기록 올리는 법

플레이북을 이미 다운받고 클로드 코드를 사용하고 있다면,
**클로드 코드에게 말하면 됩니다.** 아래 내용을 그대로 복사해서 붙여넣으세요.

---

#### 방법 A: 클로드 코드에게 시키기 (추천)

하루 스터디가 끝나면, 클로드 코드에게 이렇게 말하세요:

> "오늘 진행 기록 정리해줘.
> 내 이름은 [본인 이름]이고, 프로젝트 이름은 [프로젝트 이름]이야.
> templates/progress-template.md 양식을 따라서,
> 오늘 우리가 나눈 대화를 바탕으로 작성하고 git에 올려줘."

클로드 코드가 알아서:
1. 오늘 대화 내용에서 **내가 한 말, 결과, 배운 개발 지식**을 정리하고
2. `progress/[이름]/[프로젝트].md` 파일을 만들거나 업데이트하고
3. git에 올려줍니다

> 처음 한 번은 git 설정이 필요할 수 있어요. 클로드 코드가 안내해줄 겁니다.

**이 루틴을 매일 하루 끝에 해주세요.** 기록이 쌓이면 나중에 엄청난 자산이 됩니다.

---

#### 방법 B: 직접 하기 (git을 배우고 싶다면)

터미널에서 아래 명령어를 순서대로 입력하세요.

**1. 최신 상태로 업데이트** (다른 동료가 올린 것 받기)
```
cd ~/projects/Project_Playbook
git pull
```

**2. 내 폴더 만들기** (처음 한 번만)
```
mkdir -p progress/[내이름]
```
예시: `mkdir -p progress/홍길동`

**3. 템플릿 복사하기** (새 프로젝트 시작할 때)
```
cp templates/progress-template.md progress/[내이름]/[프로젝트이름].md
```
예시: `cp templates/progress-template.md progress/홍길동/이메일-자동화.md`

**4. 파일 수정하기**
클로드 코드에게 "progress/홍길동/이메일-자동화.md 파일을 열어서 오늘 내용을 적어줘"라고 말하세요.

**5. git에 올리기**
```
git add progress/
git commit -m "홍길동: 이메일-자동화 Day 1 진행 기록"
git push
```

> **git push에서 에러가 나면?**
> 처음 push할 때 GitHub 로그인을 요청할 수 있습니다.
> 브라우저가 열리면 로그인하세요. 한 번 하면 다시 안 물어봅니다.

---

### 미니 프로젝트가 끝나면

프로젝트가 완료되면, 클로드 코드에게 이렇게 말하세요:

> "내 미니 프로젝트가 끝났어. case-studies 폴더에 케이스 스터디를 작성해줘.
> templates/case-study-template.md 양식을 따라서,
> 우리가 나눈 대화를 바탕으로 정리해줘."

## 자주 묻는 질문

**Q: git push 했는데 에러가 나요**
→ 먼저 `git pull`을 해보세요. 다른 동료가 올린 게 있으면 받아야 합니다.
→ 그래도 안 되면 클로드 코드에게 "git push가 안 돼. 도와줘"라고 말하세요.

**Q: 다른 사람 파일을 실수로 수정했어요**
→ 걱정 마세요. `git checkout -- [파일경로]`로 되돌릴 수 있습니다.
→ 또는 클로드 코드에게 "실수로 다른 사람 파일을 수정했어. 되돌려줘"라고 하세요.

**Q: 초대 이메일이 안 와요**
→ 스팸함을 확인하거나, https://github.com/notifications 에서 직접 확인하세요.

**Q: 매번 비밀번호를 물어봐요**
→ 클로드 코드에게 "git push할 때 매번 비밀번호 물어보는데 어떻게 해?"라고 하세요.
→ credential 저장 설정을 해줄 겁니다.
