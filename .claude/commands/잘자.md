# "잘자" — 세션 종료 스킬 (v2: 경량화)

> 무거운 작업은 세션 중 "중간저장"으로 이미 처리됨.
> "잘자"는 마지막 확인 + 인사만. 3단계.

---

## 실행 순서

| 순서 | 행동 | 설명 |
|------|------|------|
| 1 | **중간저장 검증** | 이 세션에서 `/중간저장`이 실행됐는지 확인. 안 됐으면 → `/중간저장` 먼저 실행 후 계속 |
| 2 | **progress 최종 확인** | 중간저장 이후 추가 작업이 있으면 추가분만 갱신. 이미 최신이면 스킵 |
| 3 | **커밋 & 푸시** | git status → 미커밋 있으면 커밋 & 푸시. 클린이면 스킵 |
| 4 | **조언 + 인사** | 승현님 상태 기반 조언 1~2줄 + 인사 |

---

## 이전 10단계는 어디로?

| 이전 단계 | 이동한 곳 | 이유 |
|-----------|----------|------|
| progress 작성 | **중간저장** (세션 중) | 끝에 몰아서 터지는 근본 원인 |
| 대화 원문 보존 | **prompt-backup Hook** (자동) | Hook이 매 프롬프트마다 자동 백업 |
| 내일 할 일 | **중간저장** → progress "다음에 할 것" | progress와 함께 |
| 암묵지 발견 질문 | **"안녕" 브리핑** → 🔮 암묵지 검증 | 다음 세션 시작 시 여유 있게 |
| 성장큐 업데이트 | **중간저장** (세션 중) | progress와 함께 |
| schedule 체크 | **중간저장** (세션 중) | progress와 함께 |
| git status | **잘자 2단계** (유지) | 최종 확인 |
| 결과물 커밋 | **중간저장** + **잘자 2단계** | 이미 중간에 커밋됨 |
| 오늘의 조언 | **잘자 3단계** (유지) | 가벼운 작업 |
| 커밋 & 푸시 | **잘자 2단계** (유지) | 최종 확인 |

> **"중간저장"**: `commands/세션관리.md` 참조
