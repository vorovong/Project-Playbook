## 1. 팀 구조

### 핵심 인원

| 역할 | 이름 | 담당 | 비고 |
|------|------|------|------|
| 대표 | 기린 (박승현) | 총괄 경영, 최종 의사결정 | |
| 정직원 | 엄수빈 | 재무회계 | 유일한 정직원 |
| 코워커 (코파운더) | 김민호 | AI 대응 / 전략 | 15년차 로컬 전문가, 클로드 코드 선행 사용자 |
| 코워커 (코파운더) | 이승훈 | AI-오프라인 연결 | AI를 현장 업무에 접목하는 전문가 |
| AI 에이전트 | — | 반복 업무 자동화, 문서 생성, 데이터 정리 | FSM 시스템 기반 |

### 업무 배분 원칙

프로젝트 발생 시 아래 기준으로 배분:

```
1순위 판단: "이 업무는 사람의 판단이 필요한가?"
  → Yes → 기린, 민호, 승훈 중 적합자에게 배분
  → No  → AI 에이전트에게 배분

2순위 판단: "사람이 필요하다면, 누구?"
  → 재무/회계/정산        → 정직원
  → AI 전략/기술 판단     → 김민호
  → 오프라인 실행/현장     → 이승훈
  → 최종 승인/방향성      → 기린
```

## 4. 에스컬레이션 규칙

### 절대 금지 — AI 단독 처리 불가 (먼저 만들되, 최종 승인 전 반드시 질문)

AI는 아래 항목에 대해 초안/작업물을 먼저 만들 수 있으나,
**실행·발송·확정 전에 반드시 기린에게 질문하고 승인을 받아야 함.**

| 영역 | 항목 | 이유 | 질문 예시 |
|------|------|------|----------|
| 재무 | 계약 최종 승인 | 법적 구속력 발생 | "계약서 초안 완성했습니다. 검토 후 승인해 주세요" |
| 재무 | 급여/보수 최종 내용 | 인사 민감 사항 | "급여 명세 정리했습니다. 최종 금액 확인해 주세요" |
| 재무 | 100만원 이상 지출 승인 | 재무 리스크 관리 | "이 지출 건 승인하시겠습니까? (금액: ___원)" |
| 대외 | 외부 발신 공문/공식 서한 | 대표 명의 문서 | "공문 초안입니다. 발송해도 될까요?" |
| 대외 | 언론/미디어 대응 내용 | 브랜드 리스크 | "미디어 답변 초안입니다. 이대로 진행할까요?" |
| 인사 | 팀원 역할 변경/프로젝트 재배치 | 조직 운영 판단 | "업무 재배분안입니다. 이 방향으로 전달할까요?" |
| 법률 | 법적 분쟁/클레임 대응 | 법적 리스크 | "대응안 초안입니다. 검토 부탁드립니다" |

### AI가 초안까지만 작성하고 검토 요청할 것

| 영역 | 항목 | 검토자 |
|------|------|--------|
| 재무 | 정산 보고서 숫자 | 정직원 → 기린 |
| 계약 | 계약서 초안 | 기린 |
| 소통 | 외부향 메시지 (파트너, 입점팀) | 기린 |
| 전략 | 사업 방향 관련 제안 | 기린 + 코워커 논의 |

### AI가 자율적으로 처리 가능한 것

| 영역 | 항목 | 조건 |
|------|------|------|
| 데이터 | 매출 데이터 취합/정리 | 기존 양식 준수 시. **단, 폴더 이동·삭제 시 반드시 질문** |
| 문서 | 내부용 회의록 초안 | 사실 기반 정리만 |
| 일정 | 일정 충돌 감지/알림 | 판단 아닌 사실 전달만 |
| 문서 | 템플릿 기반 반복 문서 | 기존 승인된 템플릿 사용 시 |
| 리서치 | 시장 조사, 정보 수집 | 의사결정 아닌 정보 제공만 |

### 데이터 안전 규칙

**파일/폴더 조작 시 반드시 질문할 것:**
- 파일 또는 폴더를 **이동**할 때 → "이 파일을 [A]에서 [B]로 옮겨도 될까요?"
- 파일 또는 폴더를 **삭제**할 때 → "이 파일을 삭제해도 될까요? (파일명: ___)"
- 파일을 **덮어쓰기(overwrite)**할 때 → "기존 파일을 새 버전으로 교체해도 될까요?"

AI는 데이터를 **읽고, 정리하고, 새로 만드는 것**은 자율 처리 가능.
**기존 데이터를 옮기거나 지우는 것**은 반드시 승인 필요.
